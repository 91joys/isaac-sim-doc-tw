感測器（Sensors）#

對應官方 Sensors 章節的中文導覽。Isaac Sim 的感測器分三大家族：相機、RTX 感測器（光線追蹤模擬）、物理式感測器（物理引擎資料），各有適用場景。

怎麼選感測器#

需求 家族 原理
RGB／深度影像、視覺感知 相機感測器 渲染管線
光達／雷達點雲（高擬真、材質反射） RTX 感測器 RTX 光線追蹤，支援非視覺材質屬性
IMU、接觸力、關節力矩 物理式感測器 直接讀物理引擎狀態
簡單射線量測（高效能、低擬真） 物理射線／PhysX 感測器 物理場景查詢

6.0 重要變更：相機、RTX 與物理感測器的 API 皆已改版為 Experimental API。從 4.x/5.x 升級請看遷移指南（含相機、RTX、物理感測器、PhysX 光達等逐項對照）。

相機感測器（Camera）#

總覽：

  • RGB 相機：焦距、光圈、裁切等以 USD 相機屬性設定（程式取得相機參數的片段見《完整文檔》6.4 節）。
  • 深度感測器——立體深度相機模擬。
  • 結構光相機——投射圖案的主動式深度。

現成相機資產（RealSense 等品牌型號）見 Camera and Depth Sensor Assets。

RTX 感測器#

總覽——用 RTX 光線追蹤模擬主動式感測器，可考慮材質反射率等非視覺屬性：

感測器 連結
RTX Lidar（光達） 說明
RTX Radar（雷達） 說明
RTX Acoustic（聲學） 說明

配套主題：

  • RTX Sensor Annotators——取得點雲與標註輸出。
  • 非視覺材質——設定材質對光達／雷達的反射特性。
  • Multi-Tick Rendering——一個渲染 frame 內多次感測器取樣。
  • 自訂 RTX 感測器 Profile——描述市售型號的掃描模式與參數。

現成光達等資產見 Non-Visual Sensor Assets。

物理式感測器（Physics-based）#

總覽——從物理引擎直接取資料，無需渲染：

感測器 量測 連結
IMU 加速度、角速度、姿態 說明
Contact（接觸） 接觸力、接觸事件 說明
Articulation joint（關節） 關節內力／力矩 說明
Effort 關節驅動力 說明
Joint state 關節位置／速度 說明
Physics raycast 射線距離量測 說明

本站提供 IMU 感測器教學——用已知運動剖面驗證 AMR 移動是否符合預期，含完整 Standalone 驗證腳本與 ROS 2 sim-to-real 比對流程。

PhysX SDK 感測器#

總覽——基於 PhysX 場景查詢的輕量感測器：泛用感測器、PhysX 光達、光束感測器、近接感測器。

注意：6.0 起 PhysX 光達／泛用／光束感測器已被 Physics raycast 感測器取代（見遷移指南），新專案優先使用後者。

與 ROS 2／SDG 的銜接#

  • 感測器資料發布到 ROS 2：見本站《ROS 2》章節頁（相機、RTX 光達、RTX 雷達教學）。
  • 感測器輸出作為訓練資料：見本站《合成資料生成》章節頁（Annotator＋Writer）。
  • 感測器放置與校準工具：RTX Sensors Placement and Calibration。
