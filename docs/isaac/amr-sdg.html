Replicator 教學——AMR 導航隨機化與資料擷取#

對應官方 Randomization in Simulation – AMR Navigation 教學（Isaac Sim 6.0.1）的中文整理。這是合成資料生成章節的實戰進階教學：讓一台 AMR（自主移動機器人）在隨機化的環境中導航，並在「模擬事件」發生時擷取合成資料——與固定觸發、固定場景的入門範例相比，更貼近真實專案的 SDG 管線。

學習目標與前置知識#

本教學示範如何把 Isaac Sim 模擬場景與 omni.replicator 擴充功能組合起來，用多種隨機化技巧擷取合成資料。你將學會：

  • 用 USD／Isaac Sim API 實作場景隨機化：隨機化資產姿態、切換不同的背景環境。
  • 在特定模擬事件發生時（而不是固定間隔）用 Replicator 收集合成資料。
  • 動態建立與銷毀 render product 以提升執行期效能。
  • 在同一個模擬實例中建立與銷毀 Replicator 擷取圖（capture graph）。

前置知識：USD／Isaac Sim 場景建立與操作 API、omni.replicator 與其 Writer、OmniGraph 基礎（導航實作用到）、以及 Standalone 應用或 Script Editor 執行模擬的方式。

情境總覽#

教學使用 Nova Carter 機器人，配備 OmniGraph 導航堆疊（不含避障功能）。整個資料生成迴圈如下：

  1. 導航堆疊持續把機器人推向指定的目標 Xform（<..>/targetXform）——目標被放在隨機化的目標物（dolly 推車）位置上。
  2. 機器人接近目標物時，觸發 SDG 管線，從兩顆主相機感測器擷取資料。
  3. 擷取完成後重新隨機化目標物，模擬繼續。
  4. 每擷取 env_interval 幀就更換背景環境；達到 num_frames 後應用程式結束。

資料來源是 Nova Carter 的左右相機（stereo_cam_<left/right>_sensor_frame/camera_sensor_<left/right>），用 Replicator 收集 LdrColor（RGB）annotator 資料。預設寫到 <工作目錄>/_out_nav_sdg_demo，執行 num_frames=9 次、每 env_interval=3 幀換一次環境。背景環境依 DEFAULT_ENV_URLS 輪替；清單裡的 None 表示不載入 USD 環境，改在 /Environment 下用 Replicator 的 dome light＋帶碰撞體的平面組出通用（generic）環境。

use_temp_rp 旗標是本教學的效能重點：只在擷取資料時啟用 render product，平時停用——避免不擷取時仍持續渲染感測器視角的開銷，大幅加速模擬。

執行方式#

Standalone 應用#

以隨附腳本作為獨立應用執行（Windows 用 python.bat 取代 python.sh）：

./python.sh standalone_examples/replicator/amr_navigation.py

# 帶全部參數的範例
./python.sh standalone_examples/replicator/amr_navigation.py --use_temp_rp --num_frames 9 --env_interval 3
參數說明預設
--use_temp_rp使用臨時 render product（僅擷取時啟用）False
--num_frames要擷取的幀數9
--env_interval每幾幀更換一次背景環境3
--env_urls完全取代 DEFAULT_ENV_URLS；填 None 表示通用環境—

Standalone 版的進入點：SimulationApp 啟動後建立 NavSDGDemo 並呼叫 start()，主迴圈持續 update() 直到示範結束：

out_dir = os.path.join(os.getcwd(), "_out_nav_sdg_demo", "")
selected_env_urls = args.env_urls if args.env_urls is not None else DEFAULT_ENV_URLS
nav_demo = NavSDGDemo()
nav_demo.start(
    num_frames=args.num_frames,
    out_dir=out_dir,
    env_urls=selected_env_urls,
    env_interval=args.env_interval,
    use_temp_rp=args.use_temp_rp,
    seed=22,
)

while simulation_app.is_running() and nav_demo.is_running():
    simulation_app.update()

simulation_app.close()

Script Editor#

同一個 NavSDGDemo 類別也能貼進 Script Editor 執行，差異在於：

  • 不建立 SimulationApp（編輯器已在執行中），參數改用常數 NUM_FRAMES／ENV_INTERVAL／USE_TEMP_RP。
  • 入口改為 asyncio.ensure_future(nav_demo.run_async(...))——run_async 內部建立 asyncio.Event，start() 後等待完成訊號（clear() 時 set）。
  • SDG 步驟走非同步路徑：await rep.orchestrator.step_async(...)，避免凍結編輯器 UI。腳本用 builtins.ISAAC_LAUNCHED_FROM_TERMINAL 判斷目前執行環境，自動選同步或非同步版本。

完整的兩份腳本（Standalone Script／Script Editor Script）見官方 Implementation 節，以下解析共同的核心實作。

NavSDGDemo 類別與屬性#

整個示範包在 NavSDGDemo 類別裡，類別常數指向用到的資產與路徑：

class NavSDGDemo:
    """Demonstration of synthetic data generation using an AMR navigating towards a target."""

    CARTER_URL = "/Isaac/Samples/Replicator/OmniGraph/nova_carter_nav_only.usd"
    DOLLY_URL = "/Isaac/Props/Dolly/dolly.usd"
    PROPS_URL = "/Isaac/Props/YCB/Axis_Aligned_Physics"
    LEFT_CAMERA_REL_PATH = "sensors/front_hawk/left/camera_left"
    RIGHT_CAMERA_REL_PATH = "sensors/front_hawk/right/camera_right"
    ENVIRONMENT_SCOPE_PATH = "/Environment"
屬性用途
_carter_chassis・_carter_nav_target追蹤 Nova Carter 底盤（目前位置）與導航圖中的目標 Xform
_dolly目標物推車——既是導航目標的落點，也用來計算與 Carter 的距離
_dolly_light每幀隨機化、放在推車上方的球形光
_props每幀灑在推車上方、以物理落下的道具 prim 清單（YCB 物件）
_cycled_env_urls輪替的背景環境路徑（含 None＝通用 Replicator 環境）
_env_interval每幾幀更換背景環境
_timeline・_timeline_sub控制模擬時間軸（play／pause）；timeline tick 訂閱是觸發 SDG 的回饋迴圈
_stage_event_sub訂閱 stage 關閉事件——開新 stage 時自動清理示範狀態
_trigger_distanceCarter 與推車間觸發 SDG 的距離，每次擷取後重新隨機化（1.75–2.5）
_num_frames・_frame_counter追蹤並在達到指定幀數後停止
_writer・_render_products寫資料到磁碟的 Writer；掛在左右相機上的兩個 render product
_use_temp_rpTrue 時非擷取期間停用 render product
_in_running_state示範是否仍在執行（供主迴圈／is_running() 查詢）

類別常數 ENVIRONMENT_SCOPE_PATH 讓「引用的 USD 環境」與「通用備援環境」都放在同一個 /Environment scope 之下，兩者間的切換因此走同一條更新路徑。

工作流程：start 與事件迴圈#

主流程由 start() 與 _on_timeline_event 回呼組成。start() 解析環境清單、建立含導航物理場景、Nova Carter（含導航圖與目標 Xform）、推車、隨機化光源與道具的新場景，設定 SDG，播放時間軸，並註冊兩個事件訂閱：

def start(self, num_frames=10, out_dir=None, env_urls=None,
          env_interval=3, use_temp_rp=False, seed=None) -> None:
    """Start the SDG demo with the given configuration."""
    print(f"[SDG] Starting")
    if seed is not None:
        rep.set_global_seed(seed)
        random.seed(seed)
    selected_env_urls = env_urls if env_urls is not None else DEFAULT_ENV_URLS
    self._num_frames = num_frames
    self._out_dir = out_dir if out_dir is not None else os.path.join(os.getcwd(), "_out_nav_sdg_demo")
    self._cycled_env_urls = cycle(selected_env_urls)
    self._env_interval = env_interval
    self._use_temp_rp = use_temp_rp
    self._frame_counter = 0
    self._trigger_distance = 2.0
    self._load_env()
    self._randomize_dolly_pose()
    self._randomize_dolly_light()
    self._randomize_prop_poses()
    self._setup_sdg()
    self._timeline = omni.timeline.get_timeline_interface()
    self._timeline.play()
    self._timeline_sub = carb.eventdispatcher.get_eventdispatcher().observe_event(
        event_name=omni.timeline.GLOBAL_EVENT_CURRENT_TIME_TICKED,
        on_event=self._on_timeline_event,
        observer_name="amr_navigation.NavSDGDemo._on_timeline_event",
    )
    self._stage_event_sub = carb.eventdispatcher.get_eventdispatcher().observe_event(
        event_name=omni.usd.get_context().stage_event_name(omni.usd.StageEventType.CLOSING),
        on_event=self._on_stage_closing_event,
        observer_name="amr_navigation.NavSDGDemo._on_stage_closing_event",
    )
    self._in_running_state = True

_on_timeline_event 在每個 timeline tick 檢查 Carter 與推車的 2D 距離——夠近就暫停模擬、觸發 SDG。這就是「事件驅動擷取」：由模擬狀態決定何時拍照，而非固定間隔：

def _on_timeline_event(self, e: carb.eventdispatcher.Event):
    """Check distance to dolly and trigger SDG capture when close enough."""
    carter_loc = self._carter_chassis.GetAttribute("xformOp:translate").Get()
    dolly_loc = self._dolly.GetAttribute("xformOp:translate").Get()
    dist = (Gf.Vec2f(dolly_loc[0], dolly_loc[1]) - Gf.Vec2f(carter_loc[0], carter_loc[1])).GetLength()
    if dist < self._trigger_distance:
        print(f"[SDG] Starting SDG for frame no. {self._frame_counter}")
        self._timeline.pause()
        if self._is_running_in_script_editor():
            import asyncio
            task = asyncio.ensure_future(self._run_sdg_async())
            task.add_done_callback(self._on_sdg_done)
        else:
            self._run_sdg()
            self._setup_next_frame()

場景建立與環境切換#

_load_env() 建立新 stage 後依序放入：

  • 物理場景：rep.functional.physics.create_physics_scene("/PhysicsScene", enableCCD=True, broadphaseType="MBP", enableGPUDynamics=False)——為導航調校的設定。
  • 背景環境：self._load_environment(next(self._cycled_env_urls))。
  • Nova Carter：以 reference 載入 nova_carter_nav_only.usd 到 /NavWorld，走訪子節點找出 targetXform（導航目標）與 chassis_link（目前位置）。
  • 推車（dolly）：載入後對其幾何 prim 逐一 rep.functional.physics.apply_rigid_body() 加上剛體／碰撞。
  • 光源：rep.functional.create.sphere_light(intensity=250000, radius=0.3, ...)。
  • 道具：用 omni.client.list() 列出 YCB 資產資料夾的 .usd，輪替取 15 個以 reference 掛到 /NavWorld/Props/ 下。

_load_environment(env_url) 是環境切換的統一入口：先刪掉舊的 /Environment，再重建——env_url 有值就 reference 載入該 USD 場景；是 None 就組出通用環境：

def _load_environment(self, env_url: str | None) -> None:
    """Load the next environment under a shared scope."""
    if self._stage.GetPrimAtPath(self.ENVIRONMENT_SCOPE_PATH):
        omni.kit.commands.execute("DeletePrimsCommand", paths=[self.ENVIRONMENT_SCOPE_PATH])

    rep.functional.create.scope(name="Environment")
    if env_url:
        assets_root_path = get_assets_root_path()
        rep.functional.create.reference(
            usd_path=assets_root_path + env_url, parent=self.ENVIRONMENT_SCOPE_PATH, name="Scene"
        )
        return

    # env_url 為 None：通用環境 = dome light + 帶碰撞體的地板
    rep.functional.create.dome_light(intensity=500, parent=self.ENVIRONMENT_SCOPE_PATH, name="DomeLight")
    ground = rep.functional.create.plane(
        parent=self.ENVIRONMENT_SCOPE_PATH, name="GroundPlane", scale=(100, 100, 1)
    )
    rep.functional.physics.apply_collider(ground)

隨機化#

每次擷取前用三個函式隨機化場景——注意這裡示範的是直接用 USD API 隨機化（設定 xformOp 屬性），與 Replicator 的宣告式隨機化 API 互補：

  • _randomize_dolly_pose：在 ±6m 範圍內抽樣推車位置，要求與 Carter 至少相距 4m（最多重試 100 次）；找到後把導航目標也設到推車位置，再隨機化朝向。
  • _randomize_dolly_light：把光源放到推車上方 3m，顏色隨機。
  • _randomize_prop_poses：把道具逐一疊放在推車上方（水平隨機偏移、垂直每個墊高 0.2），模擬開始後自然掉落。
def _randomize_dolly_pose(self) -> None:
    """Set random dolly position ensuring minimum distance from Carter."""
    min_dist_from_carter = 4
    carter_loc = self._carter_chassis.GetAttribute("xformOp:translate").Get()
    for _ in range(100):
        x, y = random.uniform(-6, 6), random.uniform(-6, 6)
        dist = (Gf.Vec2f(x, y) - Gf.Vec2f(carter_loc[0], carter_loc[1])).GetLength()
        if dist > min_dist_from_carter:
            self._dolly.GetAttribute("xformOp:translate").Set((x, y, 0))
            self._carter_nav_target.GetAttribute("xformOp:translate").Set((x, y, 0))
            break
    self._dolly.GetAttribute("xformOp:rotateXYZ").Set((0, 0, random.uniform(-180, 180)))

SDG 管線：Writer 與 render product#

_setup_sdg() 做四件事：

  1. rep.orchestrator.set_capture_on_play(False)——關閉「播放即擷取」，改由我們手動控制擷取時機。
  2. 把 DLSS 設成 Quality 模式（rtx/post/dlss/execMode = 2）取得最佳 SDG 影像品質（選項：0 Performance／1 Balanced／2 Quality／3 Auto）。
  3. 調整相機：fStop = 0 取得明亮銳利的影像；omni:sensor:tickRate = 0 強制 autotrigger。
  4. 初始化 DiskBackend（輸出目錄）＋ BasicWriter（rgb=True），並建立 render product。

注意（multi-tick rendering 的暫時性解法）：在多 tick 渲染下，感測器各自的 tick 排程可能與 rep.orchestrator.step_async 脫鉤，導致 Writer 收不到影格；把 front_hawk 左右相機的 omni:sensor:tickRate 設為 0（autotrigger）能讓它們與 orchestrator 保持同步。此 workaround 預期未來版本會移除，詳見官方 Multi-Tick Rendering。

兩個 render product 掛在左右相機上（1024×1024），use_temp_rp 開啟時先停用、只在擷取時啟用：

def _setup_sdg_render_products(self) -> None:
    """Create and attach render products for left and right cameras."""
    left_camera_path = self._carter_chassis.GetPath().AppendPath(self.LEFT_CAMERA_REL_PATH)
    rp_left = rep.create.render_product(str(left_camera_path), (1024, 1024),
                                        name="left_sensor", force_new=True)
    right_camera_path = self._carter_chassis.GetPath().AppendPath(self.RIGHT_CAMERA_REL_PATH)
    rp_right = rep.create.render_product(str(right_camera_path), (1024, 1024),
                                         name="right_sensor", force_new=True)
    self._render_products = [rp_left, rp_right]
    # 效能考量：非使用期間可停用 render product，只在 SDG 時重新啟用
    if self._use_temp_rp:
        self._disable_render_products()
    self._writer.attach(self._render_products)

def _enable_render_products(self) -> None:
    for rp in self._render_products:
        rp.hydra_texture.set_updates_enabled(True)

def _disable_render_products(self) -> None:
    for rp in self._render_products:
        rp.hydra_texture.set_updates_enabled(False)

實際擷取一步呼叫 rep.orchestrator.step(rt_subframes=16)——rt_subframes 讓渲染器在擷取前多累積子影格，改善光線追蹤品質（Script Editor 走 step_async 版本）：

def _run_sdg(self) -> None:
    """Execute one SDG capture step synchronously."""
    if self._use_temp_rp:
        self._enable_render_products()
    rep.orchestrator.step(rt_subframes=16)
    if self._use_temp_rp:
        self._disable_render_products()

對應地，_clear_sdg_render_products() 會 detach writer、rp.destroy() 銷毀所有 render product，並刪除 /Replicator prim——這就是「在同一個模擬實例中建立與銷毀擷取圖」的作法。

下一幀與收尾#

SDG 完成後 _setup_next_frame() 準備下一幀：遞增幀數計數器；重新隨機化推車、光源與道具；達到 env_interval 就換背景環境；抽一個新的觸發距離（1.75–2.5）；重新播放時間軸並重掛 timeline 訂閱。若已達 _num_frames，先確保資料完整寫入磁碟（rep.orchestrator.wait_until_complete()，Script Editor 用 async 版本）再 clear() 收尾：

def _setup_next_frame(self) -> None:
    """Prepare scene for next frame or finish if all frames captured."""
    self._frame_counter += 1
    if self._frame_counter >= self._num_frames:
        print(f"[SDG] Finished")
        # 清理狀態前，先確定資料已寫入磁碟
        if self._is_running_in_script_editor():
            import asyncio
            task = asyncio.ensure_future(rep.orchestrator.wait_until_complete_async())
            task.add_done_callback(lambda t: self.clear())
        else:
            rep.orchestrator.wait_until_complete()
            self.clear()
        return

    self._randomize_dolly_pose()
    self._randomize_dolly_light()
    self._randomize_prop_poses()
    if self._frame_counter % self._env_interval == 0:
        self._load_next_env()
    # 為下一幀抽一個新的隨機觸發距離
    self._trigger_distance = random.uniform(1.75, 2.5)
    self._timeline.play()
    self._timeline_sub = carb.eventdispatcher.get_eventdispatcher().observe_event(
        event_name=omni.timeline.GLOBAL_EVENT_CURRENT_TIME_TICKED,
        on_event=self._on_timeline_event,
        observer_name="amr_navigation.NavSDGDemo._on_timeline_event",
    )

可帶走的模式#

這個教學值得抄進自己專案的幾個模式：

  • 事件驅動擷取：關掉 capture_on_play，用 timeline tick 訂閱檢查模擬狀態（距離、碰撞、任務階段…），條件成立才 pause() → orchestrator.step()——資料集只收「有意義的瞬間」。
  • 臨時 render product：rp.hydra_texture.set_updates_enabled(False) 是低成本的開關，機器人巡航的漫長過程不必陪著渲染兩顆 1024×1024 相機。
  • 統一的環境 scope：把可替換的內容都放在同一個 scope（/Environment）下，換場景＝刪 scope＋重建，USD 環境與程式生成環境走同一條路。
  • USD API 隨機化 × Replicator 擷取：隨機化不一定要用 Replicator 的分佈 API——普通的 random＋GetAttribute().Set() 同樣有效，且更容易表達「與機器人至少距離 4m」這類含約束的邏輯。
  • 雙模式腳本：用 builtins.ISAAC_LAUNCHED_FROM_TERMINAL 區分 Standalone／Script Editor，同一類別同時支援同步與 async 兩種執行路徑。
  • 可重現性：rep.set_global_seed(seed)＋random.seed(seed) 讓整條隨機化管線可重現（範例用 seed=22）。
