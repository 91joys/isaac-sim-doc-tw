合成資料生成（Synthetic Data Generation, SDG）#

對應官方 Synthetic Data Generation 章節的中文導覽。SDG 是 Isaac Sim 的招牌能力之一：在模擬中大量產生帶標註的訓練資料（RGB、深度、分割、邊界框、姿態…），餵給感知模型與機器人學習管線。

核心概念#

  • Replicator 是 Omniverse 的 SDG 框架：以 Python API 描述「隨機化什麼、擷取什麼、寫到哪」，一次定義、批次執行。
  • Domain Randomization（域隨機化）：隨機化光照、材質、姿態、相機位置等，讓模型學到對真實世界變化魯棒的特徵。
  • 語意標註（Semantics）：先給場景物件貼上 class 標籤，annotator 才知道要輸出什麼標註（程式化批次貼標籤的片段見《完整文檔》6.4 節）。
  • Writer：決定輸出格式（KITTI、COCO 等，也可自訂 Writer 寫任意格式）。

Replicator（感知資料生成）#

教學總覽。建議路徑：

  1. Overview 總覽——理解 Replicator 的組成。
  2. Synthetic Data Recorder——GUI 錄製器，不寫程式也能出資料集，入門首選。
  3. Getting Started Scripts——第一批 SDG 腳本。
  4. SDG Workflows——各種工作流程的選擇指南。
  5. 兩大基本範式：場景式資料生成（固定場景中隨機化）與物件式資料生成（聚焦目標物件）。

動作與事件資料生成#

章節總覽——超越單張影像，生成行為與事件層級的資料：

  • Replicator Agent（人物模擬）：以設定檔驅動的人物／機器人行為模擬與資料錄製（含設定檔指南、範例設定、RTSP 自訂 Writer 範例）。
  • 行為樹生成：以行為樹描述複雜情境。
  • 物件模擬與資料生成（Replicator Object）：宣告式設定檔驅動的物件隨機化（Mutable、Camera、Light、Harmonizer 等子系統），含 Chat IRO 自然語言介面。
  • VLM 場景描述、實體空間事件生成、事件觸發的演員反應。
  • RTX 感測器放置與校準、Omni Metropolis 管線。
