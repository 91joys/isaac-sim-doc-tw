訓練工作流程（Training Recipe）#

整理自官方 Training Recipe 與 PhysicsNeMo for PyTorch Users 章節。

PhysicsNeMo 的訓練管線就是標準 PyTorch 訓練迴圈：資料（datapipe）→ 可訓練模型 → 損失 → 優化器。差別在於每個環節都可以換上 SciML 專用、GPU 最佳化的元件，並附贈 checkpoint、日誌、分散式等品質工具。

用內建模型訓練：FNO × Darcy2D#

官方入門範例：學習 2D Darcy 流「滲透率場 → 壓力場」的映射。PhysicsNeMo 內建基準資料管線（如 Darcy2D），免設定資料載入即可原型開發：

import torch
import physicsnemo
from physicsnemo.datapipes.benchmarks.darcy import Darcy2D
from physicsnemo.metrics.general.mse import mse
from physicsnemo.models.fno.fno import FNO

normaliser = {  # 滲透率與 Darcy 場的平均值／標準差
    "permeability": (1.25, 0.75),
    "darcy": (4.52e-2, 2.79e-2),
}
dataloader = Darcy2D(
    resolution=256, batch_size=64, nr_permeability_freq=5, normaliser=normaliser
)
model = FNO(
    in_channels=1, out_channels=1,
    decoder_layers=1, decoder_layer_size=32,
    dimension=2, latent_channels=32,
    num_fno_layers=4, num_fno_modes=12, padding=5,
).to("cuda")

optimizer = torch.optim.Adam(model.parameters(), lr=0.01)
scheduler = torch.optim.lr_scheduler.LambdaLR(optimizer, lr_lambda=lambda step: 0.85**step)

dataloader = iter(dataloader)
for i in range(20):
    batch = next(dataloader)
    pred = model(batch["permeability"])
    loss = mse(pred, batch["darcy"])
    loss.backward()
    optimizer.step()
    scheduler.step()
    print(f"Iteration: {i}. Loss: {loss.detach().cpu().numpy()}")

內建模型高度可配置，開箱即可套用到不同應用；完整清單見模型架構與官方 Models API。

自訂模型：從 PyTorch 到 physicsnemo.Module#

內建模型不合用時，寫自己的模型也能享受 PhysicsNeMo 的 checkpoint 與 model registry：

  • 新模型：繼承 physicsnemo.Module（取代 torch.nn.Module）即可。
  • 既有 PyTorch 模型：不想改原始碼，用 physicsnemo.Module.from_torch() 一行轉換。
  • 注意：要用 checkpoint 功能，模型建構參數必須可 JSON 序列化。

PhysicsNeMo 模型與標準 PyTorch 模型完全互通——任何 physicsnemo.Module（或普通 torch.nn.Module）都能直接搭配 physicsnemo.sym 的 PhysicsInformer 做物理資訊訓練，不需要包裝類別。細節見官方 Modules 文檔。

推論：儲存與載入模型#

# 訓練後儲存（.mdlus 格式：連建構參數一起存）
model.save("checkpoint.mdlus")

# 推論端：不需要知道原始超參數，直接重建
model_inf = physicsnemo.Module.from_checkpoint("checkpoint.mdlus").to("cuda")
model_inf.eval()

with torch.inference_mode():
    input = torch.ones(8, 1, 256, 256).to("cuda")
    output = model_inf(input)

要載入含 optimizer／scheduler 狀態的訓練 checkpoint（續訓），用 physicsnemo.launch.utils 的 save_checkpoint／load_checkpoint 工具。

日誌與 Checkpoint#

工具用途
Console logging（PythonLogger／LaunchLogger）標準化的訓練輸出格式，記錄超參數與每輪損失
MLflow 整合initialize_mlflow() 後 LaunchLogger 自動寫入實驗追蹤
Weights & Biases 整合initialize_wandb()，適合團隊儀表板
Checkpointingsave_checkpoint()／load_checkpoint() 一次存讀模型、優化器、排程器與步數，支援中斷續訓

安裝 utils-extras extra 可取得 MLflow／W&B 相依：pip install "nvidia-physicsnemo[utils-extras]"。

升級為分散式訓練#

同一份腳本改成資料平行只需三步：初始化 DistributedManager、把模型搬到 dist.device、包一層 DistributedDataParallel：

from physicsnemo.distributed import DistributedManager
from torch.nn.parallel import DistributedDataParallel

def main():
    DistributedManager.initialize()   # 自動偵測啟動的程序數
    dist = DistributedManager()

    model = FNO(...).to(dist.device)
    if dist.distributed:
        model = DistributedDataParallel(
            model,
            device_ids=[dist.local_rank],
            output_device=dist.device,
            broadcast_buffers=dist.broadcast_buffers,
            find_unused_parameters=dist.find_unused_parameters,
        )
    # ……訓練迴圈不變……

if __name__ == "__main__":
    main()

用 torchrun --nproc_per_node=8 train.py（或 mpirun／SLURM）啟動。深入見分散式與領域平行。

標準配方清單#

  1. 選資料管線：內建 physicsnemo.datapipes（基準／氣候／GNN／CAE）或自己的 PyTorch DataLoader。
  2. 選模型：physicsnemo.models 內建家族，或 Module.from_torch() 包裝自家模型。
  3. 選損失：資料損失（MSE…）＋可選的物理殘差損失。
  4. 標準 torch.optim 優化器與排程器。
  5. 掛上日誌（MLflow／W&B）與 checkpoint 工具。
  6. 需要時以 DistributedManager + DDP 規模化；資料單卡放不下時考慮 ShardTensor。
