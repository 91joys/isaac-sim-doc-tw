效能最佳化#

整理自官方 Performance 章節及其三篇子教學：Profiling、Torch Compile、Optimized Layers。

先想 Amdahl 定律#

「最佳化系統單一部分所獲得的整體效能提升，受限於該部分實際被使用的時間比例。」

AI 應用是端到端的 CPU＋GPU 系統：如果 90% 時間卡在 I/O 與前處理，把模型加速一倍毫無意義。優先順序永遠由 profiling 實測決定，改完要重新 profile 再決定下一步。四個關鍵子系統：

子系統瓶頸樣態工具箱
模型計算kernel 效率低、launch 開銷大torch.compile、CUDA Graphs、混合精度（AMP）、NCCL、cuML／cuGraph／Warp 專用 kernel
資料載入SciML 資料集動輒 TB 級，GPU 等資料HDF5／Zarr、NVIDIA DALI、平行 I/O
資料前處理正規化、裁切、增強佔用 CPU，餓死 GPU搬上 GPU（PhysicsNeMo-Mesh、DALI、Warp）
規模化單卡快、多卡不動；checkpoint／指標聚合拖慢DDP 調校、平行 I/O、領域平行

Profiling#

官方教學以一個真實應用示範端到端加速流程。要點：

  • 多工具互補：PyTorch Profiler（op 級熱點＋TensorBoard 視覺化）、NVIDIA Nsight Systems（系統級時間線：CPU／GPU／通訊重疊）、line profiler（Python 層瓶頸）。
  • PhysicsNeMo 提供 physicsnemo.utils.profiling 包裝工具，統一掛 profiler 到訓練迴圈。
  • 先看全貌（GPU 使用率、空隙）再鑽 kernel；I/O 空隙優先於 kernel 微調。

torch.compile 與外部 kernel#

Torch Compile and External Kernels 教你把兩個世界接起來：

  • torch.compile 做端到端模型編譯（kernel 融合、去除 Python 開銷）。
  • 外部自訂 kernel（Warp、Triton、CUDA）透過 torch.library 自訂 op 註冊，讓 compile 不會在邊界斷圖（graph break）。
  • PhysicsNeMo 模型逐步最佳化為 compile 友善（StormCast／CorrDiff 的 fused ops 即是範例）；PhysicsNeMo-Mesh 亦相容 torch.compile。

GPU 最佳化層與函數#

GPU Optimized Layers and Functions：用 physicsnemo.nn 的層與 functional API 加速算子密集的 SciML 工作負載——kNN、radius search、SDF、ball query 等在 v2.0 整併為單一模組化介面（見 API 模組地圖）。實戰中的效能配置範例：

配方效能手段
Transolver 外氣動力Transformer Engine：大模型加速 25%，支援 fp8 訓練／推論
DoMINO自訂 Warp kernel 做點雲空間投影，比純 PyTorch 快 10 倍以上
GraphCastgradient checkpointing、concatenation trick、fused SiLU
StormCast／CorrDifffused ops、Apex GroupNorm、bf16 AMP、（patch-wise）梯度累積
SFNO（Makani）AMP＋activation checkpointing 塞進 GPU 記憶體、空間模型平行

官方註明這些效能指南仍在持續擴充中，新版本會有更多內容；建議搭配 NGC 容器以取得全部效能改進。
