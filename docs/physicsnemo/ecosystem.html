領域套件與資料策展#

整理自官方 Data Curation、Model Evaluation and Inference、Active Learning 與 Guardrails 章節。

模型訓練之外，PhysicsNeMo 生態系覆蓋完整生命週期：資料策展（Curator）→ 訓練（Core）→ 評估與推論（CFD／Earth2Studio／NIM）→ 迭代改進（主動學習）→ 上線防護（Guardrails）。

PhysicsNeMo Curator：工程資料 ETL#

PhysicsNeMo Curator 是 GPU 加速的資料策展子模組，解決「模擬輸出 → 訓練就緒資料集」這段最耗時的工。核心特性：平行處理（多程序／分散式）、模組化架構、多格式（HDF5／VTK／STL／Zarr）、schema 驗證、易擴展。ETL 管線四元件：

元件職責
DataSource讀輸入資料、寫輸出資料（來源與匯點）
DataTransformation格式轉換（如 HDF5 → Zarr、重新分塊、正規化）
DatasetValidator驗證輸入資料結構與內容（可選）
ParallelProcessor編排檔案級平行處理

元件可自由組合成自訂管線。官方入門教學是 HDF5 → Zarr 轉換 notebook；安裝建議走 PhysicsNeMo 容器＋pip install -e . 原始碼安裝。

官方多數訓練範例附推論腳本；預訓練 checkpoint 發佈於 NGC Catalog 與 Hugging Face。針對兩大領域另有現成下游工作流程：

PhysicsNeMo-CFD：DoMINO NIM 外氣動力#

PhysicsNeMo-CFD 把預訓練 AI 模型接進工程 CFD 工作流程（含驗證與基準測試工具）。例：用 DoMINO-Automotive-Aero NIM 對 DrivAerML 車體算流場：

from physicsnemo.cfd.inference.domino_nim import call_domino_nim

output_dict = call_domino_nim(
    stl_path="./drivaer_202.stl",
    inference_api_url="http://localhost:8000/v1/infer",
    data={
        "stream_velocity": "38.89",
        "stencil_size": "1",
        "point_cloud_size": "500000",
    },
)

Earth2Studio：AI 氣象推論#

Earth2Studio 讓你幾行程式碼跑 AI 天氣預報。例：用 DLWP 從 GFS 初始化跑 10 天預報：

from earth2studio.models.px import DLWP
from earth2studio.data import GFS
from earth2studio.io import NetCDF4Backend
from earth2studio.run import deterministic as run

model = DLWP.load_model(DLWP.load_default_package())
run(["2024-01-01"], 10, model, GFS(), NetCDF4Backend("output.nc"))

主動學習（Active Learning）#

主動學習的前提：用最少的訓練樣本達到目標模型效能——模型自己挑「學了最有幫助」的新資料點，再回頭請昂貴的求解器標註。PhysicsNeMo 的核心迴圈分四階段：

階段內容
Training／Fine-tuninglearner（代理模型）先以既有資料訓練，後續迭代用新資料微調
Querying查詢策略挑選最有資訊量的新資料點（如不確定性最高者）
Labeling取得新資料的 ground truth——跑外部求解器／模擬器或實驗
Metrology（可選）生產環境的可觀測性：把代理模型丟進求解器工作流程量測效能與穩定性

元件以 typing.Protocol 結構性子型別定義（physicsnemo.active_learning.protocols）——不強制繼承，只要執行期提供對應方法即可，方便改造既有程式碼。完整配方見外氣動力主動學習範例。

Guardrails：模型信心防護#

Guardrails（26.03 引入、26.05 強化）提供分析模型輸入與預測的工具，估計模型信心——辨識輸入是否落在訓練分佈外、預測是否可信，是代理模型上生產環境的安全網。範例見 examples/minimal/guardrails；不確定性量化可搭配 uq-extras（GPyTorch）。
