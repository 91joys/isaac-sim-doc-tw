NVIDIA PhysicsNeMo 總覽文檔#

整理自官方文檔 首頁 與 Overview 章節，依據 PhysicsNeMo 26.05（latest，PhysicsNeMo v2.x 架構）。

PhysicsNeMo 是什麼#

NVIDIA PhysicsNeMo 是一個用最先進 SciML 方法建構、訓練、微調與推論 Physics-AI 模型的開源深度學習框架，服務 AI4Science 與工程應用。它提供一組 Python 模組，讓你組出可擴展、高度最佳化的訓練與推論管線——把物理知識與資料結合，實現即時預測。

  • 不是黑盒求解器，而是 PyTorch 的互補工具箱：你依然寫標準的 PyTorch 訓練迴圈，PhysicsNeMo 提供 SciML 專用的模型、資料管線、分散式與物理殘差工具。
  • 方法譜系全覆蓋：無論你用神經算子（Neural Operators）、GNN、Transformer，或物理資訊神經網路（PINN），乃至兩者之間的混合方法，都有最佳化的實作可直接取用。
  • 為 NVIDIA GPU 深度最佳化：從單 GPU 到多節點叢集，內建資料平行與領域平行工具，只需幾行程式碼即可規模化。

名稱由來：PhysicsNeMo 的前身是 NVIDIA Modulus，2025 年更名並在 v2.0 進行大規模模組化重構（見 v2.0 Migration Guide）。舊的獨立套件 PhysicsNeMo-Sym 已上游整併為 physicsnemo.sym 命名空間。

五大核心元件#

元件角色
physicsnemo.models最佳化、可配置、易用的模型架構家族：神經算子、GNN、擴散模型、Transformer 等（見模型架構）
physicsnemo.datapipes針對點雲、網格等工程／科學資料結構調校的內建資料管線，可擴展且開箱即用
physicsnemo.distributed建構於 torch.distributed 之上的分散式子模組，幾步即可平行訓練（見分散式與領域平行）
physicsnemo.curator加速工程資料集策展（curation）流程的子模組（見資料策展）
physicsnemo.sym符號式 PDE 定義、自動空間導數計算與物理殘差評估（PhysicsInformer）；以 pip install "nvidia-physicsnemo[sym]" 安裝（見物理引導與 PINNs）

與 PyTorch 及其他框架的關係#

面向說明
PyTorchPhysicsNeMo 建構於 PyTorch 之上：模型是 physicsnemo.Module（torch.nn.Module 的擴展），訓練迴圈、優化器、autograd 全是原生 PyTorch。既有 PyTorch 模型可用 Module.from_torch() 一行轉換，享受 checkpoint 與 model registry 等功能
Warp核心相依之一。DoMINO 等模型用自訂 Warp kernel 做點雲空間投影，端到端比純 PyTorch 快 10 倍以上——本站另有 Warp 中文文檔
傳統數值求解器非取代關係：訓練資料常來自 CFD／FEM 求解器；訓練出的 AI 代理模型（surrogate）提供快幾個數量級的近即時預測，用於設計迭代與大規模系集
本站 PBDL 理論對照PhysicsNeMo 是「Physics-Based Deep Learning」方法的工程化實作——理論背景可對照 PBDL 中文文檔（物理損失、可微物理、神經算子章節）

為什麼用 PhysicsNeMo#

  • SciML 基準測試與驗證：拿你的模型與經過驗證的架構在標準基準問題上比較，附領域專屬的驗證準則。
  • 通用 SciML 配方 × 異質資料集：從 SOTA 架構清單挑模型，配內建資料管線直接套用到你的用例。
  • 開箱即用的效能與可擴展性：最佳化的 ETL 管線 + 多 GPU／多節點規模化，不用自己造輪子。
