可微分物理（Differentiable Physics）#

本頁導讀原書 Differentiable Physics 一章：讓數值求解器本身可反向傳播，成為深度學習計算圖的一部分。這是 PBDL 耦合光譜中最緊、也最強大的一端，是全書的核心。

核心概念#

可微分物理（DP）的做法是：不把物理求解器丟掉，而是賦予它計算梯度的能力，讓梯度能一路反向傳播穿過整個模擬。用原書的話說，DP「用既有的數值求解器來強化並改進 AI 系統」，而非以神經網路取代求解器。

一個物理求解器可被分解成一連串可微分算子：

u(t+Δt) = P_m ∘ … ∘ P_2 ∘ P_1 ( u(t) )

每個算子 P_i 都能計算「雅可比—向量積（Jacobian-vector product）」。反向傳播時，梯度依鏈鎖法則沿這條算子鏈往回傳——和訓練神經網路完全一樣。

梯度如何穿過求解器#

關鍵洞見是：我們從不建構完整的雅可比矩陣，而是用反向模式微分（reverse-mode AD）高效計算「轉置雅可比 × 向量」的乘積。對一個反問題，總梯度由三部分組合：

  • 損失對最終狀態的梯度：通常很簡單（例如與目標的差）。
  • 物理對控制參數的雅可比：編碼求解器輸出如何隨輸入變化。
  • 時間累積：每個時間步的貢獻加總，得到總梯度。

與 PINN 的本質差異#

PINN（物理損失）可微分物理（DP）
物理如何進入把方程當成殘差損失把真正的求解器保留在迴圈中
求解器去留訓練後多半丟棄訓練中與訓練後都持續運作
解決範圍單一反問題學會解「一整類」反問題
泛化換條件常需重訓對解族泛化，推論高效

原書強調：DP「與物理資訊神經網路有本質差異」——與其用深度學習去解單一反問題，不如用可微分物理訓練出能非常高效地解一大類反問題的網路。

為何要「單體算子」#

DP 不直接把求解器拆成框架最底層的張量運算，而是分解成有物理意義的單體算子（monolithic operators）。好處：

  • 避免儲存大量不必要的中間狀態，省記憶體。
  • 能對特定運算沿用專門的數值方法（多重網格、共軛梯度等）。

代價則是：需要對數值方法與物理問題有更深入的理解。

隱式梯度計算#

對某些複雜運算（如流體壓力投影要解的 Poisson 方程），DP 採用隱函數定理（implicit function theorem）做隱式微分：與其反傳穿過迭代求解器的每一步，不如解析地推導出——梯度只需用相同的方程、換一組邊界條件再解一次即可。這讓反向傳播能重用前向求解器，而不必儲存中間迭代狀態。

典型用法與網路整合#

  • 反問題：由目標狀態反推初始條件、材料參數或最佳控制。
  • 求解器修正器（solver-in-the-loop）：神經網路在每步模擬後修正離散化誤差或未解析物理，因求解器可微分，網路能「看見」自己的修正如何影響後續多步的演化，進而學到穩定、長程的修正策略。
  • 與監督式結合：DP 損失可與資料損失並用，兼顧物理一致性與擬合。

小結：可微分物理把「模擬」變成神經網路可以學習穿越的一層。它是 PBDL 最有威力的工具，代價是需要一個可微分的求解器（如 ΦFlow、JAX 實作）與對數值方法的掌握。控制類問題也可改用強化學習，兩者的取捨見該頁。
