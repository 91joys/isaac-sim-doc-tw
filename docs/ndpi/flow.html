Flow 追蹤與生命週期#

對應 DeepWiki 2.3 Flow Tracking and Management。nDPI 的偵測是「有狀態」的：狀態存在你提供的 flow 結構裡，而 flow 表的管理（建立、查詢、回收）是應用程式的責任。

用 5-tuple 識別連線#

Flow 由「來源 IP、目的 IP、來源 port、目的 port、L4 協定」（可加 VLAN ID）雜湊識別。雜湊是雙向一致的——先排序端點再計算，同一條連線不論封包方向都對到同一筆：

# 雙向一致雜湊的概念
if (src_ip < dst_ip) or (src_ip == dst_ip and src_port < dst_port):
    hash(src_ip, src_port, dst_ip, dst_port, protocol)
else:
    hash(dst_ip, dst_port, src_ip, src_port, protocol)

ndpiReader 用 uthash 雜湊表管理 flow（HASH_ADD / HASH_FIND / HASH_DEL / HASH_ITER），並以多個獨立 root 表（NUM_ROOTS）搭配 per-root mutex 分散多執行緒競爭。

生命週期五階段#

階段狀態發生的事
建立packet_counter = 0第一個封包到達，配置 flow 結構、初始化計數器與時間戳
偵測中detection_completed = 0逐包呼叫解析器、累積證據
已辨識detection_completed = 1寫入協定堆疊、評估風險
持續追蹤協定已知視設定繼續萃取中繼資料（extra dissection / monitoring）
回收閒置逾時預設 30 秒（MAX_IDLE_TIME）無封包即逾時；先 giveup 再釋放

偵測狀態旗標#

旗標用途
detection_completed協定偵測完成
check_extra_packets / max_extra_packets_to_check延伸解析的封包額度
fail_with_unknown禁止回退為 UNKNOWN
protocol_id_already_guessed已做過初步（port/IP）猜測

延伸封包解析（Extra Dissection）#

許多協定需要多個封包才能完成分類或抽完中繼資料。解析器在初步辨識後掛上回呼函式，讓後續封包繼續被分析，而不必跑完整的 dissector 流程：

/* 在 dissector 裡（例如 TLS）設定延伸解析 */
flow->max_extra_packets_to_check = 10;        /* 最多再看 10 個封包 */
flow->extra_packets_func = search_tls_again;  /* 回傳 1 續看、0 結束 */
協定延伸解析目的
TLS等 ClientHello + ServerHello + 憑證鏈；JA3/JA4 指紋；混淆 TLS 偵測
HTTP標頭跨封包；WebSocket 升級偵測
DNSquery 與 response 配對；快取解析出的 IP（FPC）
STUN / RTP進入 monitoring 模式，持續觀察媒體流型態與統計

TLS 也示範了「何時該停」的判斷：憑證與 ClientHello 都處理完、或雙向都已是 Application Data 時可跳過延伸解析；仍需子分類、metadata 或啟發式分析時才繼續。

封包方向追蹤#

應用程式可透過 ndpi_flow_input_info 告訴 nDPI 封包方向（NDPI_IN_PKT_DIR_C_TO_S / S_TO_C / UNKNOWN）。未知時 nDPI 依 TCP SYN（發起者為 client）、port 大小（低 port 通常是 server）、首包方向推斷。解析器內用 current_pkt_from_client_to_server() 等輔助函式判斷。

記憶體管理#

兩層配置策略：

  • 一般配置：ndpi_malloc / ndpi_free（計入 tot_ndpi_memory 總量統計）
  • Flow 配置：ndpi_flow_malloc / ndpi_flow_free（由應用程式管理）

全部可用 ndpi_set_memory_alloction_functions() 換成自訂配置器——用於記憶體追蹤、jemalloc / tcmalloc、記憶體池或除錯。

/* 回收一條 flow 的正確順序 */
ndpi_free_flow_data(flow);   /* 先釋放 flow 內部動態成員（hostname、憑證…）*/
ndpi_flow_free(flow);        /* 再釋放 flow 本體 */

踩雷提醒：自訂配置器要在任何 nDPI API 使用之前安裝。ndpiReader 曾因太晚裝 hook，導致部分記憶體用 libc 配置、卻用自訂函式釋放的未定義行為（PR #3164）。

閒置掃描（以 ndpiReader 為例）#

  1. 每隔 IDLE_SCAN_PERIOD 由 ndpi_idle_scan_walker() 走訪 flow 表
  2. 逾時的 flow 若偵測未完成，先呼叫 ndpi_detection_giveup() 取得最終猜測
  3. process_ndpi_collected_info() 萃取最後的 metadata 並輸出
  4. 釋放 flow 記憶體、從雜湊表移除
