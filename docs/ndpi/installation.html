編譯與快速上手#

對應官方 README：How To Compile nDPI 章節。本頁涵蓋各平台前置套件、編譯選項，以及用 ndpiReader 立刻分析第一個 PCAP。

各平台前置套件#

最低需求：GNU 工具鏈（autoconf、automake、libtool、pkg-config、gettext、flex、bison）＋ gcc 或 Clang。

平台安裝指令
Debian / Ubuntusudo apt-get install build-essential git gettext flex bison libtool autoconf automake pkg-config libpcap-dev libjson-c-dev libnuma-dev libpcre2-dev libmaxminddb-dev librrd-dev
Arch Linuxsudo pacman -S gcc git gettext flex bison libtool autoconf automake pkg-config libpcap json-c numactl pcre2 libmaxminddb rrdtool
FreeBSDsudo pkg install gcc git gettext flex bison libtool autoconf automake devel/pkgconf gmake libpcap json-c pcre2 libmaxminddb rrdtool（編譯請用 gmake）
macOSbrew install coreutils gcc git gettext flex bison libtool autoconf automake pkg-config libpcap json-c pcre2 libmaxminddb rrdtool
Windows三種方式：MSYS2、Mingw-w64、Visual Studio（windows/nDPI.sln）。皆需安裝 npcap 並啟用 WinPcap 相容模式

編譯#

# 基本編譯
./autogen.sh && ./configure
make

# croaring.c 編譯出錯時的替代方案
./autogen.sh && ./configure --enable-old-croaring
make

# 只編函式庫（不含工具與測試）
./autogen.sh && ./configure --with-only-libndpi
make

# out-of-tree 建置（原始碼樹保持乾淨）
./autogen.sh
mkdir build && cd build
../configure && make

注意：autogen.sh 不會自動執行 configure，兩步要分開跑（./autogen.sh && ./configure --$OPTIONS）。

執行測試#

./tests/do.sh        # PCAP 迴歸測試（比對輸出差異）
./tests/do-unit.sh   # 單元測試
./tests/do-dga.sh    # DGA 網域偵測測試

# 或一次全跑
make check

用 ndpiReader 立刻體驗#

ndpiReader 是官方參考實作：既是命令列分析工具，也是整合 nDPI 的最佳範例程式（原始碼在 example/）。它是多執行緒架構，可分析 PCAP 檔或即時網卡流量。

# 分析 PCAP 檔
./example/ndpiReader -i capture.pcap

# 即時監聽網卡
./example/ndpiReader -i eth0

# 詳細輸出（含每條 flow 的協定、信心等級與風險）
./example/ndpiReader -i capture.pcap -v 2

# 輸出 CSV
./example/ndpiReader -i capture.pcap -C out.csv

常用命令列選項#

類別參數用途
輸入來源-iPCAP 檔、播放清單或網路介面
過濾-fBPF 封包過濾條件
多執行緒-n平行處理的執行緒數
輸出格式-C / -k / -KCSV、JSON 序列化與格式選項
分析功能-P / -F / -APayload 分析、流量統計、內部統計
協定控制-p / -B / -u自訂協定檔、停用協定、除錯協定

建置官方文件（選用）#

pip install --upgrade pip
pip install -r doc/requirements.txt
make doc

# 用內建 Python 網頁伺服器瀏覽
make doc-view

下一步#

  1. 拿自己的 PCAP 跑 ndpiReader -v 2，觀察協定堆疊、信心等級與風險輸出。
  2. 讀《核心架構》建立三大結構的心智模型。
  3. 照《整合到你的應用程式》的骨架程式碼把 nDPI 嵌進自己的程式。
