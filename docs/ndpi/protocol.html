協定堆疊、分類與信心等級#

對應 DeepWiki 1. nDPI Overview 與 2.2 Protocol Definition and Content Matching。本頁說明 nDPI 如何「表達」辨識結果：兩層協定堆疊、應用分類、breed 與信心等級。

兩層協定堆疊#

辨識結果存於 detected_protocol_stack[2]：[0] 是 master 協定（傳輸／隧道層），[1] 是 app 協定（實際應用）。習慣寫作「Master.App」：

TLSNetflix
QUICYouTube
DNSGoogle
HTTP（Unknown）
STUNWhatsApp Call
  • TLS.Netflix：TLS 加密的 Netflix 流量；DNS.Facebook：查詢 Facebook 網域的 DNS
  • 單層協定（純 DNS、SSH、BitTorrent）只有 master，沒有 app 層
  • TLS.Unknown：加密流量但辨識不出上層應用

協定 ID 定義在 src/include/ndpi_protocol_ids.h，從 NDPI_PROTOCOL_UNKNOWN（0）起編號。層級關係用 ndpi_set_proto_subprotocols() 宣告（例如 WhatsApp 可跑在 TLS 或 QUIC 之上）。

信心等級（Confidence）#

辨識結果附帶「是怎麼判斷出來的」，數值越高越可靠。應用程式應把 port 比對視為初步結果：

等級值判斷方式
NDPI_CONFIDENCE_UNKNOWN0初始狀態，尚未判斷
NDPI_CONFIDENCE_MATCH_BY_PORT1只憑預設 port 猜測（最弱）
NDPI_CONFIDENCE_MATCH_BY_IP2命中已知服務的 IP 範圍
NDPI_CONFIDENCE_DPI_PARTIAL3部分協定證據
NDPI_CONFIDENCE_DPI_PARTIAL_CACHE4快取的部分比對
NDPI_CONFIDENCE_DPI5完整深度封包檢測（最可靠）
NDPI_CONFIDENCE_DPI_CACHE6快取的 DPI 結果
NDPI_CONFIDENCE_NBPF7nBPF 過濾器比對

分類例子
NDPI_PROTOCOL_CATEGORY_WEBHTTP、TLS、瀏覽器流量
NDPI_PROTOCOL_CATEGORY_STREAMINGYouTube、Netflix、Spotify
NDPI_PROTOCOL_CATEGORY_SOCIAL_NETWORKFacebook、Twitter

分類在 ndpi_set_proto_defaults() 註冊協定時指定，也可用分類檔自訂。

Breed（安全性評級）#

Breed意義
NDPI_PROTOCOL_SAFE可信任服務
NDPI_PROTOCOL_ACCEPTABLE一般可接受
NDPI_PROTOCOL_FUN娛樂類
NDPI_PROTOCOL_UNSAFE不安全（如明文舊協定）
NDPI_PROTOCOL_POTENTIALLY_DANGEROUS高風險（如 Tor、惡意寄存服務）

v5.0+ 的分類狀態 API#

新版為分類流程提供明確狀態，主迴圈大幅簡化——持續呼叫 nDPI 直到 NDPI_STATE_CLASSIFIED：

res = ndpi_detection_process_packet(...);
if (res.state == NDPI_STATE_CLASSIFIED) {
    /* 完成：可取得最終分類與所有 metadata，此 flow 不再需要餵封包 */
} else {
    /* nDPI 還需要更多封包。
       若 state 是 NDPI_STATE_PARTIAL，res.proto 是可用但可能更新的初步分類 */
}

/* 範例（QUIC flow）:
   pkt 1: proto QUIC          state NDPI_STATE_PARTIAL
   pkt 2: proto QUIC/YouTube  state NDPI_STATE_CLASSIFIED

   範例（標準 TLS flow）:
   pkt 1-3: proto Unknown       state NDPI_STATE_INSPECTING
   pkt 4:   proto TLS/Facebook  state NDPI_STATE_PARTIAL
   pkt N:   proto TLS/Facebook  state NDPI_STATE_CLASSIFIED */

遷移注意：ndpi_extra_dissection_possible() 已移除；ndpi_detection_giveup() 移除了第三個參數（要知道是否為猜測，改讀 flow->protocol_was_guessed）；部分原型從 ndpi_protocol 改為 ndpi_master_app_protocol。
