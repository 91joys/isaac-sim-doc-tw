資料結構與演算法#

對應 DeepWiki 4. Data Structures and Algorithms（含 4.1 Pattern Matching Engines、4.2 Caching）。nDPI 的高效能來自為不同比對場景挑選正確的資料結構。

總覽：誰負責什麼#

結構用途搜尋複雜度記憶體
Patricia 樹（ndpi_patricia_tree_t）IP 前綴（CIDR）比對O(k)，k = key 長度精簡（路徑壓縮）
Aho-Corasick 自動機（AC_AUTOMATA_t）多樣式字串比對（主機名稱／網域／憑證）O(n + z)，n = 文字長、z = 命中數較高
雜湊表（uthash / ndpi_str_hash）精確 key 查詢（惡意 JA4、SHA1 憑證、TCP 指紋）平均 O(1)中等
LRU 快取（ndpi_lru_cache ×7）近期流量結果重用O(1)有界
二元搜尋樹（tcpRoot / udpRoot）預設 port 查詢O(log n)小
位圖（Roaring / Binary / Fuse）協定啟用位圖、風險位圖、大型 IP 集合O(1) 位元測試高壓縮

Patricia 樹：IP 比對#

六棵樹分工：

樹用途來源
NDPI_PTREE_PROTOCOLSIPv4 協定範圍（AWS、Cloudflare…）內建於 ndpi_content_match.c.inc
NDPI_PTREE_PROTOCOLS6IPv6 協定範圍內建
NDPI_PTREE_RISK / RISK6高風險網段設定檔載入
NDPI_PTREE_RISK_MASK / RISK_MASK6風險遮罩（哪些風險適用於哪些網段）設定檔載入

查詢走 ndpi_network_ptree_match()；實作採路徑壓縮減少節點並提升快取友善度。

Aho-Corasick：字串比對#

自動機樣式用途
host_automa主機名稱字串比對確切主機（www.google.com）
domain_automa網域樣式後綴比對（*.netflix.com）
tls_cert_subject_automa憑證樣式比對 TLS 憑證主體
risky_domain_automa風險樣式辨識高風險網域

樣式在初始化時以 ndpi_add_host_url_subprotocol() 加入、finalize 時 ac_automata_finalize() 編譯；查詢走 ndpi_match_string_subprotocol()。實作最佳化：連續字元的範圍壓縮、character bitmap 加速狀態轉移、單邊優化（one_alpha）。

三種位圖實作#

  • Roaring Bitmap（CRoaring）：稀疏大集合、高壓縮——大型 IP 集合、協定分類的壓縮儲存。
  • Binary Bitmap（排序陣列）：小而密的集合。
  • Fuse Filter（機率型）：只回答「在不在集合裡」的成員測試，壓縮比最高。

Flow 的風險位圖則是單純的 u_int64_t，用 NDPI_SET_BIT / NDPI_ISSET_BIT / NDPI_CLR_BIT 操作。

統計分析工具#

結構能力
ndpi_analyze_struct資料序列的均值、變異數／標準差（Welford 線上演算法）、極值、jitter
ndpi_entropy雙向位元組分布 → Shannon 熵（可判斷加密／壓縮流量）
ndpi_bin直方圖式資料分箱（如封包長度分布）
ndpi_domain_classify網域名稱分類器

這些是行為分析與異常偵測（DGA、混淆流量、異常模型）的基礎。
