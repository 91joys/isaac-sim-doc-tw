nDPI 總覽文檔#

對應 DeepWiki：nDPI Overview 與官方 README 的中文整理。建議先讀本頁建立整體心智模型，再依側欄章節深入。

什麼是 nDPI#

nDPI®（Network Deep Packet Inspection）是 ntop 團隊維護的開源 LGPLv3 C 函式庫，用於深度封包檢測。不同於單純看 port 號的傳統分類方式，nDPI 透過有狀態的流量分析（stateful flow analysis）——比對特徵樣式、觀察行為、針對特定協定深入解析——來判斷一條連線實際承載的協定與應用程式。

nDPI 的定位是「函式庫」而非獨立工具：由你的應用程式把封包餵給它、取回分類結果。ntopng、nProbe cento 等產品都是這樣整合 nDPI 來做流量辨識與選擇性封鎖。

五大核心能力#

能力說明
協定辨識跨 OSI 各層辨識 300+ 種協定：從 DNS、TLS、QUIC 等基礎協定，到 Netflix、WhatsApp 等應用層服務
風險評估辨識 57 種安全風險：惡意流量、可疑行為、政策違規（自簽憑證、DGA 網域、明文密碼…）
中繼資料萃取抽出主機名稱（SNI / Host）、TLS 憑證、User-Agent 等協定專屬屬性
流量指紋產生 TLS（JA3 / JA4）、TCP、HTTP 指紋，用於辨識用戶端與威脅獵捕
分類標籤把協定對應到 17 種應用分類（影音、社群、VoIP、遊戲…）與 breed 安全評級

典型整合工作流程#

① 初始化ndpi_init_detection_module() —— 建立偵測模組，註冊 300+ 協定與 80+ 解析器
↓
② 設定ndpi_set_config() —— 載入自訂規則、啟用／停用協定
↓
③ 完成初始化ndpi_finalize_initialization() —— 建立 Patricia 樹、Aho-Corasick 自動機、LRU 快取
↓
④ 逐包處理ndpi_detection_process_packet() —— 每個封包都帶著 flow 狀態呼叫
↓
⑤ 取得結果ndpi_protocol（master + app + confidence）＋ metadata ＋ 風險位圖
↓
⑥ 清理ndpi_exit_detection_module() —— 程式結束時釋放資源

學習提示：理解 nDPI 的關鍵，是抓住三個核心資料結構的分工——全域偵測模組（ndpi_detection_module_struct）、每條連線的狀態（ndpi_flow_struct）、當前封包（ndpi_packet_struct）。詳見《核心架構》。

nDPI 管什麼、不管什麼#

nDPI 負責應用程式負責
解析封包、辨識協定、標記風險擷取封包（libpcap、PF_RING、DPDK…）
維護偵測狀態（寫在你給的 flow 結構裡）建立、查表、逾時回收 flow（5-tuple 雜湊表）
萃取 metadata 與指紋決定拿分類結果做什麼（統計、告警、封鎖）

執行緒安全：偵測模組在 finalize 之後為唯讀，可跨執行緒共享；flow 結構不可在無同步機制下跨執行緒共用。
