核心架構：三個關鍵資料結構#

對應 DeepWiki 2. Core Architecture 與 2.1 Core Detection Engine。整個函式庫圍繞三個結構運作：偵測模組是全域共享的唯讀情境、flow 結構記錄每條連線跨封包的狀態、packet 結構存放當前封包解析後的資訊。

ndpi_detection_module_struct（全域偵測情境）#

每個應用程式實例初始化一次，供所有 flow 共用，內含所有偵測資源：

成員型別用途
proto_defaults[]ndpi_proto_defaults_t協定中繼資料：名稱、分類、預設 port、breed
callback_buffer[]call_function_struct已註冊的協定解析器（dissector）函式表
tcpRoot / udpRoot二元搜尋樹Port → 協定的 O(log n) 查詢
protocols_ptree[]Patricia 樹 ×6IP 範圍比對（IPv4/IPv6 × 協定／風險／風險遮罩）
host_automa、domain_automa…Aho-Corasick 自動機 ×5主機名稱、網域、TLS 憑證的多樣式字串比對
lru_cache[]LRU 快取 ×7Ookla、BitTorrent、STUN、TLS 憑證等專屬快取
malicious_ja4_hashmap 等雜湊表威脅情資查詢（惡意 JA4、SHA1 憑證）
cfg設定旗標執行期行為調校

ndpi_flow_struct（每條連線的狀態）#

分類關鍵欄位說明
偵測結果detected_protocol_stack[2]兩層協定堆疊：[0] master（如 TLS）、[1] app（如 Facebook）
confidence、category信心等級與應用分類
偵測狀態detection_completed偵測完成旗標
excluded_dissectors_bitmask已排除（比對失敗）的解析器位圖，避免重複嘗試
extra_packets_func偵測完成後的延伸解析回呼
風險risk（64-bit）、risk_score、risk_infos[]風險位圖、加權分數、最多 8 則說明訊息
L4 狀態l4.tcp / l4.udp（union）TCP 序號、旗標；UDP 專屬狀態
協定狀態protos.http / tls_quic / dns / ssh…（union）40+ 種協定的解析中間狀態，用 union 省記憶體
中繼資料host_server_name、tls_quic.ja3_client…萃取出的主機名稱、指紋等
統計packet_counter、方向計數、時間戳封包數、位元組數、首末封包時間

重要邊界：nDPI 不管理 flow 的生命週期。應用程式要自己建立、查表（通常用 5-tuple 雜湊）、逾時回收 flow，並在每次呼叫時把對應的 ndpi_flow_struct 傳給 nDPI。詳見《Flow 追蹤與生命週期》。

ndpi_packet_struct（當前封包）#

  • iph / iphv6：解析後的 IP 標頭指標
  • tcp / udp：傳輸層標頭指標
  • payload、payload_packet_len：應用層資料與長度
  • current_time_ms：時間戳，用於逾時與速率計算

兩階段初始化#

階段一：模組初始化（ndpi_init_detection_module）#

  1. 配置 ndpi_detection_module_struct 並套用預設值
  2. dissectors_init() 透過 ndpi_callback_init() 註冊 80+ 個協定解析器到 callback_buffer[]
  3. 為每個協定填好 proto_defaults（名稱、分類、預設 port、breed）
  4. 設定記憶體配置器（可用 ndpi_set_memory_alloction_functions() 自訂）

階段二：完成初始化（ndpi_finalize_initialization）#

  1. ac_automata_finalize() 編譯主機名稱／網域／憑證的 Aho-Corasick 自動機
  2. load_default_ports() 把啟用協定的預設 port 灌進 tcpRoot / udpRoot
  3. ndpi_enabled_callbacks_init() 依 detection_bitmask 啟用解析器
  4. 初始化各 LRU 快取

兩階段之間可插入自訂設定：ndpi_load_protocols_file()（自訂協定）、ndpi_load_categories_file()（自訂分類）、ndpi_set_config()（執行期參數）。

清理（ndpi_exit_detection_module）#

程式結束時釋放自動機（ac_automata_release）、Patricia 樹、LRU 快取與協定表。

關鍵函式速查#

函式用途
ndpi_init_detection_module()配置並初始化偵測模組
dissectors_init()註冊所有協定解析器
ndpi_set_proto_defaults()定義協定中繼資料（名稱、port、分類）
ndpi_set_config()設定偵測參數
ndpi_finalize_initialization()建立搜尋結構並驗證設定
ndpi_detection_process_packet()偵測主進入點（見《偵測管線》）
ndpi_detection_giveup()放棄偵測，做最後啟發式猜測
