範例精讀：模擬資料融入情境#

對應官方 Simulation Data in Context 範例的完整逐步教學。這是範例目錄・第 2 例的深入版，把每一步的選單路徑、快捷鍵與屬性值都補齊。

1. 這個範例在做什麼#

本範例示範如何把穩態熱模擬資料疊到一份高保真的伺服器機架幾何上：透過 OpenUSD 組合，把 CGNS 熱資料與 USD 機架模型放進同一場景，兩邊都不轉檔。

這正是 Kit-CAE 核心理念的最佳示範——OpenUSD 是組合層，原生檔案始終是權威來源。你會得到一個既能看清機架真實外觀、又能在其熱通道（hot-aisle）裡檢視空氣溫度分佈的場景，做「有情境（in context）」的工程判讀。

2. 資料集規格#

檔案格式欄位說明
compute_thermal.cgnsCGNSTemperature機架附近的穩態空氣溫度（cell 中心值，單位 °C）。
GB300.usdUSD幾何完整高保真的 GB300 NVL72 伺服器機架模型。

網格細節：約 71,500 個節點、約 57,100 個六面體（hexahedral）元素；穩態（單一快照，無時間序列）。

前置需求：需先下載範例資料包（見快速入門・下載範例資料）。本範例檔案位於 kit_cae_user_guide_data/examples/02_simulation-data-in-context/ 目錄下。

3. 啟動 Kit-CAE#

Linux：

./repo.sh launch -n omni.cae.kit

Windows：

repo.bat launch -n omni.cae.kit

4. 匯入機架幾何#

  1. 選 File > Open。
  2. 瀏覽到 {資料路徑}/kit_cae_user_guide_data/examples/02_simulation-data-in-context/GB300.usd。
  3. GB300 機架模型即載入視埠（viewport）。
為什麼是 Open 而非 Import？機架幾何本身就是一份 USD，直接以它作為舞台的基底（base layer）最自然；稍後的熱資料才用 Import「掛」進這個舞台，兩者於是組合在同一場景。

5. 匯入熱資料#

  1. 選 File > Import。
  2. 瀏覽到 {資料路徑}/kit_cae_user_guide_data/examples/02_simulation-data-in-context/compute_thermal.cgns。
  3. 勾選 Import to Stage。
  4. 點 Import。
  5. Stage 面板出現新的 compute_thermal prim。

6. 建立兩個邊界框#

我們要兩個邊界框：一個涵蓋整份熱資料範圍（視覺參考），另一個縮小聚焦到機架後方的熱通道區，當作體積運算子的感興趣區域（ROI）。

① 第一個邊界框（完整範圍）

  1. 在 Stage 面板展開 /World/compute_thermal/compute_thermal_cgns/Base/Zone/Elements。
  2. 右鍵 Elements。
  3. 選 Create > CAE Sources > Bounding Box。

② 第二個邊界框（縮小的 ROI）

  1. 選取第一個邊界框。
  2. 按 Ctrl+D 複製。
  3. 按 R 叫出縮放（scale）gizmo。
  4. 縮放，聚焦到 GB300 後方的熱通道區。
  5. 按 W 回到平移（translate）模式。
操作快捷鍵一覽
鍵作用
Ctrl+D複製選取的 prim
W平移（translate）gizmo
R縮放（scale）gizmo
F視角貼合到選取物件

7. 建立體積（Volume）運算子#

  1. 右鍵 Elements。
  2. 選 Create > CAE Operators > Volume。
  3. 出現提示時，選 NanoVDB（另一選項 Irregular 保留精確幾何）。

NanoVDB vs. Irregular 怎麼選

模式做法與適用時機
Irregular直接在非結構化網格上運算，不做體素化，保留精確 cell 幾何。當精度比速度重要時使用——例如驗證，或邊界附近的細部分析。
NanoVDB先把資料重新取樣到稀疏體素格（sparse voxel grid），再於 GPU 渲染。適合大型資料與互動探索，速度較快。

8. 視覺化熱場#

  1. 在 Stage 面板選取 Volume 運算子 prim。
  2. 在 Property 面板捲到 Region-of-interest (ROI)。
  3. 點 Add Target。
  4. 選取較窄的那個邊界框。
  5. 捲到 Colors [Field Selection]。
  6. 點 Add Target。
  7. 瀏覽到 /World/compute_thermal/compute_thermal_cgns/Base/Zone/SolutionCellCenter/Temperature。
  8. 點 Select。
  9. 展開 Volume 運算子 > Material > Colormap。
  10. 調整色彩範圍（範例：domain 31 到 37）與傳遞函數，凸顯溫度梯度。
找不到欄位在哪？本資料的溫度是 cell 中心值，所以放在 SolutionCellCenter 之下（而非以節點為中心的 FlowSolution）。這也呼應探索資料裡先用 CAE Insights 摸清欄位位置與範圍的好習慣——31~37 這個 domain 正是從溫度範圍推來的。

9. 關鍵訣竅：看清「純資料」與「有情境」#

切換 GB300 prim 上的眼睛圖示來隱藏機架，即可單看熱場本身；再開回來，就能把結果放回幾何情境中對照。這一開一關，正是本範例的精神——同一份資料，既能獨立分析，也能融入真實硬體情境。

10. 你學到了什麼#

  • 不轉檔的組合：用 File > Open 開 USD 幾何當基底、用 File > Import 掛入 CGNS 模擬資料，OpenUSD 把兩者組合進單一場景而不改動任何一方。
  • 雙邊界框策略：一個作視覺參考、一個縮小成 ROI 限制體積運算的域——複製（Ctrl+D）＋縮放（R）是最快做法。
  • NanoVDB／Irregular 取捨：互動探索用 NanoVDB，精度驗證用 Irregular。
  • cell 中心欄位：知道去 SolutionCellCenter 找 Temperature，並用合理 domain（31~37）調 colormap。
  • 情境切換：用眼睛圖示在「純熱場」與「疊在機架上」之間來回對照。
延伸：想把時間軸也加進來，看時變結構分析；想在單一場景裡同時放 CFD 與結構兩個領域，看多領域模擬。
