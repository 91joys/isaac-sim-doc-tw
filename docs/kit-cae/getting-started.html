快速入門#

對應官方 Get Started 章節。從零把 Kit-CAE 從 GitHub 建置起來、啟動應用、下載範例資料，並完成基本偏好設定。

1. 系統需求#

硬體

  • 一張 NVIDIA RTX GPU（渲染與 GPU 加速運算必備）。

軟體先決條件

平台需求
WindowsGit（git-scm.com）、Git LFS（git-lfs.com）、Visual Studio 2019／2022／2026 並勾選「使用 C++ 的桌面開發（Desktop development with C++）」工作負載。
Linux標準開發環境，含 GCC C++ 編譯器。

安裝完 Git LFS 後，執行一次初始化：

git lfs install

2. 取得並建置應用#

① 複製儲存庫

git clone https://github.com/NVIDIA-Omniverse/kit-cae.git
cd kit-cae

② 建置應用

Windows：

repo.bat build -r

Linux：

./repo.sh build -r

建置會下載相依套件並編譯應用，首次執行需數分鐘。

3. 下載範例資料#

從官方提供的連結下載範例資料集，解壓縮到方便存取的位置。主線指南使用一份穩態空氣力學模擬檔：

檔案內容
auto_aero_solver_result.cgns汽車空氣力學穩態模擬，含壓力與速度分量，約 790 萬個節點、720 萬個 cell。
提示：此資料下載也涵蓋四個獨立範例所需的資料集，一次下載即可。

4. 啟動應用#

Windows：

repo.bat launch -n omni.cae.kit

Linux：

./repo.sh launch -n omni.cae.kit

首次啟動時會因為 GPU shader 編譯而短暫停頓，屬正常現象。

5. 偏好設定（Preferences）#

從 Edit > Preferences > CAE 頁面調整設定：

設定說明
Caching（快取）控制資料與計算快取，影響效能。
Default Operator Enabled建立運算子後立即執行（否則需手動開啟 Enabled）。
Compute Device（運算裝置）選擇 CPU、CUDA 或其他可用裝置。
Bounding Box（邊界框）選擇以 cell 為基礎（預設）或以 point 為基礎的計算方式。
Legacy Settings（舊版設定）需要時可重新啟用較早的選單選項。
座標軸：Kit-CAE 2.0 預設使用 Z-up，對齊多數 CAE 求解器的慣例。若你的資料使用不同座標軸，需自行旋轉。

6. 下一步#

應用啟動後，進入主線工作流程的第一步——把範例 CGNS 檔匯入舞台。

  1. 匯入資料：把 auto_aero_solver_result.cgns 匯入 /World。
  2. 探索資料：用 CAE Insights 看懂欄位範圍與直方圖。
  3. 建立運算子：做出第一個視覺化。
