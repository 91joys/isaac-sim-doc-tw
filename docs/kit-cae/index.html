Kit-CAE 的核心理念

Kit-CAE 最重要的一句話：OpenUSD 是科學資料集的「組合與存取層」，原生檔案始終是唯一的權威資料來源。它不把工程資料轉成別的格式，而是把「真相之源」直接帶進可組合的 OpenUSD 舞台，資料透過可插拔的 delegate 依需求載入，再由 GPU 加速的運算子處理與渲染。這讓求解器輸出、感測器輸入與 AI 代理模型可以在同一個場景中並存、共同分析。

面向Kit-CAE 的做法
資料來源CGNS、VTK、EnSight Gold、OpenFOAM、HDF5、NumPy 等原生格式，不轉檔
組合層OpenUSD 舞台——把幾何、多領域模擬結果、高保真 CAD 組合進同一場景
載入方式可插拔 Data Delegate 依需求（on-demand）載入，避免整份灌記憶體
運算與渲染CUDA-X 的 GPU 運算子 ＋ RTX／NVIDIA IndeX 體積渲染 ＋ NVIDIA Flow 粒子動畫
可延伸性CAE 專用 Schema、Data Delegate API、視覺化運算子皆開源，可直接用或改造進自有流程

建議學習路徑

  1. 總覽文檔：先理解 Kit-CAE 的定位（參考應用＋開發者工具箱）與「不轉檔的組合層」核心理念。
  2. 快速入門：從 GitHub clone → repo build → launch，跑起 omni.cae.kit 並下載範例資料。
  3. 匯入資料：把 auto_aero_solver_result.cgns 匯入舞台，理解時間序列的 Scale／Offset。
  4. 探索資料：用 CAE Insights 看懂欄位範圍與直方圖，再建立來源（邊界框、單位球）。
  5. 建立運算子：依序做出 Faces、Streamlines、Volume、Slice，並學會色彩對應與鎖定。
  6. 流場視覺化：用 NVIDIA Flow 做粒子動畫，最後挑戰 四個獨立範例。
