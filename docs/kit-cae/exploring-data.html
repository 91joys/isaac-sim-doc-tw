探索資料#

對應官方 Explore Your Data 章節。在做視覺化之前，先用 Stage 面板與 Properties 面板搞懂你手上這份資料。

先建立心理預期：Kit-CAE 提供的是通用的視覺化與分析工具，但它不會替你解讀資料的物理意義或領域含義——欄位代表什麼、單位是什麼，仍需你自己掌握。

1. Stage 面板：資料階層#

匯入的資料以樹狀結構顯示在 Stage 面板，結構依格式而異：

CGNS 多面體資料集

  • 多個 NGON_n 區塊（表面網格）
  • 一個 NFACE_n 區塊（3D 體積網格）
  • 一個 coordinates 區塊（共享的點座標）

NumPy 資料集

  • NumPyDataSet 節點，底下含 NumPyArrays（每個陣列一個）

元素型別（element type）會顯示在 Properties 面板。

2. Properties 面板#

選取任何套用了 CAE API 的 prim，即可檢視其屬性。CAE 群組會顯示：

  • 資料集 prim：資料模型屬性、格式專屬屬性。
  • 欄位陣列 prim：欄位專屬屬性與格式細節。

3. CAE Insights：欄位中繼資料檢視器#

在欄位陣列 prim 上會出現 CAE Insights 區段，顯示目前 time sample 的資訊：

項目意義
Device資料所在位置（CPU 或 GPU）。
Array type資料精度，如 float32、int64。
Shape陣列維度與分量數。
Ranges各分量的最小／最大值（向量欄位分開顯示）。
QuartilesQ1、中位數、Q3 分佈統計（僅純量欄位）。
Histogram32 格分佈直方圖（僅純量欄位），可調整 min／max 聚焦分析。
重要：CAE Insights 資料不會自動更新。切換選取或改變時間後，需點 Refresh 更新中繼資料。四分位與直方圖在 GPU 上計算，處理大型資料集時效能更好。

4. 試試看：檢視車輛資料集#

  1. 展開 Stage 樹，找到元素區段（element sections）與 FlowSolution 群組。
  2. 檢視欄位變數：Pressure（壓力）與 Velocity（速度）分量。
  3. 選取某個欄位，查看 CAE 屬性。
  4. 點 Refresh，觀察陣列 shape、數值範圍與直方圖分佈。
  5. 把直方圖範圍縮放到特定區間（例如壓力 -300 到 300），做細部分佈分析。

掌握了欄位範圍後，這些數字會直接用在色彩對應的 domain 設定上。接著可以開始建立來源與運算子。
