範例目錄#

對應官方 Examples 章節。四個獨立範例，各自展示 Kit-CAE 在不同工程場景的用法。

需要範例資料：這些範例需要 Kit-CAE 儲存庫未包含的樣本資料集，須另外下載範例資料包（同一份下載也涵蓋主線指南所需資料）。

1. 時變結構分析#

資料集：保險桿樑（bumper beam）撞擊模擬，51 個時間步、約 13,500 個節點，含變形殼網格幾何與兩個欄位變數：PlasticStrain（跨層最大塑性應變）與 ContactPressure（節點接觸壓力），EnSight Gold 二進位格式。

工作流程

  1. 匯入：File > Import 匯入 bumper_beam.case，Scale 設 2，讓時間步在時間軸上相隔兩格，為插補影格預留空間。
  2. 視覺化幾何：右鍵資料集 → Create > CAE Operators > Faces，渲染變形中的網格。
  3. 依欄位著色：在 CAE 區段把 PlasticStrain 加為 Colors [Field Selection] 目標。
  4. 精修：用 CAE Insights 分析分佈，把色彩 shader 的 domain 設為 0 ~ 0.065 並鎖定，維持跨時間步的一致色階。
  5. 時間插補：對 prim 加入 Operator Temporal 並勾選 Enable Field Interpolation，讓時間步之間的動畫更平滑。

關鍵技巧：多層應變追蹤（上下表面＋最大值）、變形網格動畫與欄位資料同步、以直方圖最佳化範圍、時間插補提升播放品質。

2. 模擬資料融入情境#

資料集：

  • compute_thermal.cgns（CGNS）：伺服器機架附近空氣溫度的穩態熱模擬，cell 中心值，單位攝氏。
  • GB300.usd（USD）：完整高保真的 GB300 NVL72 伺服器機架幾何，約 71,500 節點、57,100 個六面體元素。

工作流程

  1. 載入基礎幾何：File > Open 開啟 USD 機架模型。
  2. 匯入模擬資料：File > Import 匯入 CGNS 熱資料，勾 Import to Stage。
  3. 建立空間參考：建兩個邊界框——一個涵蓋整份熱資料，另一個縮放聚焦到機架後方的熱通道（hot-aisle）區。
  4. 設定體積渲染：建 Volume 運算子，選 NanoVDB 體素化。
  5. 定義 ROI：把運算子的 ROI 目標接到較窄的那個邊界框。
  6. 對應欄位：把 Temperature 欄位接到 colormap。
  7. 精修：調色彩範圍與傳遞函數，強調溫度梯度。

關鍵技巧——不轉檔的組合：「Kit-CAE 透過 OpenUSD 把兩者組合進單一場景，兩邊都不轉檔」，既保留資料保真度，又能把模擬結果直接疊在真實伺服器基礎設施的高保真幾何上做情境分析。

完整逐步教學：這個範例已整理成獨立的精讀頁，補齊每一步的選單路徑、快捷鍵（Ctrl+D／R／W／F）、欄位路徑與 colormap domain 值 → 範例精讀：模擬資料融入情境 →

3. 多領域模擬#

資料集：BSCW（Benchmark Supercritical Wing）機翼資料，兩個檔案：

  • bscw_cfd.cgns（CGNS）：機翼周圍 4,991 個體積節點的壓力係數。
  • bscw_struct.case（EnSight Gold）：機翼表面的彎曲應力。

兩者皆含 101 個時間步，來自非穩態 URANS CFD（M = 0.74、Re = 4.49 × 10⁶）。

工作流程

  1. 匯入兩份資料集，不轉檔，進入同一個 USD 舞台。
  2. 用 Faces 對結構資料建立機翼表面。
  3. 用 nanovdb 對 CFD 壓力場產生體積視覺化。
  4. 對機翼表面套用色彩對應（彎曲應力），結合兩個領域。
  5. 用時間軸同步播放兩份資料集。

關鍵技巧：多格式支援（直接讀各求解器原生格式）、高斯潑濺（Gaussian splatting，範例 radius factor 8.0）、欄位對應（Cp、BendingStress）、時間軸同步、色階鎖定維持跨影格一致。

4. AI 代理模型模擬資料#

資料集：汽車空氣力學推論結果，NumPy .npz 格式，128,000 個點，多欄位：速度（3 分量）、壓力（純量）、紊流動能 TKE、紊流黏度、有符號距離場（128×64×64 體積網格），另有 STL 車輛表面網格作參考幾何。來源為「Digital Twins for Fluid Simulation」藍圖的代理模型。

工作流程

  1. 匯入幾何：以 Z-up 載入 STL 車輛網格，提供空間情境。
  2. 匯入點雲：把 .npz 以 Point Cloud 型態匯入。
  3. 建立來源：建資料集邊界框，再加多個單位球種子。
  4. 放置種子：複製並重定位球，放到不同流場區域。
  5. 建立多種子運算子：加 Streamlines 運算子，參考多個種子來源。
  6. 設定串流線：設高斯潑濺 radius factor、前向方向與速度欄位。
  7. 套用色彩對應：指定欄位（速度或壓力）著色。

關鍵技巧——代理平權：「代理模型的推論資料，在匯入、運算子與視覺化演算法上，都與求解器輸出一視同仁」；再加上多種子串流線一次比較多處流動，以及互動式重定位種子、串流線即時更新。
