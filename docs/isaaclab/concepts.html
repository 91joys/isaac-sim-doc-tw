核心概念與架構#

整理自官方 Core Concepts 章節，涵蓋分層架構、兩種工作流程、致動器、感測器與運動生成器。

分層架構#

Isaac Lab 以層層堆疊的抽象組織一次模擬，由下而上為：

層級對應類別職責
AppAppLauncher啟動與初始化底層 Isaac Sim／Omniverse Kit；解析 --headless、--device、--enable_cameras 等命令列參數
SimulationSimulationContext管理物理與渲染引擎、時間步進（dt）、重力與模擬設定
SceneInteractiveScene容納機器人、剛體、感測器、地形，並自動把單一環境複製成多份平行實例
EnvironmentManagerBasedRLEnv / DirectRLEnvRL 相容介面：reset()／step()、觀測、獎勵、終止
ManagersObservationManager 等模組化處理觀測、獎勵、動作、終止與事件邏輯

關鍵設計：你只需描述「單一環境」的樣貌，InteractiveScene 會自動複製成 num_envs 份，並以向量化張量（形狀為 [num_envs, ...]）統一存取所有實例的狀態——這是 Isaac Lab 大規模平行模擬的基礎。

App 與 AppLauncher#

任何 Isaac Lab 腳本都必須先啟動模擬 App，才能匯入依賴模擬的模組。這與 Isaac Sim 的 SimulationApp 鐵則一致：

import argparse
from isaaclab.app import AppLauncher

# 解析參數並啟動 App
parser = argparse.ArgumentParser()
AppLauncher.add_app_launcher_args(parser)   # 加入 --headless、--device 等
args = parser.parse_args()
app_launcher = AppLauncher(args)
simulation_app = app_launcher.app

# ── 之後才能匯入依賴 Omniverse 的模組 ──
import isaaclab.sim as sim_utils
# ... 你的模擬程式 ...

simulation_app.close()

兩種工作流程#

原文連結

Manager-Based#

把環境拆成多個設定類別，由對應 Manager 在每步驟自動執行。各 Manager 的職責：

Manager職責
ObservationManager依設定的觀測項（term）組裝觀測向量，支援分組與雜訊
ActionManager把策略輸出映射到致動器指令（如關節位置、速度、力矩）
RewardManager加總各獎勵項並套用權重
TerminationManager判定回合結束：超時、跌倒、達標等
EventManager在重置或特定時機套用隨機化與事件（域隨機化）
CurriculumManager依訓練進度調整難度（課程學習）
CommandManager產生目標指令（如目標速度、目標位姿）

每一項邏輯以「term」為單位定義，例如一個獎勵項：

from isaaclab.managers import RewardTermCfg as RewTerm
from isaaclab.utils import configclass

@configclass
class RewardsCfg:
    # 追蹤線速度指令的獎勵
    track_lin_vel = RewTerm(
        func=mdp.track_lin_vel_xy_exp,
        weight=1.0,
        params={"command_name": "base_velocity", "std": 0.5},
    )
    # 動作變化的懲罰
    action_rate = RewTerm(func=mdp.action_rate_l2, weight=-0.01)

Direct#

在單一環境類別中直接以 Python 實作各方法，不經 Manager 層：

from isaaclab.envs import DirectRLEnv

class MyEnv(DirectRLEnv):
    def _get_observations(self) -> dict:
        obs = torch.cat([...], dim=-1)
        return {"policy": obs}

    def _get_rewards(self) -> torch.Tensor:
        return compute_reward(...)

    def _get_dones(self) -> tuple[torch.Tensor, torch.Tensor]:
        terminated = ...
        truncated = self.episode_length_buf >= self.max_episode_length - 1
        return terminated, truncated

Direct 控制更直接、開銷更低、除錯路徑更短，貼近傳統 Isaac Gym 寫法；Manager-Based 則模組化、易複用與組合。兩者選擇取決於任務規模與客製需求。

致動器（Actuators）#

致動器是連接「策略輸出」與「關節實際動作」的模型。真實馬達並非理想力源，選對致動器模型對 sim-to-real 至關重要。Isaac Lab 提供：

類型說明
Implicit（隱式）由 PhysX 內部以 PD 增益求解，效能最佳，適合多數關節
Ideal PD（顯式）在 Python 端以理想 PD 計算力矩，行為透明可控
DC Motor加入力矩—速度飽和曲線，貼近直流馬達特性
Actuator Net（致動器網路）以神經網路擬合真實致動器（如 ANYmal 的 MLP/LSTM），最擬真

感測器（Sensors）#

Isaac Lab 內建多種感測器，輸出皆以向量化張量提供，可直接進入觀測：

  • Camera：RGB、深度、語意分割、實例分割等多種資料型別。
  • Contact Sensor：偵測接觸力與接觸事件（如足端著地、碰撞）。
  • IMU：輸出線加速度、角速度與姿態。
  • Ray Caster：以射線投射高效取得地形高度圖或距離量測（比實體光達便宜許多）。
  • Frame Transformer：追蹤剛體／關節座標系之間的相對位姿。
  • 視覺觸覺感測器（Visuo-Tactile）：模擬觸覺影像回饋。

運動生成器（Motion Generators）#

當你希望策略在「任務空間」而非「關節空間」下指令時，運動生成器負責把末端執行器目標轉為關節指令：

  • 差分逆運動學（Differential IK）：以雅可比矩陣把末端執行器的位置／姿態增量轉為關節速度，適合位置控制的機械臂任務。
  • 操作空間控制（Operational Space Control, OSC）：在任務空間直接控制力／阻抗，適合接觸密集或需柔順（compliant）行為的任務。
