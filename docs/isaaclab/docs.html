1. 什麼是 Isaac Lab#

原文連結

NVIDIA Isaac Lab 是一個開源、GPU 加速的機器人學習（robot learning）框架，建構於 NVIDIA Isaac Sim 之上。它的目標是簡化機器人研究中常見的工作流程——強化學習（RL）、從示範學習（imitation learning）與運動規劃（motion planning）——讓研究者把心力放在演算法本身，而非模擬基礎設施。

設計上，Isaac Lab 強調四個原則：

  • 模組化（Modular）：以可組合的元件（Manager、致動器、感測器）拼裝環境，容易客製與擴充。
  • 敏捷（Agile）：從想法到可訓練環境的迭代週期短。
  • 開放（Open）：以開源授權釋出，程式碼、資產與環境皆可取用與修改。
  • 內建即用（Batteries-included）：附帶大量現成的環境、機器人資產、感測器與任務。

核心能力#

  • 大規模平行模擬：以 PhysX 在單一 GPU 上同時模擬數千個環境實例，向量化收集經驗。
  • 向量化渲染（Tiled Rendering）：在 GPU 上一次渲染大量攝影機視角，支援視覺型（vision-based）策略訓練。
  • 域隨機化（Domain Randomization）：對物理參數、外觀、光照等進行隨機化，強化 sim-to-real 的泛化能力。
  • 豐富感測器：攝影機、接觸感測器、IMU、ray caster、frame transformer 與視覺觸覺感測器。
  • 多框架 RL 整合：開箱即用地對接 RSL-RL、RL-Games、SKRL、Stable-Baselines3。
  • 雲端與叢集部署：支援容器化、多 GPU／多節點訓練與 Ray 任務派發。

一句話定位：Isaac Sim 是「機器人模擬器應用」，Isaac Lab 是架在它之上、專為「大規模機器人學習」而生的訓練框架。

2. 生態系與定位#

原文連結

Isaac Lab 位於 NVIDIA 機器人模擬堆疊的上層，各層分工如下：

層級角色
OpenUSD統一的場景描述格式，資產、機器人與環境皆以 USD 表示
PhysXGPU 加速物理引擎，提供大規模平行剛體／關節模擬
Isaac Sim建立在 Omniverse Kit 上的機器人模擬器：渲染、感測器、資產匯入
Isaac Lab機器人學習框架：環境抽象、Manager 系統、RL／IL 訓練管線

與 Isaac Gym / Orbit 的關係#

Isaac Lab 是先前多個專案的統一後繼者，取代並整併了：

  • Isaac Gym（IsaacGymEnvs）：NVIDIA 早期的 GPU RL 環境集。
  • OmniIsaacGymEnvs（OIGE）：建構於 Isaac Sim 的 RL 環境集。
  • Orbit：模組化機器人學習框架，是 Isaac Lab 的直接前身。

官方為上述三者各提供了遷移指南，協助既有專案升級到 Isaac Lab。

3. 安裝#

原文連結（完整步驟見本站安裝與部署頁）

系統需求#

項目需求
作業系統Ubuntu 22.04 或 Windows 11
記憶體至少 32 GB RAM
GPU VRAM至少 16 GB
Python需與 Isaac Sim 綁定的版本一致（Isaac Sim 5.X 為 Python 3.11）
驅動程式Linux 建議 NVIDIA driver 580.65.06 以上

四種安裝路徑#

方式說明難度
Pip 安裝（推薦）以 pip 安裝 Isaac Sim，Isaac Lab 從原始碼（git）安裝，最簡單快速容易
二進位安裝下載 Isaac Sim 預建二進位包，Isaac Lab 從原始碼安裝；適合避開 pip 或較舊系統中等
完整原始碼建置Isaac Sim 與 Isaac Lab 皆從原始碼建置；僅在需要修改 Isaac Sim 本身時採用進階
僅 pip 套件兩者皆以 pip 套件安裝，只支援外部擴充，不含訓練／範例特殊用途

前置條件：安裝 Isaac Lab 前必須先安裝 Isaac Sim——Isaac Lab 建構於其上。Isaac Sim 4.2.0 以下版本已不再支援。

從原始碼安裝後，日常操作透過 isaaclab.sh（Windows 為 isaaclab.bat）進行：

# 安裝（含指定 RL 框架相依）
./isaaclab.sh --install       # 或 -i

# 執行工作區的 Python（會使用 Isaac Sim 綁定的直譯器）
./isaaclab.sh -p scripts/tutorials/00_sim/create_empty.py

# 驗證安裝：開一個空模擬視窗
./isaaclab.sh -p scripts/tutorials/00_sim/launch_app.py

4. 核心架構#

原文連結（詳見本站核心概念與架構頁）

Isaac Lab 以一組層層堆疊的抽象組織模擬：

層級職責
App（AppLauncher）啟動並初始化底層 Isaac Sim／Omniverse 應用，處理命令列參數（headless、裝置、渲染模式等）
Simulation（SimulationContext）管理物理與渲染引擎、時間步進與模擬設定
Scene（InteractiveScene）容納被生成的物件：機器人、剛體、感測器、地形，並處理多環境複製（clone）
EnvironmentRL 相容介面：標準化的 reset()／step()、觀測、獎勵與終止
Managers模組化元件，分別負責觀測、獎勵、動作、終止、事件（隨機化）等邏輯

其中 InteractiveScene 是關鍵：你只需描述「一個環境」長什麼樣，框架便會自動把它複製成數千份平行環境，並以向量化張量統一存取所有實例的狀態。

5. 兩種工作流程#

原文連結

Isaac Lab 提供兩種設計任務的方式，兩者可訓練出相同的策略，差別在於程式碼的組織風格：

Manager-Based 工作流程#

把環境拆解成多個設定類別（config classes），交由對應的 Manager 執行：

  • ObservationManager：組裝觀測向量。
  • RewardManager：加總各項獎勵。
  • TerminationManager：判定回合結束（超時、跌倒、成功等）。
  • ActionManager：把策略輸出映射到致動器指令。
  • EventManager：在重置或時間軸上套用隨機化與事件。

優點是高度模組化、易於複用與組合——換一項獎勵或觀測只要改設定，適合大型、需長期維護的任務。

Direct 工作流程#

在單一環境類別中，用 Python 直接實作 _get_observations()、_get_rewards()、_get_dones() 等方法，不經 Manager 層。

優點是控制更直接、效能開銷更低、除錯路徑更短，也更接近傳統 Isaac Gym 的寫法；適合研究性質、需要細緻掌控或高度客製化獎勵邏輯的任務。

Manager-BasedDirect
組織方式設定類別 ＋ Manager單一環境類別、直接實作方法
複用／組合強中
控制粒度中高
適合場景大型、需維護、元件複用研究、客製、效能敏感

6. 環境設計與 Manager 系統#

設計一個可訓練的環境，典型流程為：

  1. 定義 Scene：用 InteractiveSceneCfg 宣告地形、機器人、物件與感測器。
  2. 定義 MDP 元件：以設定類別描述觀測、動作、獎勵、終止與事件（Manager-Based），或直接實作方法（Direct）。
  3. 組裝環境設定：把上述組合成 ManagerBasedRLEnvCfg 或 DirectRLEnvCfg，並設定 num_envs、episode_length_s、decimation 等。
  4. 註冊環境：透過 gymnasium.register() 給環境一個 Isaac-...-v0 的 ID，即可用 gym.make() 建立。

致動器（Actuator）是另一關鍵抽象：Isaac Lab 提供理想 PD、隱式（implicit）、顯式（explicit）與致動器網路（actuator net）等模型，讓關節控制更貼近真實硬體，改善 sim-to-real。感測器與運動生成器（差分逆運動學 IK、操作空間控制 OSC）詳見核心概念與架構頁。

7. 可用環境#

原文連結（完整清單見本站可用環境頁）

Isaac Lab 內建大量現成任務，涵蓋：

  • 經典控制：Cartpole、Humanoid、Ant。
  • 機械臂操作：伸取、抬升方塊、開抽屜（Franka、UR10 等）。
  • 接觸密集裝配：插銷、齒輪嚙合、螺帽鎖固；AutoMate（100 種插接任務）、FORGE（含力感測與動力學隨機化）。
  • 足式移動：平地與崎嶇地形的速度追蹤（ANYmal、Unitree、Spot）。
  • 導航：朝向目標位置與朝向前進。
  • 多旋翼：無人機控制、Quadcopter 懸停。
  • 多智能體：支援同時控制多個智能體的經典與操作任務。

任務命名慣例為 Isaac-[任務]-[機器人]-[變體]-[工作流程]-v[版本]，例如 Isaac-Lift-Cube-Franka-v0（Manager-Based）、Isaac-Velocity-Rough-Anymal-C-Direct-v0（Direct）。

8. 強化學習訓練#

原文連結（詳見本站強化學習頁）

Isaac Lab 開箱支援四大 RL 框架：RSL-RL、RL-Games、SKRL、Stable-Baselines3。每個框架都遵循一致的流程——安裝框架模組、以 --task 指定任務執行訓練、再用 checkpoint 回放評估：

# 安裝所需 RL 框架
./isaaclab.sh -i rl_games

# 訓練（headless 加速，4096 個平行環境）
./isaaclab.sh -p scripts/reinforcement_learning/rl_games/train.py \
    --task Isaac-Ant-v0 --headless --num_envs 4096

# 回放已訓練策略
./isaaclab.sh -p scripts/reinforcement_learning/rl_games/play.py \
    --task Isaac-Ant-v0 --num_envs 32

各框架皆支援 headless 執行、多環境回放、載入預訓練 checkpoint、錄製影片（需 ffmpeg）與 TensorBoard 記錄（輸出至 logs/）。設定管理採用 Hydra，可在命令列直接覆寫超參數。

9. 進階功能與資源#

Isaac Lab 另提供多項面向規模化與研究再現的功能：

  • Hydra 設定系統：以組合式設定管理環境與代理超參數，命令列即可覆寫。
  • 多 GPU／多節點訓練：跨多張 GPU 與多台機器擴展訓練吞吐。
  • 群體式訓練（PBT）：Population Based Training 自動調整超參數。
  • Tiled Rendering：向量化攝影機渲染，支撐視覺型策略。
  • Ray 任務派發：分散式訓練與超參數搜尋。
  • 可再現性與決定性：控制隨機種子與模擬決定性以利實驗重現。
  • 實驗性功能：Newton Physics 整合。
  • Sim2Real：策略部署到真實機器人的指引。

更多內容見官方 Features、API 參考 與 疑難排解。
