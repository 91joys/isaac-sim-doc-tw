強化學習#

整理自官方 Reinforcement Learning 章節。Isaac Lab 開箱支援四大 RL 框架，並以 Hydra 管理設定、支援多 GPU／多節點訓練。

四大 RL 框架#

框架特點模組名
RSL-RL輕量、訓練快；支援策略蒸餾（teacher-student）；ETH／NVIDIA 足式移動研究常用rsl_rl
RL-Games高效能、支援分散式訓練；效能導向rl_games
SKRL同時支援 PyTorch 與 JAX 後端；內建多智能體演算法（MAPPO、IPPO）skrl
Stable-Baselines3整合簡單、社群成熟；適合單／多環境入門sb3

安全提醒：RL-Games 搭配 Ray 工作流程做分散式訓練或超參數搜尋時，因 Ray 的安全風險，不建議在非受控網路環境下使用。

選擇建議#

  • 足式移動／需策略蒸餾 → RSL-RL：與官方 locomotion 任務契合度高、上手快。
  • 追求極致吞吐 → RL-Games：大規模平行下效能佳。
  • 多智能體或想用 JAX → SKRL：後端與演算法選擇最廣。
  • 熟悉 SB3／教學入門 → Stable-Baselines3：API 熟悉、除錯資源多。

訓練與回放#

每個框架都遵循一致三步：安裝框架模組 → 以 --task 指定任務訓練 → 用 checkpoint 回放評估。腳本位於 scripts/reinforcement_learning/<框架>/。

RSL-RL 範例#

# 安裝框架相依
./isaaclab.sh -i rsl_rl

# 訓練（headless、4096 個平行環境）
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
    --task Isaac-Velocity-Rough-Anymal-C-v0 --headless --num_envs 4096

# 回放最新 checkpoint（帶畫面、少量環境）
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
    --task Isaac-Velocity-Rough-Anymal-C-v0 --num_envs 32

其他框架#

# RL-Games
./isaaclab.sh -i rl_games
./isaaclab.sh -p scripts/reinforcement_learning/rl_games/train.py --task Isaac-Ant-v0 --headless

# SKRL
./isaaclab.sh -i skrl
./isaaclab.sh -p scripts/reinforcement_learning/skrl/train.py --task Isaac-Cartpole-v0 --headless

# Stable-Baselines3
./isaaclab.sh -i sb3
./isaaclab.sh -p scripts/reinforcement_learning/sb3/train.py --task Isaac-Cartpole-v0 --headless

各框架共通支援：headless 執行、自訂平行環境數、載入預訓練 checkpoint、錄製影片（需 ffmpeg）、TensorBoard 記錄（輸出至 logs/）。

# 以 TensorBoard 檢視訓練曲線
./isaaclab.sh -p -m tensorboard.main --logdir logs

Hydra 設定系統#

原文連結

Isaac Lab 以 Hydra 管理環境與代理設定，可在命令列直接覆寫任意設定欄位，無需改程式碼：

# 覆寫環境數與部分環境參數
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
    --task Isaac-Cartpole-v0 --headless \
    env.scene.num_envs=2048 \
    agent.max_iterations=500

規模化與再現#

  • 多 GPU／多節點訓練：以分散式資料平行擴展吞吐，見官方 Multi-GPU/Multi-Node。
  • 群體式訓練（PBT）：自動搜尋超參數。
  • Ray 任務派發：分散式訓練與超參數搜尋（注意 Ray 安全性）。
  • 可再現性與決定性：固定隨機種子並控制模擬決定性以重現實驗。

Sim2Real 部署#

訓練完成的策略可輸出（如 TorchScript／ONNX）並部署到真實機器人。搭配擬真致動器模型與域隨機化，能顯著縮小 sim-to-real 落差。官方另提供 Sim2Real Policy Deployment 指引。
