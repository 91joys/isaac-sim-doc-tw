Newton Physics 總覽文檔#

對應官方 Overview 與 FAQ 章節的中文整理。本頁幫你在 10 分鐘內建立 Newton 的完整心智模型。

1. Newton 是什麼#

Newton 是一套 GPU 加速、可擴充、可微分的開源物理模擬引擎，專為機器人學、研究與先進模擬工作流程設計。它建構於 NVIDIA Warp 之上並整合 MuJoCo Warp，提供高效能模擬、現代化 Python API 與彈性的架構。

  • GPU 加速：以 NVIDIA Warp 實現快速、可擴展的模擬。
  • 多求解器：XPBD、VBD、MuJoCo、Featherstone、SemiImplicit、Kamino、ImplicitMPM、Style3D 共 8 種實作。
  • 模組化設計：容易擴充新的求解器與元件（可自製求解器）。
  • 可微分：支援可微分模擬，用於機器學習與最佳化。
  • 豐富的匯入／匯出：支援 URDF、MJCF、USD 等資產格式。
  • 開源：由 Disney Research、Google DeepMind 與 NVIDIA 發起的 Linux Foundation 專案，採 Apache-2.0 授權。

1.1 與相關專案的關係#

專案關係
NVIDIA WarpNewton 的基底。Warp 是撰寫高效能、可微分 GPU kernel 的 Python 框架；Newton 在其上加入高階模擬 API、可互換求解器與資產 I/O，成為完整的物理引擎。
warp.simNewton 的前身（Warp 內建模組），已於 Warp 1.8 棄用、1.10 移除。遷移方式見 warp.sim 遷移指南。
MuJoCo WarpNewton 的關鍵求解器之一——由 Google DeepMind 與 NVIDIA 以 Warp 重新實作的 GPU 加速版 MuJoCo。Newton 在資產（MJCF）與求解器兩個層面都與 MuJoCo 相容。
PhysX兩者目標不同、互不取代：Newton 主攻機器人學習與可微分多物理；PhysX 主攻工業數位孿生的成熟即時物理 SDK。PhysX 目前無法作為 Newton 的求解器。
Isaac Lab／Isaac SimIsaac Lab 已提供實驗性 Newton 整合（四足／雙足行走與基本操作環境）；Isaac Sim 的 Newton 物理後端整合開發中。詳見 FAQ・整合。

2. 核心架構：七大物件#

Newton 的資料流：資產（URDF/MJCF/USD）→ Importer → ModelBuilder → Model → State ⇄ Solver（搭配 Control 與 Contacts）→ Sensors / Viewer。

物件角色
ModelBuilder建構模擬模型的入口——用程式加入基本體，或匯入外部資產（add_urdf()、add_mjcf()、add_usd()），最後 finalize() 產出 Model。
Model封裝模擬世界的物理結構、參數與組態：剛體、關節、形狀與物理性質。建立後不隨時間變動。
State某個時間點的動態狀態（位置、速度），由求解器逐步更新。可選的「擴充屬性」可儲存衍生量（如剛體加速度）供感測器使用。
Contacts由 Model.collide() 產生的當前接觸集合，可選擴充屬性（如接觸力）供感測與分析。
Control控制輸入——關節目標（target）與力（force），在模擬迴圈中施加。
Solver推進模擬：積分動力學、處理接觸、滿足約束。8 種後端可選，見求解器。
Sensors／Viewer感測器從 State、Contacts、sites 與 shapes 計算觀測值；Viewer 即時或離線視覺化。

3. 模擬迴圈六步驟#

  1. 用 ModelBuilder 建構或匯入模型。
  2. finalize() 產出 Model。
  3. 建立感測器，配置一或多個 State、Control 與 Contacts。
  4. 呼叫 Model.collide() 填入當前狀態的接觸集合。
  5. 以當前 state、control、contacts 呼叫求解器 step()。
  6. 更新感測器、檢查輸出、渲染或匯出結果。
import warp as wp
import newton

# 1-2. 建構模型
builder = newton.ModelBuilder()
body = builder.add_body(
    xform=wp.transform((0.0, 1.0, 0.0), wp.quat_identity()),
    mass=1.0,
)
builder.add_shape_sphere(body, radius=0.25)
builder.add_ground_plane()
model = builder.finalize()

# 3. 求解器與狀態
solver = newton.solvers.SolverXPBD(model)
state_0 = model.state()
state_1 = model.state()
control = model.control()
contacts = model.contacts()

newton.eval_fk(model, model.joint_q, model.joint_qd, state_0)

# 4-5. 步進模擬
for step in range(120):
    state_0.clear_forces()
    model.collide(state_0, contacts)
    solver.step(state_0, state_1, control, contacts, 1.0 / 60.0)
    state_0, state_1 = state_1, state_0

雙 State 交換（ping-pong）：求解器從 state_0 讀、寫入 state_1，每步後交換兩者——這是 Newton 的標準模式，也讓 CUDA Graph 捕捉成為可能。

4. 大規模平行：一行複製 1024 個世界#

Newton 為 RL 訓練場景設計了「多世界（Worlds）」機制——同一個 Model 內可含上千個互不碰撞的獨立世界，由 GPU 一次全部步進：

# 建構單一機器人模板
template = newton.ModelBuilder()
template.add_mjcf("humanoid.xml")

# 複製為 1024 個平行世界
builder = newton.ModelBuilder()
builder.replicate(template, world_count=1024)
builder.add_ground_plane()
model = builder.finalize()

# 求解器同時步進全部 1024 個世界
solver = newton.solvers.SolverMuJoCo(model)

細節（世界指派、每世界重力、GPU 執行緒分割）見核心概念・多世界。
