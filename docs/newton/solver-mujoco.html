SolverMuJoCo#

對應官方 newton.solvers.SolverMuJoCo API 頁面與 MuJoCo 後端指南(1.4.0 stable)。繼承 SolverBase 與 CouplingInterface。Newton 以相容版本鎖定(~=)同時釘住 mujoco 與 mujoco-warp 以保持版本對齊。

概述#

SolverMuJoCo 提供介接 MuJoCo 物理引擎的介面,並透過 mujoco_warp 做 GPU 加速。它同時支援 MuJoCo(CPU)與 mujoco_warp(GPU)兩種後端,能高效模擬帶接觸與約束的 articulation 系統——是機器人 RL 訓練的主力求解器。

📦 安裝需求:此求解器需要安裝 mujoco_warp 及其相依套件,安裝方式見 mujoco_warp 儲存庫。另外注意:Newton 模型的 shape_collision_radius 不會被 MuJoCo 使用——MuJoCo 會依幾何定義自行計算包圍球半徑(geom_rbound)。

基本用法#

solver = newton.solvers.SolverMuJoCo(model)

# 模擬迴圈
for i in range(100):
    solver.step(state_in, state_out, control, contacts, dt)
    state_in, state_out = state_out, state_in

預設使用 MuJoCo 自己的碰撞管線(use_mujoco_contacts=True),此時 step() 的 contacts 可傳 None;若設為 False 改用 Newton 的接觸求解,就必須把 Newton 的 contacts 傳進 step()。

參數解析優先序#

求解器選項(例如 impratio)依以下優先序解析:

  1. 建構子引數——若提供,所有 world 使用相同值。
  2. Newton 模型自訂屬性(model.mujoco.<option>)——支援 per-world 值。
  3. MuJoCo 預設值——上述皆未設定時使用。

除錯:匯出 MJCF 與 MuJoCo 檢視器#

可以把建構子從 newton.Model 產生的 MuJoCo 模型存成 MJCF 檔,再用 MuJoCo 模擬器打開檢查:

solver = newton.solvers.SolverMuJoCo(model, save_to_mjcf="model.xml")

也可以透過 MuJoCo 自己的檢視器觀看模擬,確認 Newton 模型轉成 MuJoCo 模型後長什麼樣子:

import newton

solver = newton.solvers.SolverMuJoCo(model)

for _ in range(num_frames):
    # 推進模擬
    solver.step(state_in, state_out, control, contacts, dt)
    state_in, state_out = state_out, state_in

    solver.render_mujoco_viewer()

⚠️ MuJoCo 檢視器只支援渲染單一 world 的 Newton 模型,除非 use_mujoco_cpu=True 或建構時設 separate_worlds=False。檢視器僅作為除錯工具。render_mujoco_viewer() 可用參數控制左右 UI、接觸點/接觸力顯示與透明幾何(show_left_ui、show_right_ui、show_contact_points、show_contact_forces、show_transparent_geoms);用 close_mujoco_viewer() 關閉。

reset():發散後的正確重置#

solver.reset(state, world_mask=None, flags=None)

把關節狀態重置為模型預設值,並清除 MuJoCo 的內部緩衝。這在 RL 訓練重置環境時特別重要:MuJoCo 會跨 step() 保留求解器狀態(加速度 warm-start、致動器 activation)與外力輸入,模擬發散(如 NaN)之後,即使關節狀態已重置,這些緩衝仍可能「毒害」下一步。因此 reset() 一律per-world 歸零 qacc_warmstart、qfrc_applied、xfrc_applied、act 與 ctrl。

  • world_mask:形狀 (world_count,) 的布林遮罩,選擇要重置哪些 world;None 表示全部。
  • flags:StateFlags 位元遮罩,控制重置哪些量——JOINT_Q 重置 state.joint_q、JOINT_QD 重置 state.joint_qd;None 表示全部重置。內部 MuJoCo 緩衝無論如何都會清除。
  • MuJoCo 是廣義座標求解器,body_q/body_qd 會在下一次 step() 由前向運動學從關節座標導出,所以 BODY_Q/BODY_QD(及粒子)flag 在此無效、會被忽略。
  • 當 update_data_interval != 1 時,每步的 qpos/qvel 同步是關閉或稀疏的,重置後的關節座標會立即推入 qpos/qvel。

控制與致動器(CtrlSource / CtrlType)#

MuJoCo 致動器的控制輸入來源由 SolverMuJoCo.CtrlSource 決定:

枚舉值說明
CtrlSource.JOINT_TARGET (0)從 Newton 的 joint_target_q/joint_target_qd 陣列映射(Newton 標準控制介面)。
CtrlSource.CTRL_DIRECT (1)直接使用 control.mujoco.ctrl(MuJoCo 原生控制)。

JOINT_TARGET 模式下,CtrlType 決定讀哪個目標陣列:

枚舉值說明
CtrlType.POSITION (0)從 joint_target_q 映射,增益取自 joint_target_ke;僅位置控制的致動器同時從 joint_target_kd 同步阻尼,POSITION_VELOCITY 模式的 kd 則由獨立的速度致動器處理。
CtrlType.VELOCITY (1)從 joint_target_qd 映射,增益取自 joint_target_kd。
CtrlType.GENERAL (2)搭配 CTRL_DIRECT 使用,對應 motor/general 致動器。

另有兩個輔助枚舉:EqType 描述 MuJoCo 等式約束類型(CONNECT=像球關節把兩剛體約束在一點、WELD=像固定關節焊接兩剛體、JOINT=把一個純量關節座標約束為另一個的四次多項式,即 mimic);TrnType 描述致動器傳動類型(JOINT、JOINT_IN_PARENT、TENDON、SITE、BODY、SLIDERCRANK)。

關節支援與限制#

  • 支援的關節類型:PRISMATIC、REVOLUTE、BALL、FIXED、FREE、D6;不支援 DISTANCE 與 CABLE。
  • 支援的屬性/控制:joint_armature、joint_friction、joint_effort_limit、joint_limit_ke/joint_limit_kd、joint_target_ke/joint_target_kd、joint_target_mode、joint_f——是所有求解器中支援最完整的。
  • 支援等式約束(CONNECT、WELD、JOINT)與 mimic 約束(僅 REVOLUTE 與 PRISMATIC)。
  • 不支援:joint_velocity_limit、joint_enabled。

完整跨求解器比較見官方 Joint Feature Support,或本站求解器總覽的關節支援矩陣。

Newton → MuJoCo 關節類型映射#

MuJoCo 有自己的建模慣例,許多 Newton 屬性的映射方式不同(或根本不映射)。以下各節整理自官方 1.4.0 新增的 MuJoCo 後端指南。

Newton 類型MuJoCo 對應備註
FREEmjJNT_FREE初始位姿取自 body_q。
BALLmjJNT_BALL每軸致動器透過 gear 映射。
REVOLUTEmjJNT_HINGE
PRISMATICmjJNT_SLIDE
D6最多 3×mjJNT_SLIDE＋3×mjJNT_HINGE每個啟用的線性/角度 DOF 各自成為一個 MuJoCo 關節,加 _lin/_ang 後綴(同群多軸時再附數字索引,如 _lin0、_lin1)。
FIXED(無關節)子 body 直接巢狀在父 body 下;連到世界的固定關節會產生 mocap body,由 mjData.mocap_pos/mocap_quat 驅動。
DISTANCE(無關節)距離約束被捨棄,但 body 記帳如自由 body 處理。
CABLE不支援不轉發給 MuJoCo。

幾何類型映射#

Newton 類型MuJoCo 對應備註
SPHERE/CAPSULE/CYLINDER/BOX/ELLIPSOIDmjGEOM_* 同名直接對應。
PLANEmjGEOM_PLANE必須附著在靜態 body(body=-1),否則轉換時拋 ValueError。MuJoCo 的平面碰撞永遠是無限大,shape_scale 只影響渲染(未設時預設 5×5×5)。
HFIELDmjGEOM_HFIELD高度場資料以 [0,1] 正規化儲存並原樣轉發;geom 原點依 min_z 平移,使最低點落在正確的世界高度。
MESH/CONVEX_MESHmjGEOM_MESHMuJoCo 只支援凸碰撞網格;非凸網格會被 MuJoCo 編譯器(而非 Newton)取凸包,改變碰撞邊界。網格來源的 maxhullvert 會被轉發。
CONE、GAUSSIAN不支援MuJoCo geom 類型表中不存在。

帶 SITE 旗標的 shape 轉為 MuJoCo site(不參與碰撞的參考座標系,用於感測器附著與空間肌腱纏繞錨點)。只有 SPHERE、CAPSULE、CYLINDER、BOX 是 MuJoCo 原生 site 幾何,其他類型會靜默退回 SPHERE。另外,非凸 trimesh、SDF 接觸、hydroelastic 接觸等 Newton 碰撞功能不屬於 MuJoCo 幾何模型,只能透過 Newton 碰撞管線(use_mujoco_contacts=False)取得。

致動器映射(joint_target_mode)#

Newton 的 per-DOF joint_target_mode 會建立對應的 MuJoCo general actuator:

模式建立的 MuJoCo 致動器
POSITION1 個:gainprm=[kp]、biasprm=[0, -kp, -kd]
VELOCITY1 個:gainprm=[kd]、biasprm=[0, 0, -kd]
POSITION_VELOCITY2 個——位置致動器(gainprm=[kp]、biasprm=[0, -kp, 0])＋速度致動器(gainprm=[kd]、biasprm=[0, 0, -kd])
NONE、EFFORT不建立致動器

joint_effort_limit 轉發為關節上的 actfrcrange(prismatic/revolute/D6)或致動器上的 forcerange(ball)。完整的 MuJoCo general-actuator 模型(任意 gain/bias/dynamics、顯式 transmission、ctrl/force/act range)只能透過 mujoco 自訂屬性命名空間取得,以此宣告的額外致動器會排在 joint-target 致動器之後。

等式約束與 Mimic#

MuJoCo 等式約束類型請用 SolverMuJoCo.EqType(頂層別名 newton.EqType 於 Newton 1.4 起棄用):CONNECT(錨點在 data[0:3])、WELD(錨點＋相對位姿＋力矩縮放)、JOINT(多項式係數 data[0:5]);Mimic 經 add_constraint_mimic() 轉為 mjEQ_JOINT,僅支援 REVOLUTE 與 PRISMATIC。Newton 核心 API 沒有專門的等式約束 builder 呼叫,要透過自訂屬性建構:

builder.add_custom_values(
    **{
        "mujoco:equality_constraint_type": int(newton.solvers.SolverMuJoCo.EqType.CONNECT),
        "mujoco:equality_constraint_body1": body1,
        "mujoco:equality_constraint_body2": body2,
        "mujoco:equality_constraint_anchor": wp.vec3(0.0, 0.0, 0.0),
        "mujoco:equality_constraint_enabled": True,
        "mujoco:equality_constraint_world": builder.current_world,
    }
)

迴圈閉合(loop closures)#

閉迴圈關節不會產生 MuJoCo 關節,而是按關節類型約束兩 body 的相對運動:FIXED 鎖 6 個相對 DOF;REVOLUTE 鎖 5 個(繞鉸軸的旋轉自由);BALL 鎖 3 個平移 DOF。其他類型(PRISMATIC、FREE、DISTANCE、CABLE)用作迴圈閉合時會警告並被靜默跳過——迴圈不會閉合;D6 依 DOF 分派(1 個角軸=revolute 閉合、3 個=ball 閉合,其他組態跳過)。閉迴圈關節上撰寫的任何驅動(PD 目標、joint_f)、極限、armature、摩擦與力矩/速度上限都會被忽略,其 DOF 也不計入 MuJoCo 的 nq/nv。

執行期狀態同步#

每次 step() 都經過三階段:

  1. 推送 Newton → MuJoCo:把 Newton 的 State/Control 輸入寫進 MuJoCo 工作資料;use_mujoco_contacts=False 時同時轉換 Newton 側接觸。關節狀態重同步頻率由 update_data_interval 控制(用於 substepping)。
  2. 積分:mujoco_warp 把 MuJoCo 模型往前推 dt。
  3. 拉回 MuJoCo → Newton:從積分後的 MuJoCo 資料填充輸出 State;kinematic root 直接沿用 state_in 的值。

⚠️ 接觸不會自動拉回 Newton 的 Contacts 物件——需要接觸點、力或材質索引時,請顯式呼叫 update_contacts()。

多 world 支援#

separate_worlds=True(多 world GPU 模式的預設)只從第一個 world 建構 MuJoCo 模型,再由 mujoco_warp 複製到所有 world。這要求所有 Newton world 結構完全相同(相同的 body、關節、shape),建構時驗證、不符則拋 ValueError。Body、關節、等式約束與 mimic 約束不能有負的 world 索引;只有 shape 可以放在全域 world(-1),它們會跨 world 共享而不複製。

Margin/gap 與 multi-CCD#

shape_margin 映射到 MuJoCo geom_margin、shape_gap 映射到 geom_gap(接觸對值同理映射到 pair_margin/pair_gap)。MuJoCo 把表面距離落在 (margin, margin+gap] 的接觸回報為不出力的非作用接觸。MJCF/USD 的 margin/gap 採 MuJoCo 3.9+ 直接語意;只有要重現 Newton 3.9 前的匯入轉換時才傳 legacy_margin_gap=True 給 add_mjcf()/add_usd()。

  • Multi-contact CCD:enable_multiccd=True 允許每對 geom 最多 4 個接觸點(預設 1 個);但任一 geom 的 MuJoCo geom_margin 非零時,該對仍退回單一接觸。
  • Margin 歸零:mujoco_warp 拒絕 box-box 對(其預設 NATIVECCD 路徑)及 enable_multiccd=True 時任何 box/mesh 對的非零 margin。為保持相容,只要模型存在 box geom(或 multiccd＋mesh 的組合),SolverMuJoCo 會在編譯期把 geom_margin 全模型歸零;Newton 模型的 shape_margin 陣列不受影響,且 use_mujoco_contacts=False 時會在執行期還原撰寫的 margin。

關節極限與接觸剛性/阻尼的 solref 轉換#

joint_limit_ke/kd 與 shape_material_ke/kd 都是力空間增益(如 N·m/rad、N/m),而 MuJoCo 用 solref=(timeconst, dampratio) 慣例。SolverMuJoCo 會先以逆質量權重(dof_invweight0 或兩 body 的 body_invweight0 和)與阻抗參數 1-dmax 縮放增益,再轉成正值 solref,以保留 Newton 的力空間語意,同時允許 MuJoCo 的 refsafe 時步鉗制軟化對步長而言過硬的約束。重點:

  • 從 MJCF/USD 撰寫的 solreflimit/solref 是原生 MuJoCo 參數,原樣保留、不重縮放;內部以 solreflimit_mode/solref_mode 記錄該值是力空間增益、原生撰寫值還是 MJCF 隱含預設(這些 SOLREF_MODE_* 是內部常數,並非公開 API)。
  • joint_limit_ke <= 0 或 joint_limit_kd <= 0 時還原 MuJoCo 預設 solreflimit=(0.02, 1.0)。
  • refsafe 啟用時,正值 solref 的有效 timeconst 有 2*dt 下限——一旦到達下限,再拉高增益也不會更硬,應改縮小時間步長。
  • use_mujoco_contacts=True 與 MuJoCo CPU 後端不套用逐接觸的兩體縮放因子,力空間 shape 會退回舊式 convert_solref(ke, kd, 1, 1) 單位質量近似。
  • save_to_mjcf 對力空間模式的關節不是完整語意的往返匯出——MJCF 只儲存 solreflimit,重建模型後需重新套用 joint_limit_ke/kd。

參數解讀、穩定性取捨與任務導向的調校指引見官方 MuJoCo-Warp Contact Tuning。

MuJoCo 專屬參數(USD/MJCF 與自訂屬性)#

MuJoCo 有許多 Newton 核心 API 沒有對應的參數,匯入 MJCF/USD 時透過兩種機制處理:

  1. 自訂屬性命名空間:專屬的 mujoco 命名空間(model.mujoco.<name>)由 MJCF 元素與 OpenUSD MuJoCo schema 屬性(mjc:*)填充。啟用方式是在加入任何內容之前先在 ModelBuilder 上呼叫 SolverMuJoCo.register_custom_attributes(builder)。肌腱(tendon)與 MuJoCo 式 <pair> 接觸覆寫也都經由此命名空間實作(spatial tendon 支援 site/geom/pulley 纏繞元素,其他類型警告後跳過)。
  2. 直接映射到 Newton 內建屬性:部分 MuJoCo 屬性在匯入時直接映射為 Newton 內建屬性,例如由 solreflimit 導出的關節極限剛性/阻尼;MuJoCo 關節 damping 映射為 joint_damping(舊的 model.mujoco.dof_passive_damping 是棄用別名,存取時發 DeprecationWarning)。

求解器選項在建構時讀取一次;之後編輯 model.mujoco.<option> 不會生效——解析後的值已固化在底層 MuJoCo 模型中。

不支援的 MuJoCo 功能#

反方向上,載入 MJCF/USD 資產時以下 MuJoCo 建模概念不會被匯入,轉換時也不會重建:

  • 感測器(<sensor>)——Newton 有自己獨立的感測器管線。
  • 相機與燈光——Newton 使用自己的 viewer 與照明管線,來源資產中的相機/燈光 primitive 被忽略。
  • Keyframes(<keyframe>)——MuJoCo 的存檔狀態/重置機制。
  • Composite 與 flex(<composite>、<flex>)——MuJoCo 內建可變形體;Newton 有專門的布料/MPM/FEM 求解器。
  • Skinned meshes(<skin>)——僅供視覺化,不匯入。
  • 使用者外掛(<plugin>)與任意自訂元素(<custom>、<numeric>、<text>)——Newton 專屬使用者資料請改用自訂屬性系統。
  • 致動器傳動——僅支援 joint、tendon、site、body;jointinparent 與 slidercrank 不轉換,使用它們的致動器會在建構時警告並跳過。

注意事項(Caveats)#

  • shape_collision_radius 被忽略——MuJoCo 依幾何定義自行計算包圍球半徑(geom_rbound)。
  • 非凸網格取凸包——改變有效碰撞邊界。
  • 速度上限不轉發——joint_velocity_limit 沒有 MuJoCo 對應,被忽略。
  • Kinematic root 的 armature 被覆寫——kinematic articulation root 的 DOF armature 被替換為極大內部值(1e10),使 MuJoCo 視其為被規定的(prescribed)座標;使用者提供的 armature 被靜默丟棄。
  • 碰撞過濾 bitmask 退化——shape_collision_group 經圖著色轉為 contype/conaffinity,最多支援 32 色;超過的 shape 退回 contype=1/conaffinity=1,會與所有 shape 碰撞、繞過原本的過濾。

Kinematic link 與固定 root#

Newton 只允許在 articulation root 上設 is_kinematic=True(其後代仍可為動態)。執行期 SolverMuJoCo 讓 kinematic root 保持使用者規定而非動態積分:回填 Newton 狀態時,kinematic root 的 joint_q/joint_qd 沿用前值而不被 MuJoCo 的積分結果覆寫;施加在 kinematic body 上的力/力矩在 MuJoCo 側被忽略;但它們仍參與接觸,可作為移動或固定的障礙物。轉換時的 root 映射規則:

  • 非固定關節的 kinematic root → 一般 MuJoCo 關節,但 DOF 掛上極大 armature,等效於無限質量座標。
  • 以固定關節連到世界的 root(不論是否 kinematic)→ MuJoCo mocap body,位姿經 mjData.mocap_pos/mocap_quat 驅動;所有軸都鎖住的 D6 root(如一般 USD PhysicsJoint 匯入)同樣處理。
  • 不屬於 articulation 的世界附著 shape → 一般靜態 MuJoCo 幾何,不是 mocap body。

建構求解器之後若編輯了固定 root articulation 的 joint_X_p/joint_X_c,要呼叫 notify_model_changed(ModelFlags.JOINT_PROPERTIES) 把更新後的固定 root 位姿同步進 MuJoCo。

方法說明
step(state_in, state_out, control, contacts, dt)推進一個時間步。
