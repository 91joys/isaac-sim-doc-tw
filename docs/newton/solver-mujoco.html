SolverMuJoCo#

對應官方 newton.solvers.SolverMuJoCo API 頁面(1.4.0 stable)。繼承 SolverBase 與 CouplingInterface。

概述#

SolverMuJoCo 提供介接 MuJoCo 物理引擎的介面,並透過 mujoco_warp 做 GPU 加速。它同時支援 MuJoCo(CPU)與 mujoco_warp(GPU)兩種後端,能高效模擬帶接觸與約束的 articulation 系統——是機器人 RL 訓練的主力求解器。

📦 安裝需求:此求解器需要安裝 mujoco_warp 及其相依套件,安裝方式見 mujoco_warp 儲存庫。另外注意:Newton 模型的 shape_collision_radius 不會被 MuJoCo 使用——MuJoCo 會依幾何定義自行計算包圍球半徑(geom_rbound)。

基本用法#

solver = newton.solvers.SolverMuJoCo(model)

# 模擬迴圈
for i in range(100):
    solver.step(state_in, state_out, control, contacts, dt)
    state_in, state_out = state_out, state_in

預設使用 MuJoCo 自己的碰撞管線(use_mujoco_contacts=True),此時 step() 的 contacts 可傳 None;若設為 False 改用 Newton 的接觸求解,就必須把 Newton 的 contacts 傳進 step()。

參數解析優先序#

求解器選項(例如 impratio)依以下優先序解析:

  1. 建構子引數——若提供,所有 world 使用相同值。
  2. Newton 模型自訂屬性(model.mujoco.<option>)——支援 per-world 值。
  3. MuJoCo 預設值——上述皆未設定時使用。

參數型別預設說明
solverint | str | NoneNewton 預設 "newton"約束求解器類型:"cg" 或 "newton"(或對應 MuJoCo 整數常數)。
integratorint | str | NoneNewton 預設 "implicitfast"積分器類型:"euler"、"rk4" 或 "implicitfast"。

ls_iterationsint | NoneMuJoCo 預設 50line search 迭代次數。
tolerancefloat | NoneMuJoCo 預設 1e-8求解器提前終止容差。
ls_tolerancefloat | NoneMuJoCo 預設 0.01line search 提前終止容差。
jacobianint | str | NoneMuJoCo 預設 "auto"Jacobian 計算方式:"dense"、"sparse" 或 "auto"。
ls_parallelbool | None—已棄用且被忽略(mujoco_warp 3.9.1 移除了平行 line search),傳入會發出 DeprecationWarning。

接觸與碰撞#

參數型別預設說明
use_mujoco_contactsboolTrue使用 MuJoCo 接觸求解;設 False 改用 Newton 接觸求解(此時 contacts 必須經由 step() 傳入)。
disable_contactsboolFalse完全停用 MuJoCo 的接觸計算。
coneint | str | NoneNewton 預設 "pyramidal"接觸摩擦錐類型:"pyramidal" 或 "elliptic"。
impratiofloat | NoneMuJoCo 預設 1.0摩擦/法向約束阻抗比。
nconmaxint | None自動估計每個 world 的接觸點數上限;取使用者值與估計值中較大者。
njmaxint | None自動估計每個 world 的約束數上限;取使用者值與估計值中較大者。
enable_multiccdboolFalse啟用 multi-CCD 接觸生成(每對 geom 最多 4 個接觸點,而非 1 個)。注意:任一 geom 的 margin > 0 時該對永遠只產生單一接觸。
ccd_iterationsint | NoneMuJoCo 預設 35CCD(連續碰撞偵測)最大迭代次數。
ccd_tolerancefloat | NoneMuJoCo 預設 1e-6CCD 容差。
sdf_iterationsint | NoneMuJoCo 預設 10SDF 最大迭代次數。
sdf_initpointsint | NoneMuJoCo 預設 40SDF 初始化點數。

介質力(流體/磁場)#

參數型別預設說明

viscosityfloat | NoneMuJoCo 預設 0.0介質黏度,用於升力與阻力。
windtuple | NoneMuJoCo 預設 (0, 0, 0)風速向量 (x, y, z)。
magnetictuple | NoneMuJoCo 預設 (0, -0.5, 0)全域磁通量向量 (x, y, z)。

後端、world 與匯出#

參數型別預設說明
use_mujoco_cpuboolFalse設 True 改用 MuJoCo-C 的 CPU 後端,而非 mujoco_warp。
separate_worldsbool | Nonenot use_mujoco_cpu設 True 時每個 Newton world 映射為獨立的 MuJoCo world。
update_data_intervalint1每隔多少模擬步從 Newton 狀態更新 MuJoCo Data 物件;0 表示初始化後不再更新。

skip_visual_only_geomsboolTrue純視覺用(不參與碰撞)的幾何不匯出到 MuJoCo spec,避免與使用顯式 <contact> 定義的模型不一致。

除錯:匯出 MJCF 與 MuJoCo 檢視器#

可以把建構子從 newton.Model 產生的 MuJoCo 模型存成 MJCF 檔,再用 MuJoCo 模擬器打開檢查:

solver = newton.solvers.SolverMuJoCo(model, save_to_mjcf="model.xml")

也可以透過 MuJoCo 自己的檢視器觀看模擬,確認 Newton 模型轉成 MuJoCo 模型後長什麼樣子:

import newton

solver = newton.solvers.SolverMuJoCo(model)

for _ in range(num_frames):
    # 推進模擬
    solver.step(state_in, state_out, control, contacts, dt)
    state_in, state_out = state_out, state_in

    solver.render_mujoco_viewer()

⚠️ MuJoCo 檢視器只支援渲染單一 world 的 Newton 模型,除非 use_mujoco_cpu=True 或建構時設 separate_worlds=False。檢視器僅作為除錯工具。render_mujoco_viewer() 可用參數控制左右 UI、接觸點/接觸力顯示與透明幾何(show_left_ui、show_right_ui、show_contact_points、show_contact_forces、show_transparent_geoms);用 close_mujoco_viewer() 關閉。

reset():發散後的正確重置#

solver.reset(state, world_mask=None, flags=None)

把關節狀態重置為模型預設值,並清除 MuJoCo 的內部緩衝。這在 RL 訓練重置環境時特別重要:MuJoCo 會跨 step() 保留求解器狀態(加速度 warm-start、致動器 activation)與外力輸入,模擬發散(如 NaN)之後,即使關節狀態已重置,這些緩衝仍可能「毒害」下一步。因此 reset() 一律per-world 歸零 qacc_warmstart、qfrc_applied、xfrc_applied、act 與 ctrl。

  • world_mask:形狀 (world_count,) 的布林遮罩,選擇要重置哪些 world;None 表示全部。
  • flags:StateFlags 位元遮罩,控制重置哪些量——JOINT_Q 重置 state.joint_q、JOINT_QD 重置 state.joint_qd;None 表示全部重置。內部 MuJoCo 緩衝無論如何都會清除。
  • MuJoCo 是廣義座標求解器,body_q/body_qd 會在下一次 step() 由前向運動學從關節座標導出,所以 BODY_Q/BODY_QD(及粒子)flag 在此無效、會被忽略。
  • 當 update_data_interval != 1 時,每步的 qpos/qvel 同步是關閉或稀疏的,重置後的關節座標會立即推入 qpos/qvel。

控制與致動器(CtrlSource / CtrlType)#

MuJoCo 致動器的控制輸入來源由 SolverMuJoCo.CtrlSource 決定:

枚舉值說明
CtrlSource.JOINT_TARGET (0)從 Newton 的 joint_target_q/joint_target_qd 陣列映射(Newton 標準控制介面)。
CtrlSource.CTRL_DIRECT (1)直接使用 control.mujoco.ctrl(MuJoCo 原生控制)。

JOINT_TARGET 模式下,CtrlType 決定讀哪個目標陣列:

枚舉值說明
CtrlType.POSITION (0)從 joint_target_q 映射,增益取自 joint_target_ke;僅位置控制的致動器同時從 joint_target_kd 同步阻尼,POSITION_VELOCITY 模式的 kd 則由獨立的速度致動器處理。
CtrlType.VELOCITY (1)從 joint_target_qd 映射,增益取自 joint_target_kd。
CtrlType.GENERAL (2)搭配 CTRL_DIRECT 使用,對應 motor/general 致動器。

另有兩個輔助枚舉:EqType 描述 MuJoCo 等式約束類型(CONNECT=像球關節把兩剛體約束在一點、WELD=像固定關節焊接兩剛體、JOINT=把一個純量關節座標約束為另一個的四次多項式,即 mimic);TrnType 描述致動器傳動類型(JOINT、JOINT_IN_PARENT、TENDON、SITE、BODY、SLIDERCRANK)。

關節支援與限制#

  • 支援的關節類型:PRISMATIC、REVOLUTE、BALL、FIXED、FREE、D6;不支援 DISTANCE 與 CABLE。
  • 支援的屬性/控制:joint_armature、joint_friction、joint_effort_limit、joint_limit_ke/joint_limit_kd、joint_target_ke/joint_target_kd、joint_target_mode、joint_f——是所有求解器中支援最完整的。
  • 支援等式約束(CONNECT、WELD、JOINT)與 mimic 約束(僅 REVOLUTE 與 PRISMATIC)。
  • 不支援:joint_velocity_limit、joint_enabled。

完整跨求解器比較見官方 Joint Feature Support,或本站求解器總覽的關節支援矩陣。

其他方法與映射屬性#

方法說明
step(state_in, state_out, control, contacts, dt)推進一個時間步。
update_contacts(contacts, state=None)在 use_mujoco_contacts 模式下,把 MuJoCo 的接觸回填到 Newton 的 Contacts。
get_max_contact_count()回傳 MuJoCo 能產生的剛體接觸數上限。
notify_model_changed(flags)通知求解器模型屬性已變更(執行期改參數時使用)。
import_mujoco()(classmethod)匯入 MuJoCo Warp 相依並快取為類別變數。
register_custom_attributes(builder)(classmethod)在 mujoco 命名空間下宣告要配置到 Model 的自訂屬性;USD 屬性採 MuJoCo USD schema 的 mjc: 前綴(如 "mjc:condim")。
coupling_eval_effective_mass(...) 等CouplingInterface 的多物理耦合掛鉤:評估耦合端點的有效質量/慣量與 MuJoCo 內部套用的重力加速度。

此外,求解器暴露大量 MuJoCo↔Newton 的索引映射屬性(進階/內部用途),例如 mjc_body_to_newton(body 映射)、mjc_dof_to_newton_dof(DOF 映射)、mjc_geom_to_newton_shape 與其逆映射 newton_shape_to_mjc_geom(僅在 use_mujoco_contacts=False 時建立)、mjc_actuator_to_newton_idx(致動器映射)、mjc_eq_to_newton_eq/mjc_eq_to_newton_mimic(等式約束映射)、mjc_tendon_to_newton_tendon(肌腱映射)等,完整列表見官方 API 頁面。

何時選用?#

情境適合嗎
機器人 RL 訓練、操作任務(廣義座標、成熟接觸模型)✅ 主力首選
需要 armature、關節摩擦、力矩上限、等式/mimic 約束✅ 唯一全支援的求解器
與 MJCF/MuJoCo 生態相容、用 MuJoCo 檢視器除錯✅
粒子、布料、軟體❌ 不支援,改用 SolverSemiImplicit/SolverXPBD/SolverVBD
可微分模擬❌ 不支援梯度,改用 SolverSemiImplicit 或 SolverFeatherstone
