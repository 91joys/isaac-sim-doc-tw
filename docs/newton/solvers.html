求解器（newton.solvers）#

對應官方 newton.solvers 章節。求解器負責積分 Newton 模型的動力學：輸入 Model＋State＋Control＋Contacts＋時間步長 dt，經 solver.step() 輸出更新後的 State。

八種求解器一覽#

求解器一句話定位
SolverMuJoCo介接 MuJoCo Warp（GPU 加速版 MuJoCo）——機器人 RL 訓練的主力，預設使用 MuJoCo 自己的碰撞管線
SolverXPBDeXtended Position-Based Dynamics 隱式積分器，剛體＋軟體，最大座標
SolverVBDVertex Block Descent（粒子）＋ Augmented VBD（剛體）隱式求解器，布料強項；實驗性
SolverFeatherstoneFeatherstone CRBA 半隱式（symplectic Euler）廣義座標求解器，articulation 動力學
SolverSemiImplicit半隱式 symplectic Euler，最大座標，簡單通用、可微分入門首選
SolverKamino處理閉鏈（kinematic loops）、欠／過致動、關節極限、硬摩擦接觸與回彈衝擊；實驗性
SolverImplicitMPM隱式 MPM（Material Point Method），顆粒與彈塑性材料（沙、雪、泥）
SolverStyle3DProjective Dynamics 布料專用求解器

怎麼選？#

  • 機器人 RL／操作：SolverMuJoCo（廣義座標、成熟接觸模型、與 MJCF 生態相容）。
  • 需要 Featherstone 廣義座標但不想依賴 MuJoCo：SolverFeatherstone。
  • 剛體＋粒子混合、位置式穩定性：SolverXPBD。
  • 布料：SolverVBD（含自碰撞）或 SolverStyle3D（Projective Dynamics）。
  • 顆粒／彈塑性（沙雪泥）：SolverImplicitMPM。
  • 閉鏈機構、硬接觸：SolverKamino（實驗性）。
  • 可微分入門：SolverSemiImplicit 或 SolverFeatherstone。

關節支援矩陣#

只有 SolverFeatherstone 與 SolverMuJoCo 以廣義座標操作 articulation；最大座標求解器（SemiImplicit、XPBD、Kamino）把關節當成成對剛體約束處理；VBD 以軟約束（AVBD）支援部分關節；Style3D 與 ImplicitMPM 不支援關節。

可微分模擬（DiffSim）#

典型流程：在 wp.Tape() 內執行前向 rollout → 從模擬狀態算純量 loss → tape.backward(loss) 把梯度填入可微分的 state／control／model 陣列。起點是 finalize(requires_grad=True)：

import warp as wp
import newton

@wp.kernel
def loss_kernel(particle_q: wp.array(dtype=wp.vec3), target: wp.vec3, loss: wp.array(dtype=float)):
    delta = particle_q[0] - target
    loss[0] = wp.dot(delta, delta)

builder = newton.ModelBuilder()
builder.add_particle(pos=wp.vec3(0.0, 0.0, 0.0), vel=wp.vec3(1.0, 0.0, 0.0), mass=1.0)

model = builder.finalize(requires_grad=True)
solver = newton.solvers.SolverSemiImplicit(model)

state_in = model.state(requires_grad=True)
state_out = model.state(requires_grad=True)
control = model.control()
loss = wp.zeros(1, dtype=float, requires_grad=True)
target = wp.vec3(0.25, 0.0, 0.0)

tape = wp.Tape()
with tape:
    state_in.clear_forces()
    solver.step(state_in, state_out, control, None, 1.0 / 60.0)
    wp.launch(loss_kernel, dim=1, inputs=[state_out.particle_q, target], outputs=[loss])

tape.backward(loss)
initial_velocity_grad = state_in.particle_qd.grad.numpy()

參考工作流程見官方 DiffSim examples；本站另有完整實戰教學：可微分物理教學——讓梯度學會丟球。

自訂求解器#

所有求解器繼承 SolverBase。Newton 的模組化架構明確支援自製求解器、積分器與數值方法，也支援求解器之間的耦合（多物理／協同模擬）——官方已展示單向耦合範例（機械臂操作布料、四足行走於非剛性地形），雙向與隱式耦合在路線圖上。
