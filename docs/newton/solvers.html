求解器（newton.solvers）#

對應官方 Solvers 總覽（1.4.0）章節。求解器負責積分 Newton 模型的動力學：輸入 Model＋State＋Control＋Contacts＋時間步長 dt，經 solver.step() 輸出更新後的 State。建構子與方法細節見 solver API 參考；症狀導向的診斷與參數調校見官方 Simulation Tuning。

選擇求解器#

MuJoCo 與 Kamino 目前有專屬的官方後端指南，其餘求解器則透過 API 參考與下方比較表格說明：

  • MuJoCo 後端指南 —— 廣義座標剛體模擬與 MuJoCo／MJCF 工作流程（本站中文詳解：SolverMuJoCo）。
  • Kamino 後端指南 —— 帶運動學迴圈與硬摩擦接觸的約束剛體機構；實驗性（本頁摘要）。

對於剛體 articulation 機器人，先決定哪種座標表示適合你的應用：SolverMuJoCo 與 SolverFeatherstone 使用廣義座標（generalized coordinates），而 SolverXPBD、SolverSemiImplicit 與 SolverKamino 使用最大座標（maximal coordinates）。若是可變形體、粒子或可微分模擬，先用下方功能矩陣縮小範圍，再循連結閱讀對應 API 文檔。

怎麼選？#

  • 機器人 RL／操作：SolverMuJoCo（廣義座標、成熟接觸模型、與 MJCF 生態相容）。
  • 需要 Featherstone 廣義座標但不想依賴 MuJoCo：SolverFeatherstone。
  • 剛體＋粒子混合、位置式穩定性：SolverXPBD。
  • 布料：SolverVBD（含自碰撞）或 SolverStyle3D（Projective Dynamics）。
  • 顆粒／彈塑性（沙雪泥）：SolverImplicitMPM。
  • 閉鏈機構、硬接觸：SolverKamino（實驗性）。
  • 可微分入門：SolverSemiImplicit 或 SolverFeatherstone。

接觸材質支援#

ModelBuilder.ShapeConfig 與對應的 Model shape 材質陣列儲存的是與求解器無關的接觸材質資料。下表整理各欄位目前被哪些內建求解器實際使用——外部求解器可能只用其中一部分，或按自己的公式解讀這些欄位：

材質欄位使用它的求解器
mu（摩擦係數）全部：Featherstone、SemiImplicit、XPBD、MuJoCo、VBD、Kamino、Style3D、ImplicitMPM
ke／kd（接觸剛性／阻尼）Featherstone、SemiImplicit、MuJoCo、VBD
kf／ka（摩擦剛性／黏附）Featherstone、SemiImplicit
restitution（回彈係數）XPBD（需 enable_restitution=True）、Kamino
mu_torsional／mu_rolling（扭轉／滾動摩擦）XPBD、MuJoCo
kh（hydroelastic 模量）由 hydroelastic 接觸生成消耗，供使用 Newton 產生接觸的 Featherstone、SemiImplicit，以及 use_mujoco_contacts=False 的 MuJoCo。見官方 Hydroelastic Contacts

關節支援矩陣#

並非每個求解器都支援所有關節類型與屬性。只有 SolverFeatherstone 與 SolverMuJoCo 以廣義（reduced）座標操作 articulation；最大座標求解器（SemiImplicit、XPBD、Kamino）把關節當成成對剛體約束處理，不使用 articulation 的運動學樹結構；VBD 以軟約束（AVBD）支援部分關節；Style3D 與 ImplicitMPM 不支援關節。

SolverKamino 指南摘要#

SolverKamino 以最大座標模擬受約束的剛體多體系統，專為帶有運動學迴圈（kinematic loops）、欠致動／過致動、關節極限、硬摩擦接觸與回彈衝擊的機械組件設計。與其他最大座標求解器不同，Kamino 聚焦於受約束的剛體機械組件，而非粒子或可變形體模擬。

⚠️ 實驗性（BETA 1）：官方目前不建議依賴 SolverKamino——其公開 API、行為、功能支援、效能與實作都可能在未事先通知的情況下變更。只有當「運動學迴圈＋硬接觸約束」是主要需求、且可以接受實驗性求解器時才評估使用。

建構與設定細節見 SolverKamino API 參考；可執行的工作流程見 Kamino examples。

可微分模擬（DiffSim）#

典型流程：在 wp.Tape() 內執行前向 rollout → 從模擬狀態算純量 loss → tape.backward(loss) 把梯度填入可微分的 state／control／model 陣列。起點是 finalize(requires_grad=True)：

import warp as wp
import newton

@wp.kernel
def loss_kernel(particle_q: wp.array(dtype=wp.vec3), target: wp.vec3, loss: wp.array(dtype=float)):
    delta = particle_q[0] - target
    loss[0] = wp.dot(delta, delta)

builder = newton.ModelBuilder()
builder.add_particle(pos=wp.vec3(0.0, 0.0, 0.0), vel=wp.vec3(1.0, 0.0, 0.0), mass=1.0)

model = builder.finalize(requires_grad=True)
solver = newton.solvers.SolverSemiImplicit(model)

state_in = model.state(requires_grad=True)
state_out = model.state(requires_grad=True)
control = model.control()
loss = wp.zeros(1, dtype=float, requires_grad=True)
target = wp.vec3(0.25, 0.0, 0.0)

tape = wp.Tape()
with tape:
    state_in.clear_forces()
    solver.step(state_in, state_out, control, None, 1.0 / 60.0)
    wp.launch(loss_kernel, dim=1, inputs=[state_out.particle_q, target], outputs=[loss])

tape.backward(loss)
initial_velocity_grad = state_in.particle_qd.grad.numpy()

參考工作流程見官方 DiffSim examples；本站另有完整實戰教學：可微分物理教學——讓梯度學會丟球。

自訂求解器#

所有求解器繼承 SolverBase。Newton 的模組化架構明確支援自製求解器、積分器與數值方法，也支援求解器之間的耦合（多物理／協同模擬）——官方已展示單向耦合範例（機械臂操作布料、四足行走於非剛性地形），雙向與隱式耦合在路線圖上。
