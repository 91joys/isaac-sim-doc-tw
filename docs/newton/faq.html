FAQ・整合・遷移#

整理自官方 FAQ、Integrations 與 Migration Guide。技術問答請至 GitHub Discussions。

常見問題#

Warp、warp.sim 與 Newton 的差異？#

  • Warp 是撰寫高效能可微分 GPU kernel 的 Python 框架；Newton 是建構於 Warp 之上的完整物理引擎（高階模擬 API、可互換求解器、資產 I/O）。
  • warp.sim 是 Newton 的前身（NVIDIA 開發的 Warp 內建模組），Warp 1.8 棄用、1.10 移除。遷移見下方指南。

Newton 支援 MuJoCo 模擬嗎？#

是。Newton 以 MuJoCo Warp（Google DeepMind 與 NVIDIA 以 Warp 重新實作的 GPU 加速 MuJoCo）作為關鍵求解器，並可匯入 MJCF／URDF／OpenUSD 資產——在資產與求解器兩個層面都與 MuJoCo 相容。

Newton 會取代 PhysX 嗎？#

不會，兩者目標不同：Newton 主攻機器人學習與可擴充的可微分多物理；PhysX 主攻工業數位孿生的成熟、多平台即時物理 SDK，持續維護更新。PhysX 目前無法作為 Newton 的求解器（但支援方式在評估中）。Isaac Lab 的實驗性 Newton 整合不支援 PhysX，但 Isaac Lab 計畫持續支援 PhysX 後端。

支援多物理耦合／協同模擬嗎？#

是。Newton 明確設計為可擴充多求解器的多物理場景，提供求解器之間耦合的 API。官方已展示單向耦合範例（機械臂操作布料、四足行走於非剛性地形）；雙向耦合與特定求解器間的隱式耦合在路線圖上。也支援自訂求解器、積分器與數值方法。

Newton 可以獨立使用嗎？#

可以，Newton 與其 Python API 可作為獨立模擬框架使用；視覺化以除錯為目的（見視覺化），需要高品質渲染可搭配 Isaac Lab，或匯出時間取樣 USD 到 Omniverse／Isaac Sim。

warp.sim → Newton 遷移指南#

主要 API 名稱對照#

warp.simNewton
FeatherstoneIntegrator／SemiImplicitIntegrator／VBDIntegrator／XPBDIntegratorSolverFeatherstone／SolverSemiImplicit／SolverVBD／SolverXPBD
integrator.simulate(model, state0, state1, dt, None)solver.step(state0, state1, control, None, dt)
parse_urdf()／parse_mjcf()／parse_usd()ModelBuilder.add_urdf()／add_mjcf()／add_usd()（add_usd 直接接受 URL）
匯入器的關節／接觸參數引數改設於 builder.default_joint_cfg／builder.default_shape_cfg
eval_fk(model, q, qd, mask, state)eval_fk(model, q, qd, state, mask=None)（mask 改為可選）
Model.ground（特殊地面處理）已移除——改用 builder.add_ground_plane()
Model.shape_geo.*／Model.shape_materials.* 結構攤平為 Model.shape_type、shape_scale、shape_material_ke 等逐形狀陣列

重要行為變更#

  • spatial_vector 順序：Newton 公開陣列改用 (linear, angular) 順序（如 State.body_qd 存 (lin_vel, ang_vel)）；warp.sim 沿用 Warp 原生的 (ang, lin)。剛體線速度是質心的世界座標速度，需要 body 原點速度時用 v_origin = v_com − ω × r_com_world 換算。
  • Control 職責拆分：joint_target_q／joint_target_qd 存位置／速度目標，joint_act 存前饋致動輸入，joint_f 存廣義力——joint_act 不再是目標陣列。JointMode 改為 JointTargetMode（EFFORT、POSITION_VELOCITY）。
  • joint-target 版面：Newton 1.3 新增 opt-in 旗標 newton.use_coord_layout_targets，讓 joint_target_q 改用與 joint_q 對齊的座標版面（free joint 7 座標 vs 6 DOF 的差異）。
  • MJCF 密度：Newton 的 MJCF 匯入器對所有形狀類型都採用 geom_density（warp.sim 只對球與盒），模擬結果可能不同、需重調參數。
  • 匯入器上軸：MJCF／URDF 匯入器 up_axis 預設 +Z；所有匯入器現在會旋轉資產以符合 builder 的 up_axis（而非覆寫 builder 設定）。

完整逐項對照（Renderers、Collisions 等）見官方 Migration Guide。
