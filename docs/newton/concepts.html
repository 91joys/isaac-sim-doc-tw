核心概念#

整理自官方 Concepts 章節：Worlds、Articulations、Mass and Inertia、Sites 與 Conventions。

多世界（Worlds）#

Newton 允許在單一 Model 內建立多個互相獨立的模擬「世界」——這是 RL 大規模平行訓練的基礎。每個世界以索引將粒子、剛體、形狀、關節、articulation 分組：

  • 世界索引 0, 1, 2, …：世界專屬實體，不與其他世界碰撞。
  • 全域索引 -1：全域實體（如地面），與所有世界碰撞。全域實體使用世界 0 的重力。

三種建立方式：

import newton

# 方式一：begin_world / end_world 手動分組
builder = newton.ModelBuilder()
builder.add_ground_plane()          # 全域（world -1）
builder.begin_world()               # world 0
b0 = builder.add_body(mass=1.0)
builder.add_shape_sphere(body=b0, radius=0.1)
builder.end_world()

# 方式二：add_world —— 把預先建好的子 builder 當成一個世界加入
arm = newton.ModelBuilder()
# ... 在 arm 中建構手臂 ...
scene = newton.ModelBuilder()
scene.add_ground_plane()
scene.add_world(arm)
scene.add_world(arm)

# 方式三：replicate —— 一次複製 N 個世界（RL 最常用）
scene2 = newton.ModelBuilder()
scene2.add_ground_plane()
scene2.replicate(arm, world_count=1024)
model = scene2.finalize()

兩個實務提醒：
① replicate() 的 spacing 會把物體移離原點、降低數值穩定性——物理上保持 spacing=(0,0,0)，視覺分隔改用 viewer.set_world_offsets()。
② 若子 builder 含網格，先呼叫 approximate_meshes() 再 replicate()，讓所有世界共用同一份簡化網格。

每個世界可有自己的重力（例如地球與月球並存）：

model.set_gravity((0.0, 0.0, -9.81), world=0)  # 地球
model.set_gravity((0.0, 0.0, -1.62), world=1)  # 月球

模型並提供 body_world、shape_world 等分組陣列與 *_world_start 起始索引陣列，方便在自訂 Warp kernel 中以「(世界, 實體)」2D 執行緒網格平行處理。異質世界（不同內容）支援仍屬實驗性。

Articulation 與座標表示#

Articulation 是由關節連接的剛體集合（機器人）。描述其組態有兩種參數化：

廣義座標（generalized / reduced）最大座標（maximal）
狀態儲存State.joint_q（關節位置）、State.joint_qd（關節速度）State.body_q（每體 7 參數：位置＋XYZW 四元數）、State.body_qd（6 參數：線速度＋角速度）
例：雙擺2 個關節角＋2 個關節速度2 個剛體位姿＋速度
浮動基座廣義座標另含基座位姿（位置＋四元數）——
採用的求解器SolverMuJoCo、SolverFeatherstoneSolverXPBD、SolverSemiImplicit、SolverVBD

兩種表示之間用運動學互換：newton.eval_fk()（廣義 → 最大）與 newton.eval_ik()（最大 → 廣義）。注意：碰撞偵測 Model.collide() 需要 state 中的最大座標是最新的——這就是快速入門範例在迴圈前呼叫 eval_fk() 的原因。

builder = newton.ModelBuilder()
body = builder.add_link()
builder.add_shape_box(body)   # 加形狀以產生慣量
joint = builder.add_joint_revolute(parent=-1, child=body, axis=wp.vec3(0.0, 0.0, 1.0))
builder.add_articulation([joint])
builder.joint_q[-1] = 0.5     # 初始關節角
builder.joint_qd[-1] = 10.0   # 初始關節速度

model = builder.finalize()
state = model.state()
newton.eval_fk(model, model.joint_q, model.joint_qd, state)  # 同步 body 位姿

關節類型#

Newton 支援 PRISMATIC（滑動）、REVOLUTE（旋轉）、BALL（球）、FIXED（固定）、FREE（自由）、DISTANCE（距離）、D6（六自由度）與 CABLE（纜線，VBD 專用）——各求解器的支援程度見求解器・關節支援矩陣。閉鏈（loop closure）機制：迴路閉合關節照常以 add_joint_* 建立，但不放入 articulation，讓 articulation 圖保持樹狀。

質量與慣量#

每個動態剛體都需要正質量與物理上合理的慣量張量。三種指定方式：

  1. 直接指定：add_link(mass=…, inertia=…)（質量 kg、對質心的 3×3 慣量張量）。
  2. 密度推算：由碰撞形狀的 ShapeConfig 密度自動計算（官方建議優先——與碰撞幾何一致、免手動簿記）。
  3. 檔案匯入：USD／MJCF／URDF 中的質量屬性自動解析。

最佳實務：

  • 沒有帶密度形狀的動態剛體，務必在 add_link() 明確給 mass 與 inertia。
  • 要做預定運動（prescribed motion）用 is_kinematic=True，不要靠零質量讓物體不動。
  • 手動指定的質量屬性用 lock_inertia=True 保護，避免被後續加入的形狀改寫（MJCF 匯入器遇到 <inertial> 即用此機制）。
  • 開發期設 validate_inertia_detailed=True，finalize() 時會對每個被修正的剛體提出警告。

Sites（抽象標記點）#

Site 是附著在剛體（或世界）上的無碰撞參考框架，用途：感測器安裝點（IMU）、相對變換的參考框架、MuJoCo 互通。

body = builder.add_body(xform=wp.transform((0, 0, 1), wp.quat_identity()))
builder.add_site(body, label="imu_0")                      # 體原點
builder.add_site(body, xform=wp.transform(...), label="tip")  # 帶偏移

MJCF／USD 匯入時可用參數控制只載入碰撞形狀與 sites。搭配感測器的用法見感測器與致動器。

慣例（Conventions）#

項目Newton 慣例
上軸Z-up（預設）、右手座標系；可用 ModelBuilder(up_axis=newton.Axis.Y) 改為 Y-up，重力向量自動對齊所選上軸
四元數XYZW 儲存順序（向量部在前、純量 w 在後，同 Warp）——與 MuJoCo 的 WXYZ 不同，跨引擎交換資料時務必注意
剛體速度body_qd 線性部分＝質心的世界座標速度；FREE 關節的 joint_qd 為 parent 框架下的 child-COM twist
顏色表面顏色預設視為 display／sRGB 值
各引擎上軸對照Newton＝Z、MuJoCo＝Z、USD＝Y（可設 Z）、Isaac Lab/Sim＝Z；Newton 本身座標系無關（agnostic），任何選擇都能運作

空間 twist／wrench 的參考點慣例、與 Drake／MuJoCo／Isaac 的詳細對照，見官方 Conventions 全文。
