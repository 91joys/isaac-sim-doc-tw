SolverSemiImplicit#

對應官方 newton.solvers.SolverSemiImplicit API 頁面（1.4.0 stable）。繼承 SolverBase 與 CouplingInterface。

概述#

SolverSemiImplicit 是使用 symplectic Euler（辛歐拉）的半隱式積分器。建好 Model 與 State 物件後,就能用它把模擬狀態往前推進一個時間步。

幾個關鍵性質:

  • 變分積分器(variational integrator),能量守恆——長時間模擬不會像顯式 Euler 那樣持續注入能量。
  • 非無條件穩定——時間步 dt 必須小到能承受模型中的剛度(stiffness)與阻尼力;剛度越高,dt 要越小(常見做法是每個渲染影格內跑多個 substep)。
  • 最大座標(maximal coordinates)——關節以成對剛體約束(attachment spring)實現,而非廣義座標。
  • 簡單、通用、可微分入門首選——支援剛體、粒子、布料(無自碰撞)、軟體,官方 diffsim 範例多以它示範。

數學背景見 Semi-implicit Euler method(Wikipedia)。

基本用法#

solver = newton.solvers.SolverSemiImplicit(model)

# 模擬迴圈
for i in range(100):
    solver.step(state_in, state_out, control, contacts, dt)
    state_in, state_out = state_out, state_in

建構子參數#

SolverSemiImplicit(
    model,
    *,
    angular_damping=0.05,
    friction_smoothing=1.0,
    joint_attach_ke=1.0e4,
    joint_attach_kd=1.0e2,
    enable_tri_contact=True,
    deterministic=None,
)
參數型別預設值說明
modelModel—要模擬的模型。
angular_dampingfloat0.05剛體積分使用的角速度阻尼係數。
friction_smoothingfloat1.0摩擦速度正規化使用的 Huber norm delta(見 warp.norm_huber()),用來平滑摩擦力在零速度附近的不連續。
joint_attach_kefloat1.0e4關節附著彈簧剛度。最大座標求解器用「彈簧」把成對剛體綁在一起實現關節——太軟關節會分離,太硬則需要更小的 dt。
joint_attach_kdfloat1.0e2關節附著彈簧阻尼。
enable_tri_contactboolTrue是否啟用三角形接觸(布料/網格接觸)。
deterministicwarp.DeterministicMode | NoneNone對此求解器會發出 atomic 操作的 kernel 模組選擇性啟用確定性模式;None 表示繼承目前的 wp.config.deterministic 設定。

step() 方法#

solver.step(state_in, state_out, control, contacts, dt)

以給定的控制輸入把模型向前模擬一個時間步。

參數型別說明
state_inState輸入狀態。
state_outState輸出狀態(寫入結果)。
controlControl | None控制輸入;None 表示使用 Model 上的控制值。
contactsContacts | None接觸資訊(由 model.collide() 產生);None 表示不使用接觸。
dtfloat時間步長(通常以秒為單位)。

⚠️ 可微分模擬注意:處理粒子間接觸的 eval_particle_contact kernel 可能會破壞含粒子碰撞模擬的梯度計算。若要停用,請在呼叫 step() 之前把 newton.Model.particle_grid 設為 None。

關節支援與限制#

  • 支援的關節類型:PRISMATIC、REVOLUTE、BALL、FIXED、FREE、DISTANCE(視為 FREE)、D6;不支援 CABLE。
  • 支援的屬性/控制:joint_enabled、joint_limit_ke/joint_limit_kd、joint_target_ke/joint_target_kd(PD 目標)、joint_f(前饋力)。
  • BALL 關節不強制關節極限與目標。
  • 不支援:joint_armature、joint_friction、joint_effort_limit、joint_velocity_limit、joint_target_mode。
  • 不支援等式約束(equality constraints)與 mimic 約束。

完整跨求解器比較見官方 Joint Feature Support,或本站求解器總覽的關節支援矩陣。

何時選用?#

情境適合嗎
可微分模擬(diffsim)入門、梯度式最佳化✅ 首選之一(另見 SolverFeatherstone)
剛體＋粒子＋布料＋軟體混合場景、簡單通用✅ 支援面最廣的半隱式求解器
高剛度材料、大時間步⚠️ 非無條件穩定,需縮小 dt 或增加 substep
精確 articulation 動力學(機器人)❌ 建議改用 SolverFeatherstone(廣義座標)或 SolverMuJoCo
需要關節 armature/摩擦/力矩上限❌ 不支援,改用 SolverMuJoCo

可微分工作流程完整範例見可微分物理教學與求解器總覽的 DiffSim 章節。
