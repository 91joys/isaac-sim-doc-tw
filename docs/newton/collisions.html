碰撞與接觸（Collisions and Contacts）#

對應官方 Collisions and Contacts 章節——原文非常詳盡，本頁整理核心觀念與最常用的模式。

碰撞管線總覽#

Newton 的碰撞偵測分兩階段：

  1. Broad phase（粗篩）：以 AABB 快速剔除包圍盒不重疊的形狀對。
  2. Narrow phase（細算）：對存活的形狀對計算實際接觸幾何。演算法依形狀表示而定：
    • 凸包與基本體（球、盒、膠囊、圓柱、圓錐、橢球、凸網格）→ MPR/GJK，直接產生少量接觸點。
    • 無預計算 SDF 的三角網格 → Warp BVH 即時距離查詢＋可選接觸縮減；開箱即用但高面數網格較慢，且不支援 hydroelastic。
    • 預計算 SDF（mesh.build_sdf(...) 或基本體的 ShapeConfig SDF 參數）→ O(1) 距離查詢，支援距離查詢與 hydroelastic 兩種接觸生成。

效能提示：SDF 或 hydroelastic 等昂貴碰撞的場景，可以「每 frame 呼叫一次 collide()」而非每個 substep 都呼叫，通常能顯著提速。

Broad Phase 模式#

模式說明
EXPLICIT（預設）使用預先計算的形狀對——結合靜態配對效率與進階接觸演算法
NxN全配對 AABB，O(N²)，適合小場景（<100 形狀）
SAPSweep-and-prune，O(N log N)，適合較大且空間相干的場景
 
from newton import CollisionPipeline

pipeline = CollisionPipeline(model)                      # 預設 EXPLICIT
pipeline = CollisionPipeline(model, broad_phase="nxn")   # 小場景
pipeline = CollisionPipeline(model, broad_phase="sap")   # 大場景

contacts = pipeline.contacts()
pipeline.collide(state, contacts)

支援的形狀類型（GeoType）#

類型說明
PLANE無限平面（地面）
HFIELD高度場地形（2D 高程網格）
SPHERE／CAPSULE／BOX／CYLINDER／CONE／ELLIPSOID基本體
MESH任意三角網格（含非凸）
CONVEX_MESH凸包網格

SDF 是「碰撞資料」而非獨立形狀類型：網格用 mesh.build_sdf(...) 建立後傳入 add_shape_mesh(...)；基本體經 ShapeConfig 的 SDF 參數產生。
非凸網格建議用 builder.approximate_meshes(method="convex_hull") 以凸包近似（預設方法），或針對個別形狀保留視覺幾何。

碰撞過濾#

層次一：世界索引#

不同世界的形狀永不碰撞；世界 -1 的全域形狀（如地面）與所有世界碰撞。這是 RL 多環境隔離的機制，見核心概念・多世界。

層次二：碰撞群組（同世界內）#

群組規則行為
0停用碰撞
正數（1, 2, …）只與同群組或任何負群組碰撞
負數（-1, -2, …）與所有正、負群組碰撞，但不與相同負群組碰撞
 
# 群組 1：只與群組 1 及負群組碰撞
builder.add_shape_sphere(body1, radius=0.5, cfg=builder.ShapeConfig(collision_group=1))

# 群組 -1：與所有東西碰撞（除了其他 -1）
builder.add_shape_sphere(body2, radius=0.5, cfg=builder.ShapeConfig(collision_group=-1))

層次三：自碰撞與逐對排除#

  • Articulation 內自碰撞：載入時 add_usd("robot.usda", enable_self_collisions=True)（MJCF 同）。預設父子相鄰體不碰撞（collision_filter_parent=True）。
  • 粒子 vs 形狀：ShapeConfig(has_shape_collision=…, has_particle_collision=…) 細部控制。
  • 特定形狀對排除：builder.add_shape_collision_filter_pair(a, b)；USD 匯入時自動讀取 UsdPhysics 的 CollisionGroup 與 physics:filteredPairs。

接觸資料（Contacts）#

Contacts 儲存碰撞偵測結果，由 solver.step() 消耗。剛體接觸的主要欄位：

屬性說明
rigid_contact_count目前有效的剛體接觸數（純量）
rigid_contact_shape0/1碰撞形狀索引
rigid_contact_point0/1各形狀上的接觸點（body 框架）
rigid_contact_normal接觸法線，由形狀 0 指向形狀 1（世界框架）
rigid_contact_match_index跨 frame 接觸匹配結果：≥0 匹配舊索引、-1 新接觸、-2 已斷開（需啟用 contact_matching）
force（擴充屬性）每接觸空間力，供 SensorContact 使用，由 solver.update_contacts() 填入
 
contacts = model.contacts()
model.collide(state, contacts)

n = contacts.rigid_contact_count.numpy()[0]
points0 = contacts.rigid_contact_point0.numpy()[:n]
normals = contacts.rigid_contact_normal.numpy()[:n]
shape0  = contacts.rigid_contact_shape0.numpy()[:n]

另有軟接觸（粒子—形狀）欄位 soft_contact_*。

Hydroelastic 接觸#

選擇性啟用的功能：以 SDF 產生「接觸面」而非接觸點，力分布更真實連續——特別適合機器人操作（manipulation）情境：非凸形狀互動更穩定、大接觸面力分布更佳、面對面接觸的摩擦更真實。

需求：配對雙方都設 is_hydroelastic=True、雙方都有 SDF 資料（網格 build_sdf；基本體用 sdf_max_resolution／sdf_target_voxel_size）、僅支援體積形狀（不含平面、高度場、非水密網格）。預設關閉時，SDF 形狀使用高效的硬 SDF 點接觸。

常用模式速覽#

  • 靜態地面：builder.add_ground_plane()；或把形狀掛在 body=-1 上手動建立靜態幾何。
  • 形狀預設值：在加形狀前設定 builder.default_shape_cfg（摩擦、restitution、密度等）。
  • 接觸縮減：SDF／hydroelastic 路徑自動適用；MPR/GJK 凸對本來就只產生少量接觸點。
  • 確定性：接觸排序可設為確定性（deterministic contact ordering），利於重現。
  • 可微分接觸：SDF 距離可反向傳播，見官方 Differentiable Contacts 一節。
