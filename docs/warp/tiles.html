Tile 程式設計#

整理自官方 Tiles 章節。Tile 原語自 Warp 1.5.0 引入，概念上類似 OpenAI Triton 的 block 程式設計模型。

Tile 是什麼#

一般 Warp kernel 是 SIMT 模型——每條執行緒獨立處理自己的元素。Tile 原語讓一個 block 的執行緒協作處理一塊資料（tile），得以利用高效記憶體存取、共享記憶體與 Tensor Core 等硬體單元。線性代數 tile 運算（tile_matmul、tile_fft、tile_cholesky）背後由 cuBLASDx／cuFFTDx（MathDx）驅動——需要以 CUDA 12.6.3+ 建置的 Warp。

tile 是最多 4 維的資料陣列，元素可以是純量、向量、矩陣或 struct。

執行模型與 launch_tiled#

TILE_SIZE = wp.constant(256)
TILE_THREADS = 64

@wp.kernel
def compute(a: wp.array2d[float], b: wp.array2d[float]):
    i = wp.tid()                        # block 索引
    t = wp.tile_load(a[i], TILE_SIZE)   # 整個 block 協作載入一列
    s = wp.tile_sum(t)                  # 協作歸約:單元素 tile
    wp.tile_store(b[i], s)

wp.launch_tiled(compute, dim=[a.shape[0]], inputs=[a, b], block_dim=TILE_THREADS)

wp.launch_tiled() 為 launch grid 的每個元素分配 block_dim 條執行緒——kernel 內 wp.tid() 回傳的是 block 索引，不是執行緒索引。它等價於 wp.launch() 多加一個 trailing 維度，但更方便。2D tile 直接從 2D 陣列載入：

t = wp.tile_load(array, shape=(TILE_M, TILE_N), offset=(i*TILE_M, j*TILE_N))

Tile 儲存：register 與 shared#

Register tile（預設）Shared memory tile
存放位置分散在整個 block 的暫存器（如 block 64 執行緒存 256 元素 tile → 每緒 4 個）block 共享記憶體
速度最快快，支援任意隨機存取
限制執行緒無法有效讀其他執行緒持有的元素——操作以 map／reduce／reshape 高階形式表達容量有限——tile 太大會編譯或 launch 失敗

Warp 自動決定最佳位置，需要整塊隨機存取的操作（如 tile_matmul、元素賦值 tile[row, col] = val）會自動遷移到 shared memory 並插入執行緒同步。shared tile 配置保證 16-byte 對齊；載入時 Warp 以三層機制極大化頻寬（向量化 float4 → coalesced byte-copy → 純量 fallback），確定條件滿足時可用 tile_load(..., aligned=True) 跳過執行期對齊檢查。

範例：通用矩陣乘法（GEMM）#

import numpy as np
import warp as wp

TILE_M = wp.constant(8)
TILE_N = wp.constant(4)
TILE_K = wp.constant(8)
TILE_THREADS = 64

@wp.kernel
def tile_gemm(A: wp.array2d[float], B: wp.array2d[float], C: wp.array2d[float]):
    i, j = wp.tid()   # 輸出 tile 索引

    sum = wp.tile_zeros(shape=(TILE_M, TILE_N), dtype=wp.float32)
    K = A.shape[1]

    for k in range(int(K / TILE_K)):
        a = wp.tile_load(A, shape=(TILE_M, TILE_K), offset=(i*TILE_M, k*TILE_K))
        b = wp.tile_load(B, shape=(TILE_K, TILE_N), offset=(k*TILE_K, j*TILE_N))
        wp.tile_matmul(a, b, sum)   # sum += a @ b（累加!）

    wp.tile_store(C, sum, offset=(i*TILE_M, j*TILE_N))

M, K, N = TILE_M * 7, TILE_K * 6, TILE_N * 5
A_wp = wp.array(np.random.rand(M, K).astype(np.float32))
B_wp = wp.array(np.random.rand(K, N).astype(np.float32))
C_wp = wp.zeros((M, N), dtype=wp.float32)

wp.launch_tiled(tile_gemm, dim=(int(M/TILE_M), int(N/TILE_N)),
                inputs=[A_wp, B_wp, C_wp], block_dim=TILE_THREADS)

Tile 運算總覽#

類別代表函式
建構tile_zeros、tile_ones、tile_full、tile_arange、tile_randf、tile／untile（SIMT↔tile）、tile_view、tile_broadcast、tile_reshape、tile_astype
載入／儲存tile_load、tile_store、tile_atomic_add、tile_assign、tile_extract、tile_scatter_add、indexed 變體
Map／歸約tile_map、tile_reduce、tile_sum/min/max、tile_argmin/argmax、tile_sort、tile_scan_inclusive/exclusive
逐元素算術+ - *、+=／-=、一元負號——全 block 協作執行
線性代數（MathDx）tile_matmul、tile_fft／tile_ifft、tile_cholesky／tile_cholesky_solve、三角求解等

tile 也能在 @wp.func 間傳遞（簽名建議用 typing.Any）;Tile Stack 與 register tile 軟體管線化等進階技巧見官方原文。tile 運算支援自動微分，與 wp.Tape 正常搭配。

Tile 與 SIMT 混用#

同一個 kernel 內可自由切換兩種模式：wp.tile() 把每條執行緒的值聚成 tile（SIMT → tile），wp.untile() 反向拆回（tile → SIMT）。常見模式——各執行緒獨立算出自己的值，再協作歸約；官方範例顯示用 tile 做全陣列歸約比純 wp.atomic_add() 快約 4 倍。

Tile 常見問題精選#

  • 協作分支規則：tile 運算是全 block 協作的——不可放在發散分支（依執行緒 index 而異的 if）內，否則死鎖或錯誤結果。
  • tile 形狀必須是編譯期常數：用 wp.constant() 或 Python 常數，不能用執行期變數。
  • 垃圾值／NaN 輸出:最常見原因是把未初始化的 tile 傳給 tile_matmul()——它是累加語意，輸出 tile 要先 tile_zeros()。
  • Shared memory 溢位：非 MathDx tile 顯示超出上限警告＋CUDA_ERROR_LAUNCH_OUT_OF_RESOURCES;MathDx 操作則在 LTO 編譯時失敗。縮小 tile 或降低 block_dim。
  • Failed to compile LTO:MathDx 執行期生成連結時物件失敗——常見於 CUDA 版本過舊（<12.6.3）或 shared memory 需求過大;檢查 LTO Cache 與建置設定。
  • CPU 與 GPU 行為差異：CPU 上 block_dim 固定為 1，tile 操作退化為序列執行——功能等價但無協作加速。
  • 越界處理：tile_load 對出界元素自動補零，tile_store 自動裁切——部分 tile（陣列尺寸非 tile 整數倍）可安全處理。
