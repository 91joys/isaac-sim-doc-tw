執行期核心功能#

整理自官方 Runtime 章節——Warp 執行期 API 的完整參考：陣列、資料型別、CUDA Graph 與空間計算原語。

陣列進階#

陣列是 Warp 最基本的記憶體抽象。建構方式：

wp.empty(shape=1024, dtype=wp.vec3, device="cpu")   # 未初始化
wp.zeros(shape=1024, dtype=float, device="cuda")     # 零初始化
wp.full(shape=1024, value=10, dtype=int, device="cuda")

r = np.random.rand(1024)
a = wp.array(r, dtype=float, device="cpu")               # 複製到 Warp 自有記憶體
a = wp.array(r, dtype=float, copy=False, device="cpu")   # 零拷貝包裝 NumPy 資料
a = wp.array(r, dtype=float, device="cuda")              # 複製上 GPU
  • 多維外部資料必須明確給 dtype（如 (1024, 3) 的 NumPy 陣列 → dtype=wp.vec3）。
  • 可從任何實作 __cuda_array_interface__ 的物件（CuPy、PyTorch……）零拷貝建構——但該協定不含裝置資訊，務必明確指定 device。
  • 搬移與複製：array.to("cuda") 跨裝置搬移;wp.copy(dest, src) 跨記憶體空間複製;另有 wp.zeros_like、wp.ones_like、wp.full_like、wp.empty_like、wp.clone。

多維陣列與切片#

wp.zeros(shape=(1024, 16), dtype=float, device="cuda")

@wp.kernel
def test(input: wp.array2d[float]):
    i, j = wp.tid()
    value = input[i, j]   # 取單一元素
    row = input[i]        # 取 1D 切片（一列）

切片可串接（array[i][j][k]）並可傳給 @wp.func（函式需宣告對應維度）；目前僅支援單索引切片。

索引陣列（Indexed Arrays）#

用整數索引陣列對 wp.array 做進階索引會得到 wp.indexedarray——現有陣列的輕量 view，讓 kernel 不需複製就能操作任意子集：

arr = wp.array((1.23, 2.34, 3.45, 4.56, 5.67, 6.78), device="cuda")
idx = wp.array((1, 3, 5), dtype=wp.int32, device="cuda")
sub = arr[idx]        # indexedarray，只看奇數位置的元素

結構化陣列與 kernel 內陣列#

  • struct 陣列：@wp.struct 型別可作為 dtype;CPU 上的 struct 陣列可用 NumPy structured array 零拷貝檢視與修改,也能先在 NumPy 準備好再轉成 Warp 陣列（支援巢狀 struct 與向量成員）。
  • kernel 內建立陣列（Local Arrays）僅限兩種形式:wp.array(ptr=..., shape=..., dtype=...) 把現有記憶體重新詮釋成別的形狀,或 wp.zeros(shape=N, dtype=float) 配置編譯期常數大小的暫存陣列（支援索引、切片與 wp.lower_bound 等操作）。

資料型別#

純量#

bool、int8/16/32/64、uint8/16/32/64、float16、bfloat16、float32、float64；float 與 int 是 float32／int32 的別名。

bfloat16 注意：NumPy 原生不支援 bfloat16，Warp 內部以 uint16 儲存——沒裝 ml-dtypes 時 .numpy() 與 print() 會顯示原始整數值（pip install ml-dtypes 即可正常顯示）。wp.to_torch()／wp.to_jax() 不受影響，永遠產生正確的 bfloat16 張量。

向量・矩陣・四元數・變換#

類別內建型別自訂長度／精度
向量vec2/3/4＋型別後綴（vec3d＝double、vec3h＝half、vec3i＝int……）、6D spatial_vectorwp.types.vector(length=5, dtype=wp.float64)
矩陣mat22/33/44＋後綴、spatial_matrix（6×6）wp.types.matrix(shape=(3,2), dtype=wp.float16)
四元數quat（quatf/quatd/quath）——XYZW 順序（向量部在前）—
變換transform（7D：位置 p ＋ 四元數 q）wp.types.transformation(dtype=wp.float64)
 
@wp.kernel
def compute():
    # 由平移與旋轉建構變換
    t = wp.transform(
            wp.vec3(1.0, 2.0, 3.0),
            wp.quat_from_axis_angle(wp.vec3(0.0, 1.0, 0.0), wp.radians(30.0)))

    p = wp.transform_point(t, wp.vec3(10.0, 0.5, 1.0))   # 變換點
    v = wp.transform_vector(t, wp.vec3(10.0, 0.5, 1.0))  # 變換向量（忽略平移）

向量／矩陣／四元數／struct 都支援索引、切片、unpacking 與型別轉換；struct 還能定義運算子多載（__add__ 等）。完整函式清單見官方 Built-Ins。

常數#

kernel 可以讀取外部定義的 Python 變數，視為編譯期常數。wp.constant() 已非必要，但能提供型別檢查並標明意圖：

TYPE_SPHERE = wp.constant(0)
TYPE_CUBE = wp.constant(1)

@wp.kernel
def collide(geometry: wp.array[int]):
    t = geometry[wp.tid()]
    if t == TYPE_SPHERE:
        print("sphere")
    elif t == TYPE_CUBE:
        print("cube")

內建數學常數：wp.PI、wp.HALF_PI、wp.TAU、wp.E、wp.PHI、wp.LN2、wp.INF、wp.NAN 等（皆有小寫版本）。wp.NAN 僅限浮點型別，比較遵循 IEEE 754（NaN ≠ NaN），用 wp.isnan() 檢測。被參照的 Python 變數若會變動，行為細節見程式碼生成。

CUDA Graph#

從 Python 發射 kernel 的 CPU 開銷遠大於原生程式。CUDA Graph 可以錄下一大批 kernel launch，之後以極低的 CPU 開銷重播——RL 訓練等重複執行相同計算的場景必備：

# 建議用 ScopedCapture（確保例外時也會結束擷取）
with wp.ScopedCapture(device="cuda") as capture:
    for i in range(100):
        wp.launch(kernel=compute1, inputs=[a, b], device="cuda")

# 重播（可多次，幾乎沒有 Python 開銷）
wp.capture_launch(capture.graph)
  • 也可用 wp.capture_begin()／wp.capture_end()（務必搭配 try-finally，否則例外會讓 stream 卡在擷取狀態）。
  • 只有 launch 呼叫會被錄進 graph——kernel 之外的 Python 邏輯不會。建 graph 有開銷，重複使用才划算。
  • 與會在擷取期間做懶初始化 CUDA 呼叫的函式庫（如 PyTorch）共用時，用 capture_mode=wp.CaptureMode.RELAXED。
  • 條件執行：wp.capture_if()（依 GPU 上的條件值走 on_true／on_false 分支）與 wp.capture_while()（GPU 端 while 迴圈）讓 graph 內含動態控制流，不需回到 CPU 判斷。
  • CPU Graph：同一套 API 也支援在 CPU 裝置上錄製;錄好的 graph 還能序列化存檔，之後從 Python 或獨立 C++ 程式重播（部署場景不需 Python）。

空間計算原語#

Warp 內建幾何資料結構，全部以 uint64 的 id 傳入 kernel 使用：

Mesh（三角網格）#

mesh = wp.Mesh(points=points, indices=indices)   # 可另附 velocities

@wp.kernel
def closest_point_example(mesh: wp.uint64, query_points: wp.array[wp.vec3], closest: wp.array[wp.vec3]):
    tid = wp.tid()
    query = wp.mesh_query_point(mesh, query_points[tid], max_dist=10.0)
    if query.result:
        closest[tid] = wp.mesh_eval_position(mesh, query.face, query.u, query.v)

wp.launch(closest_point_example, dim=query_points.shape,
          inputs=[mesh.id, query_points], outputs=[closest], device="cuda:0")
  • 幾何查詢：wp.mesh_query_point()（最近點，回傳命中面與重心座標）、wp.mesh_query_ray()（光線投射，回傳距離、法線、面索引）等。
  • 頂點位置可在執行期直接改 points buffer，之後呼叫 mesh.refit() 重建 BVH;拓撲（indices）不可更新，需重建 Mesh。

HashGrid(空間鄰居查詢)#

DEM、SPH 等粒子法的鄰居搜尋加速結構：

grid = wp.HashGrid(dim_x=8, dim_y=8, dim_z=8, device="cuda:0")
grid.build(points=p, radius=r)

@wp.kernel
def count_neighbors(grid: wp.uint64, points: wp.array[wp.vec3],
                    radius: float, neighbor_count: wp.array[int]):
    tid = wp.tid()
    point = points[tid]
    count = int(0)
    for index in wp.hash_grid_query(grid, point, radius):
        if index != tid and wp.length(points[index] - point) <= radius:
            count += 1
    neighbor_count[tid] = count

查詢回傳的是「半徑內格子裡的所有點」，可能包含半徑外的點——使用者要自己檢查距離（kernel 通常本來就要算距離，避免重複計算）。

Volume・Texture・BVH#

  • Volume（NanoVDB 稀疏體積）：大範圍網格資料（SDF、流場）的稀疏表示。可從 .nvdb 檔（load_from_nvdb()）、記憶體緩衝或稠密 NumPy 陣列建立;kernel 內可直接存取 voxel 或用最近點／三線性內插從世界／局部空間取樣。
  • Texture：1D／2D／3D 紋理物件（RGBA 或單通道），提供硬體內插取樣。
  • BVH:對一組 AABB 建立包圍體階層,kernel 內用 wp.bvh_query_ray()／wp.bvh_query_aabb() 遍歷。
  • Marching Cubes：wp.MarchingCubes 從 3D 純量場萃取等值面三角網格；也可自訂實作。

物件生命週期陷阱：Mesh／HashGrid／Volume／BVH 的 id 只是整數，不會保住 Python 物件——若只存 id 而丟掉物件參考，物件可能被 GC 回收導致 kernel 讀到懸空記憶體。務必在使用期間保留這些物件的參考。

亂數生成#

kernel 內先用 wp.rand_init(seed, offset) 初始化 RNG 狀態（PCG 演算法），再呼叫 wp.randf() 等亂數函式：

@wp.kernel
def rand_kernel(seed: int, out_rand: wp.array[float]):
    i = wp.tid()
    rng = wp.rand_init(seed, i)   # offset 用執行緒索引，確保每條執行緒序列不同
    out_rand[i] = wp.randf(rng)
  • RNG 狀態會在每次呼叫時內部更新——連續呼叫產生不同的數。
  • 避免相關序列：不同 kernel 或同一 kernel 的多次 launch 若用同一個 seed，會產生相同／相關的亂數——每次 launch 遞增 seed。
