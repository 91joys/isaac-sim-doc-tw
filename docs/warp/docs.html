NVIDIA Warp 總覽文檔#

整理自官方文檔首頁與 Basics、FAQ 章節，依據 Warp 1.14.0（stable）。

Warp 是什麼#

Warp 是一個把普通 Python 函式 JIT 編譯成高效能 kernel 程式碼的 Python 框架，編譯出的 kernel 可在 CPU 或 GPU 上執行，速度接近原生 C++/CUDA。它為 GPU 加速的模擬、機器人學與機器學習而設計，有三個核心賣點：

  • kernel 級程式設計：你控制的是「每一條執行緒做什麼」，而非整個張量的運算——對模擬與圖學中常見的稀疏、條件分支、異質工作負載遠比張量模型方便。
  • 內建空間計算原語：三角網格（Mesh）、稀疏體積（NanoVDB Volume）、雜湊網格（HashGrid）、BVH、點雲、USD 資料都是一等公民，物理模擬與幾何處理開箱即用。
  • 全程可微分：kernel 可自動生成反向傳播版本，用 wp.Tape() 記錄計算圖後即可 tape.backward()，與 PyTorch／JAX／Paddle 組成端到端可微分的機器學習管線。

最小可執行範例——把 1024 個向量內積跑上 GPU：

import warp as wp
import numpy as np

@wp.kernel
def simple_kernel(a: wp.array[wp.vec3],
                  b: wp.array[wp.vec3],
                  c: wp.array[float]):
    tid = wp.tid()          # 取得執行緒索引
    c[tid] = wp.dot(a[tid], b[tid])

n = 1024
a = wp.array(np.random.rand(n, 3), dtype=wp.vec3, device="cuda")
b = wp.array(np.random.rand(n, 3), dtype=wp.vec3, device="cuda")
c = wp.zeros(n, dtype=float, device="cuda")

wp.launch(simple_kernel, dim=n, inputs=[a, b, c], device="cuda")
print(c.numpy())            # 自動同步並取回 CPU

三個核心概念：kernel、launch、array#

概念角色重點
@wp.kernel計算單元Python 函式 → C++/CUDA kernel，與 CUDA kernel 一一對應；參數必須有型別註記，不能存取全域 Python 狀態
wp.launch()啟動 kernel指定 dim（1D–4D 執行緒網格）、inputs 與 device；kernel 內用 wp.tid() 取得執行緒索引
wp.array資料容器固定大小、最多 4 維的同質容器，住在 CPU 或 GPU 記憶體；.numpy() 取回主機端並自動同步

與 CPython 最大的差異是強型別：所有變數都有型別（由運算式與函式簽名推斷）、內建型別採值語義（a = b 是複製不是參考）、不支援 tuple 初始化與 Python list——小型集合用 wp.vec3 等向量型別，大型集合用 wp.zeros()。詳見基礎入門。

編譯模型#

Warp 採 Python → C++/CUDA 的 JIT 編譯模型：

  1. 同一個 Python 模組內所有 @wp.kernel 在第一次 launch 時整批編譯成動態函式庫與 PTX。
  2. 編譯結果以模組雜湊為鍵快取（如 ~/.cache/warp/1.14.0），程式重啟後載入僅需數十毫秒——首次 (compiled)，之後 (cached)。
  3. 使用 C++/CUDA 作為中間表示，因此很容易在 kernel 中掛接低階常式與現有 C++ 函式庫。

進階的靜態運算式（wp.static()）、動態 kernel 建立與 AOT 編譯見泛型與程式碼生成。

與其他 Python GPU 框架的關係#

框架模型與 Warp 的差異
Numbakernel 級Numba 支援較大的 Python 子集；Warp 則提供 kernel 自動微分（機器學習關鍵）、以 C++/CUDA 為中間表示、內建幾何資料結構
Taichikernel 級概念相近；Warp 用 C++/CUDA 中間表示便於掛接 C++ 函式庫，且網格／稀疏體積／USD 是一等公民
CuPy張量級CuPy 是 GPU 版 NumPy（陣列整體運算）；Warp 讓你控制個別執行緒
PyTorch／JAX張量級非競爭關係：神經網路推論等張量運算用 PyTorch／JAX，稀疏／條件邏輯／異質負載用 Warp kernel，兩者以 __cuda_array_interface__ 零拷貝共享資料（見互操作性）

一句話定位：需要「對每條執行緒的控制權」與「可微分」時用 Warp；純張量運算留給 PyTorch／JAX；Warp 不是物理引擎的替代品——需要完整物理引擎請用建構於 Warp 之上的 Newton。

生態系：誰在用 Warp#

  • Newton Physics：GPU 加速可微分物理引擎（Disney Research／Google DeepMind／NVIDIA），整個引擎以 Warp 撰寫——本站另有 Newton 中文文檔。
  • MuJoCo Warp：Google DeepMind 與 NVIDIA 維護的 GPU 最佳化 MuJoCo 物理模擬器——本站有完整中文整理：總覽與使用、效能調校、差異與 FAQ。
  • Rewarped:平行可微分多物理模擬的強化學習平台。
  • XLB：Autodesk 維護的 Lattice Boltzmann 流體求解器（Warp 後端）。
  • warp-mpm：MPM 模擬器，用於 PhysGaussian 等生成動力學研究。

Warp 也整合於 NVIDIA Omniverse（omni.warp extension），Isaac Sim 中亦可直接使用。
