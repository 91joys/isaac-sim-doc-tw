互操作性#

整理自官方 Interoperability、PyTorch Interoperability 與 JAX Interoperability 章節。

快速對照#

Warp 透過標準協定（__array__、__array_interface__、__cuda_array_interface__）與其他框架互通，NumPy／CuPy／PyTorch／JAX／Paddle 都適用：

框架轉換函式零拷貝梯度感知
NumPywp.from_numpy／array.numpy()僅 CPU否
PyTorchwp.from_torch／wp.to_torch是是
JAXwp.from_jax／wp.to_jax是經 jax_kernel
Paddlewp.from_paddle／wp.to_paddle是是
CuPy／Numba__cuda_array_interface__ 協定是否
DLPackwp.from_dlpack／wp.to_dlpack是否

最快上手：外部陣列直接傳給 kernel#

實作了 __array_interface__（CPU）或 __cuda_array_interface__（GPU）的物件可以不經轉換直接放進 wp.launch() 的 inputs：

@wp.kernel
def saxpy(x: wp.array[float], y: wp.array[float], a: float):
    i = wp.tid()
    y[i] = a * x[i] + y[i]

# NumPy 陣列直接傳入（CPU）
x = np.arange(n, dtype=np.float32)
y = np.ones(n, dtype=np.float32)
wp.launch(saxpy, dim=n, inputs=[x, y, 1.0], device="cpu")

# CuPy／PyTorch 張量直接傳入（GPU;陣列所在裝置需與 launch 裝置一致）
x = cp.arange(n, dtype=cp.float32)
y = cp.ones(n, dtype=cp.float32)
wp.launch(saxpy, dim=n, inputs=[x, y, 1.0], device="cuda:0")

PyTorch#

t = wp.to_torch(w)     # Warp → Torch（零拷貝）
w = wp.from_torch(t)   # Torch → Warp（零拷貝）

# 裝置與 dtype 對映工具
torch_device = wp.device_to_torch("cuda:0")
torch_dtype = wp.dtype_to_torch(wp.float32)
warp_dtype = wp.dtype_from_torch(t.dtype)
  • 梯度自動橋接：轉換時 requires_grad 陣列的梯度 buffer 也一併轉換——Warp 陣列可直接參與 PyTorch autograd,如把 wp.to_torch(xs) 交給 torch.optim.Adam,用 wp.Tape 算梯度、Adam 更新。
  • 接進 autograd 圖:定義 torch.autograd.Function,在 forward() 內 launch Warp kernel、backward() 內用 tape 反向;包在 torch.compile() 中會自動視為 opaque callable。
  • 共用 CUDA Graph:PyTorch 與 Warp 操作可以錄進同一個 graph,但兩者必須在同一條非預設 stream 上——用 wp.stream_from_torch()／wp.stream_to_torch() 互換 stream 後再 ScopedCapture。

效能調校#

  • wp.from_torch() 雖零拷貝但有 CPU 開銷——轉換一次、重複使用，不要在迴圈內反覆轉換。
  • 無法重用時（每輪都產生新張量），傳 return_ctype=True 取得低階陣列描述子——跳過建構 wp.array 物件，官方基準快約 2 倍;描述子可直接傳給 kernel，但不能用在其他需要 wp.array 的地方。

JAX#

warp_array = wp.from_jax(jax_array)
jax_array = wp.to_jax(warp_array)

殺手級功能——jax_kernel() 把 Warp kernel 變成 JAX primitive，可在 @jax.jit 函式內呼叫：

from warp import jax_kernel

@wp.kernel
def triple_kernel(input: wp.array[float], output: wp.array[float]):
    tid = wp.tid()
    output[tid] = 3.0 * input[tid]

jax_triple = jax_kernel(triple_kernel)

@jax.jit
def f():
    x = jnp.arange(0, 64, dtype=jnp.float32)
    return jax_triple(x)
  • kernel 簽名輸入在前、輸出在後，至少一個輸出陣列;多個輸出用 num_outputs 指定。
  • 可微分版本支援 JAX 的自動微分（forward／reverse）;另支援 sharding／多 GPU 場景。
  • 追求極致效能與 stream 同步控制時，可改用底層 DLPack 協定。

Paddle・DLPack・CuPy／Numba#

  • Paddle：wp.from_paddle／wp.to_paddle 零拷貝且梯度感知（可把 Warp 陣列交給 paddle.optimizer.Adam）;stream 互換用 wp.stream_from_paddle()。效能調校原則同 PyTorch。
  • DLPack：跨框架標準張量協定——wp.from_dlpack()／wp.to_dlpack()，適合需要精細控制的場景。
  • CuPy／Numba：純協定層互通——Warp GPU 陣列暴露 __cuda_array_interface__，雙向皆可直接使用。
  • NumPy：array.numpy() 對 CPU 陣列回傳零拷貝 view、對 GPU 陣列先複製回主機;np.array(warp_array) 也可直接建構。dtype 對映用 wp.dtype_to_numpy()。

提醒：從 __cuda_array_interface__ 建構 Warp 陣列時，該協定不含裝置資訊——務必明確指定 device，確保與資料實際所在的 GPU 一致。
