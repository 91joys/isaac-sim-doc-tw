基礎入門#

整理自官方 Basics 章節。官方另提供 NVIDIA Accelerated Computing Hub 上的 Colab 教學筆記本（Warp 入門、Ising 模型模擬）。

初始化#

第一次呼叫 wp.launch() 等 Warp 函式時，Warp 會自動初始化並印出可用裝置、驅動版本與 kernel 快取位置。也可以用 wp.init() 明確初始化：

import warp as wp

wp.init()

Kernel#

計算 kernel 就是加上 @wp.kernel 裝飾器的 Python 函式，概念上與 CUDA kernel 一一對應——launch 時 kernel 本體會被平行執行指定次數：

import warp as wp

@wp.kernel
def simple_kernel(a: wp.array[wp.vec3],
                  b: wp.array[wp.vec3],
                  c: wp.array[float]):
    tid = wp.tid()          # 取得執行緒索引
    x = a[tid]              # 讀取兩個 vec3
    y = b[tid]
    r = wp.dot(x, y)        # 內積
    c[tid] = r              # 寫回結果

要點：

  • kernel 會編譯成原生 C++/CUDA 程式碼，因此所有輸入參數都必須有靜態型別註記。
  • kernel 可能在 CPU 或 GPU 上執行，不能存取任意的 Python 全域狀態——資料一律透過參數（陣列）進出。
  • kernel 內可用的內建函式（數學、向量、四元數、幾何……）見官方 Built-Ins。

用 wp.launch() 啟動 kernel，支援 1D–4D 執行緒網格：

# 1D：1024 條執行緒
wp.launch(kernel=simple_kernel,  # 要啟動的 kernel
          dim=1024,              # 執行緒數
          inputs=[a, b, c],      # 參數
          device="cuda")         # 執行裝置

# 2D：處理 1024×1024 影像
wp.launch(kernel=compute_image, dim=(1024, 1024), inputs=[img], device="cuda")

多維網格在 kernel 內用多重指派取得索引：

@wp.kernel
def compute_image(pixel_data: wp.array2d[wp.vec3]):
    i, j = wp.tid()

陣列（wp.array）#

wp.array 是固定大小的多維容器（最多 4 維），存放同質的 Warp 資料型別元素，可住在主機（CPU）或裝置（GPU）記憶體。配置方式與 NumPy／PyTorch 類似：

# 未初始化的 vec3 陣列
v = wp.empty(shape=n, dtype=wp.vec3, device="cuda")

# 零初始化的四元數陣列
q = wp.zeros(shape=n, dtype=wp.quat, device="cuda")

# 從 NumPy 初始化（自動搬到指定裝置）
a = np.ones((10, 3), dtype=np.float32)
v = wp.from_numpy(a, dtype=wp.vec3, device="cuda")
  • kernel 參數註記用 wp.array2d、wp.array3d、wp.array4d 別名表達維度。
  • 從外部資料（NumPy、list、tuple）建立時預設會複製;若輸入連續且在同一裝置，可用 copy=False 直接別名外部記憶體（見互操作性）。
  • array.numpy() 把 GPU 資料取回 CPU——會自動同步等待未完成的工作;對 CPU 陣列呼叫則回傳零拷貝的 NumPy view。

常用運算子 + - * / 對陣列已有多載：

a = wp.array(np.arange(10), dtype=wp.float32)
b = wp.array(np.arange(10), dtype=wp.float32)
c = 2.0 * a + b     # 逐元素運算
c *= 10.0           # 就地運算
print(c)
# [  0.  30.  60.  90. 120. 150. 180. 210. 240. 270.]

更進一步的逐元素映射見 warp.map()；陣列的索引、切片與結構化用法見執行期核心功能。

使用者函式（@wp.func）#

用 @wp.func 定義可被 kernel 呼叫的函式（同模組或跨模組皆可），回傳型別註記可省略：

@wp.func
def square(x: float):
    return x * x

# 可接受陣列與 struct
@wp.func
def lookup(foos: wp.array[wp.uint32], index: int):
    return foos[index]

# 可回傳多個值
@wp.func
def multi_valued_func(a: wp.float32, b: wp.float32):
    return a + b, a - b, a * b, a / b

# 可用同名多簽名多載
@wp.func
def custom(x: int):
    return x + 1

@wp.func
def custom(x: float):
    return x + 1.0
  • kernel 能做的事函式幾乎都能做，唯一例外是 wp.tid()——需要執行緒索引就從參數傳入。
  • @wp.func 也能直接從 Python 呼叫（實驗性）：從 kernel 呼叫時由 Warp 編譯，從 Python 呼叫時由直譯器執行，因此後者不能用 tile API 等 kernel 專屬功能。
  • tile 參數建議用 typing.Any 讓編譯器自動推斷型別（見泛型）。

使用者結構（@wp.struct）#

@wp.struct
class MyStruct:
    pos: wp.vec3
    vel: wp.vec3
    active: int
    indices: wp.array[int]

所有屬性在類別定義時就要有合法型別註記。struct 可以作為 wp.array 的 dtype，也可以直接當 kernel 參數傳入。細節與限制見執行期核心功能與 Limitations。

Python scope 與 kernel scope#

Warp API 分成兩個層次：

Python scopeKernel scope
在哪裡呼叫一般 Python 程式碼（kernel／函式之外）@wp.kernel／@wp.func 內
文檔warp API ReferenceBuilt-Ins
例子wp.launch、wp.zeros、wp.copy、wp.Tapewp.tid、wp.dot、wp.atomic_add、wp.mesh_query_point

kernel-scope API 一般也能在 Python scope 使用，但反之不行。kernel 內不支援完整的 Python 語言——沒實作的功能與「上了 GPU 就是效能陷阱」的功能（動態型別、list comprehension、例外、GC……）都被排除。

編譯模型與 kernel 快取#

Warp 採 Python → C++/CUDA 編譯模型：同一 Python 模組的全部 kernel 在該模組第一次 launch 時編譯成動態函式庫與 PTX，並在應用程式重啟間快取。每次模組載入會印出：模組名、雜湊前 7 碼、目標裝置、耗時，以及 (compiled)／(cached)／(error) 狀態：

# 第一次執行——完整編譯
Module __main__ 0b0ecab load on device 'cuda:0' took 4136.19 ms  (compiled)

# 第二次執行——從快取載入
Module __main__ 0b0ecab load on device 'cuda:0' took 30.98 ms  (cached)

設 wp.config.log_level = wp.LOG_DEBUG 可在每次模組載入開始時也印出訊息。更深入的程式碼生成機制見泛型與程式碼生成。

語言特性：與 CPython 的差異#

內建型別與值語義#

Warp 內建類似高階著色語言的數學型別：vec2/vec3/vec4、mat22/mat33/mat44、quat、array 等。所有內建型別採值語義——a = b 產生的是 b 的複本，不是參考。

強型別#

所有變數都必須有型別（由來源運算式與函式簽名推斷）；kernel 參數必須加型別註記。方便起見，函式簽名可用 typing.Any 代替具體型別（見泛型）：

from typing import Any

@wp.kernel
def generic_kernel(a: wp.array[Any],
                   b: wp.array[Any],
                   c: Any):
    ...

不支援 tuple 初始化與 list#

# ✗ 無效
a = (1.0, 2.0, 3.0)
my_data = [1.0, 2.0, 3.0]

# ✓ 有效：向量型別（2–4 個元素）
a = wp.vec3(1.0, 2.0, 3.0)

# ✓ 有效：固定大小的堆疊陣列（支援索引與切片）
my_data = wp.zeros(shape=16, dtype=float)
