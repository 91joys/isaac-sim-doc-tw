安裝與快速入門#

整理自官方 Installation 與 Compatibility & Support 章節。

系統需求#

作業系統架構平台支援狀態加速器
Windows 10/11x86-64PC✅ 支援CUDA
Windows 11arm64PC規劃中CUDA（規劃中）
Linuxx86-64PC／伺服器✅ 支援CUDA
Linuxaarch64Jetson／伺服器✅ 支援CUDA
macOSarm64（Apple Silicon）Mac✅ 支援僅 CPU
macOSx86-64（Intel）Mac已終止（用 1.9.x 以前版本）無
  • Python 3.10 以上；必要相依套件只有 NumPy。
  • GPU 加速需要支援 CUDA 的 NVIDIA GPU:CUDA 12.x 套件最低 Maxwell（GTX 9xx、sm_52）,CUDA 13.x 套件最低 Turing（RTX 20xx、sm_75）。
  • 驅動需求由「建置 Warp 所用的 CUDA Toolkit 版本」決定:CUDA 12.x 建置需驅動 525+,CUDA 13.x 建置需驅動 580+。PyPI 上的 wheel 目前以 CUDA 12.9 建置。
  • 沒有 GPU 也能用——所有平台都支援純 CPU 執行。
  • float16 原子操作需要 compute capability 7.0+（Volta）,較舊 GPU 上會回傳零。

pip 安裝（推薦）#

# 標準安裝（PyPI，CUDA 12.9 runtime）
pip install warp-lang

# 加裝範例程式的選用相依（usd-core、pyglet 等）
pip install "warp-lang[examples]"

其他安裝管道：

# Nightly build（main 分支、CUDA 12 runtime，無 macOS 版）
pip install -U --pre warp-lang --extra-index-url=https://pypi.nvidia.com/

# conda-forge（預設裝最新 CUDA 變體）
conda install conda-forge::warp-lang
conda install "conda-forge::warp-lang=*=*cpu*"      # 純 CPU 變體
conda install "conda-forge::warp-lang=*=*cuda129*"  # 指定 CUDA 12.9

# CUDA 13 runtime 版 wheel（GitHub Releases，此為 Windows x86-64 例）
pip install https://github.com/NVIDIA/warp/releases/download/v1.14.0/warp_lang-1.14.0+cu13-py3-none-win_amd64.whl

驅動太舊時 Warp 初始化會發出 Insufficient CUDA driver version 警告，CUDA 裝置不可用但 CPU 仍可執行。解法：更新驅動、改裝相容的預建套件，或用相容的 CUDA Toolkit 從原始碼建置。另外 Tile 的完整 MathDx 功能需要 CUDA 12.6.3+。

快速入門：第一個 kernel#

安裝完成後，先確認 Warp 能找到你的裝置：

import warp as wp
wp.init()

Warp 1.14.0 initialized:
CUDA Toolkit 12.9, Driver 12.9
Devices:
    "cpu"      : "x86_64"
    "cuda:0"   : "NVIDIA GeForce RTX 3090" (24 GiB, sm_86, mempool enabled)
Kernel cache:
    C:\Users\you\AppData\Local\NVIDIA\warp\Cache\1.14.0

接著寫第一個 kernel——計算 1024 個向量的內積：

import numpy as np
import warp as wp

@wp.kernel
def dot_products(a: wp.array[wp.vec3],
                 b: wp.array[wp.vec3],
                 c: wp.array[float]):
    tid = wp.tid()
    c[tid] = wp.dot(a[tid], b[tid])

n = 1024
a = wp.array(np.random.rand(n, 3), dtype=wp.vec3, device="cuda")
b = wp.array(np.random.rand(n, 3), dtype=wp.vec3, device="cuda")
c = wp.zeros(n, dtype=float, device="cuda")

wp.launch(dot_products, dim=n, inputs=[a, b, c], device="cuda")
print(c.numpy())   # .numpy() 會自動同步 GPU 並取回資料

第一次執行時模組會被編譯（訊息顯示 (compiled)，可能需要數秒）；之後執行直接從快取載入（(cached)，數十毫秒）。語法細節見基礎入門。

內建範例庫#

warp/examples 目錄涵蓋物理模擬、幾何處理、最佳化與 tile GPU 程式設計，安裝 warp-lang[examples] 後從命令列直接執行：

python -m warp.examples.<子目錄>.<範例名>

# 例如：
python -m warp.examples.core.example_fluid      # 2D 流體
python -m warp.examples.core.example_sph        # SPH 粒子
python -m warp.examples.fem.example_diffusion_3d
python -m warp.examples.optim.example_diffray   # 可微分光線追蹤
python -m warp.examples.tile.example_tile_mlp   # tile MLP
子目錄主題代表範例
core核心功能DEM、流體、CUDA Graph 擷取、marching cubes、mesh 光線投射、SPH、NanoVDB、波動方程
fem有限元素3D 擴散、混合彈性、APIC 流體、磁靜力學、自適應網格
optim可微分最佳化可微分光線追蹤（diffray）、流體 checkpoint、粒子斥力
tileTile 程式設計MLP 推論、N-body、Monte Carlo 幾何處理

從原始碼建置#

需求：Visual Studio 2019+（Windows）／GCC 9.4+（Linux）、Python 開發標頭、CUDA Toolkit 12.0+、Git LFS。

git clone https://github.com/NVIDIA/warp.git
cd warp
python build_lib.py        # 產出 warp/bin/ 平台二進位檔
pip install -e .           # -e 讓後續修改即時生效
  • CUDA Toolkit 路徑用 CUDA_PATH 環境變數或 --cuda-path="..." 覆寫。
  • 預設靜態連結 CUDA 函式庫（自足的二進位檔）；--use-dynamic-cuda 改為動態連結。
  • --quick 快速建置會跳過 CUDA 向後相容層，最低驅動需求改由 Toolkit 版本決定。
  • Linux ARM＋CUDA 12.9 建置有已知 bug，無法涵蓋 Maxwell／Pascal／Volta 架構;需要時改用 12.9 以前的 Toolkit。

Conda 環境注意：建置環境的 C++ runtime 版本必須「等於或舊於」執行環境，否則會出現 GLIBCXX_3.4.30 not found 之類錯誤——可在執行環境 conda install -c conda-forge libstdcxx-ng=12.1 解決。

Docker 與 Omniverse#

  • Docker：安裝 NVIDIA Container Toolkit 後以 docker run --gpus all 讓容器看得到 GPU。建議以 nvidia/cuda 的 devel 映像為基底建置、runtime 映像部署（多階段建置可大幅縮小映像）。官方文檔附完整 Dockerfile 範例。
  • Omniverse：Omniverse Kit 內建 omni.warp extension，在 Isaac Sim 等 Kit 應用中啟用即可使用 Warp（版本隨 Kit 發佈）。

版本與支援策略#

  • 版號 X.Y.Z 不採用語意化版本:Y 為每月例行的功能版（唯一可含棄用／破壞性變更的版本）,Z 為不定期的錯誤修正版。
  • 功能狀態分為 Experimental → Stable → Deprecated → Removed;棄用功能至少維護 4 個月（約 4 個功能版）,期間顯示 DeprecationWarning。
  • Nightly 版格式 X.Y.Z.devYYYYMMDD,發佈於 NVIDIA PyPI index,不受穩定版相容性保證。
