領域模組#

整理自官方 Domain Modules 章節：Sparse Matrices、FEM Toolkit 與 Rendering。

warp.sparse：稀疏矩陣#

warp.sparse 實作模擬常用的稀疏線性代數,核心是 BSR（Block Sparse Row）矩陣——CSR 是 block 為 1×1 的特例。最常見的建構方式是 COO 座標三元組：

from warp.sparse import bsr_from_triplets, bsr_zeros, bsr_identity, bsr_diag

rows = wp.array([0, 1, 2], dtype=int)
cols = wp.array([1, 2, 3], dtype=int)
vals = wp.array([...], dtype=float)

A = bsr_from_triplets(rows_of_blocks=2, cols_of_blocks=2,
                      rows=rows, columns=cols, values=vals)

# 特殊矩陣
Z = bsr_zeros(4, 4, block_type=wp.float32)      # 1×1 block = CSR
B = bsr_zeros(2, 2, block_type=wp.mat33)        # 3×3 block
I = bsr_identity(rows_of_blocks=4, block_type=wp.float32)
D = bsr_diag(diag=wp.array([1.0, 2.0, 3.0, 4.0]))
  • block 可為矩形（wp.mat23）;bsr_copy()／bsr_assign() 可轉換 block 大小（需整除相容）與純量型別。
  • 支援矩陣加減、縮放、轉置、矩陣乘法與 matrix-vector 乘積等常用運算。

迭代線性求解器#

warp.optim.linear 提供四種迭代求解器（皆可加前置條件器）：CG（共軛梯度）、CR（共軛殘差）、BiCGSTAB、GMRES：

from warp.optim.linear import cg

x = cg(A, b, x=x, maxiter=100, tol=1e-6)   # 解 Ax = b

除稀疏矩陣外也接受稠密 2D Warp 陣列，或用 LinearOperator 介面自訂運算子。

warp.fem：有限元素工具組#

warp.fem 用 Galerkin 有限元素法求解微分方程描述的物理系統（擴散、對流、流體、彈性……），設計目標是快速實驗各種 FEM 形式與離散化方案。

Integrand（積分形式）#

from warp.fem import integrand, Sample, Domain, Field, grad

@integrand
def linear_form(s: Sample, domain: Domain, v: Field):
    x = domain(s)                        # 取樣點的世界座標
    return v(s) * wp.max(0.0, 1.0 - wp.length(x))

@integrand
def diffusion_form(s: Sample, u: Field, v: Field, nu: float):
    return nu * wp.dot(grad(u, s), grad(v, s))

integrand 是普通的 Warp kernel（可呼叫任意 Warp 函式），但接受三種特殊參數：Sample（目前積分取樣點）、Field（抽象場，呼叫時代換為離散場／test／trial 場，以 u(s) 求值）、Domain（抽象積分域，提供 normal() 等運算子）。integrand 不能直接 wp.launch()，要經由 integrate()／interpolate() 呼叫，Field 參數用 fields={...}、一般參數用 values={...} 傳入。

基本工作流程#

  1. Geometry：2D／3D 規則網格、NanoVDB 體積、三角／四邊形／四面體／六面體非結構網格。
  2. FunctionSpace：make_polynomial_space() 為幾何配上形函數——Lagrange（連續／不連續）、Serendipity（k≤3）、Nédélec／Raviart-Thomas 向量形函數、B-spline。
  3. 積分域：Cells（單元）或 BoundarySides（邊界）。
  4. 線性形式 → 右端項：make_test() 定 test 函數，integrate() 得 wp.array。
  5. 雙線性形式 → 系統矩陣：make_trial() 定 trial 函數，integrate() 得 BsrMatrix——test／trial 可屬不同空間（Mixed FEM）。
  6. 求解：交給上述迭代求解器。
# 節錄自 warp/examples/fem/example_diffusion.py
geo = Grid2D(res=wp.vec2i(resolution))
scalar_space = make_polynomial_space(geo, degree=degree)

domain = Cells(geometry=geo)
test = make_test(space=scalar_space, domain=domain)
rhs = integrate(linear_form, fields={"v": test})

trial = make_trial(space=scalar_space, domain=domain)
matrix = integrate(diffusion_form, fields={"u": trial, "v": test},
                   values={"nu": viscosity})

x = cg(matrix, rhs, ...)

進階功能：自適應網格（adaptivity）、粒子式積分（PicQuadrature，支援 APIC 流體）、非協調接觸、場運算子（divergence、curl……）與視覺化輔助。入門範例：example_diffusion.py、example_diffusion_3d.py、example_mixed_elasticity.py 等（見範例庫）。

warp.render：渲染#

元件用途
OpenGLRenderer互動式即時播放——主要用於除錯（需 pyglet）
UsdRenderer把場景輸出成 USD 檔，再交給外部 3D 應用（Omniverse、Blender……）做精緻渲染（需 usd-core）
wp.RegisteredGLBufferCUDA 圖形互通——把 OpenGL buffer 註冊給 CUDA，直接映射為 Warp 陣列在 kernel 中讀寫，免 CPU↔GPU 來回複製

warp.sim 呢？#

舊版 Warp 內建的 warp.sim 物理模擬模組已棄用並移出——其功能由 Newton Physics 承接（同樣建構於 Warp、由 Disney Research／Google DeepMind／NVIDIA 維護）。物理模擬需求請參考本站的 Newton 中文文檔，遷移指南見 Newton FAQ・整合・遷移。
