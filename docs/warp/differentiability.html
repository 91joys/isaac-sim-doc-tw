可微分性#

整理自官方 Differentiability 章節——Warp 最獨特的能力：每個 kernel 都自動有反向版本。

wp.Tape：記錄與反向傳播#

Warp 預設為每個 kernel 生成前向與反向（adjoint）兩個版本。反向 kernel 可用來計算損失函數的梯度，再回傳給 PyTorch 等機器學習框架。用法三步驟：

  1. 參與梯度計算的陣列以 requires_grad=True 建立;
  2. 用 wp.Tape 記錄 kernel launch;
  3. tape.backward(loss) 反向傳播，之後從 array.grad 讀梯度。
a = wp.zeros(1024, dtype=wp.vec3, device="cuda", requires_grad=True)

tape = wp.Tape()

# 前向:記錄計算
with tape:
    wp.launch(kernel=compute1, inputs=[a, b], device="cuda")
    wp.launch(kernel=compute2, inputs=[c, d], device="cuda")
    wp.launch(kernel=loss, inputs=[d, l], device="cuda")

# 反向傳播
tape.backward(l)

# 損失對輸入 a 的梯度
print(a.grad)
  • 重複使用同一批 buffer 做多次反向傳播前，先呼叫 tape.zero() 歸零梯度。
  • 輸出梯度在反向過程中預設會被消耗（歸零）;要在 backward 後檢查中間陣列的梯度，建立時加 retain_grad=True——但僅限每個元素最多被寫一次的陣列，否則梯度會被重複計算。

陣列覆寫陷阱#

自動微分需要保存前向的中間結果供反向使用;覆寫先前結果會導致錯誤的梯度。PyTorch／JAX 每個運算都隱式配置新記憶體來迴避這個問題;Warp 的記憶體由使用者管理——當輸出陣列被後續 launch 覆寫時，Warp 只會正確傳播「最後一次寫入」的梯度（與 PyTorch／JAX 行為一致）。習慣寫法：每個 kernel 的輸出用獨立陣列，如同 PyTorch 的 y = f(x) 隱式配置 y：

@wp.kernel
def kernel_func(x: wp.array[float], y: wp.array[float]):
    tid = wp.tid()
    y[tid] = x[tid] ** 2.0 + 3.0 * x[tid] + 1.0

x = wp.array([1.0, 2.0, 3.0], dtype=float, requires_grad=True)
y = wp.zeros_like(x)   # 輸出用獨立陣列

tape = wp.Tape()
with tape:
    wp.launch(kernel_func, x.shape, inputs=[x], outputs=[y])
tape.backward(grads={y: wp.ones_like(x)})

print(x.grad)   # [5. 7. 9.]

開發時可開啟覆寫追蹤（wp.config.verify_autograd_array_access = True），Warp 會在偵測到「先寫後讀以外的覆寫模式」時發出警告。另外 wp.copy()、wp.clone() 都是可微分的，梯度會流過複製操作。

Jacobian 計算#

對多值函數 f: Rⁿ→Rᵐ，把選擇向量 e 傳給 tape.backward(grads={output: e}) 即可一次算出 Jacobian 的一整列（Jᵀe）：

jacobian = np.empty((output_dim, input_dim), dtype=np.float32)

tape = wp.Tape()
with tape:
    output_buffer = launch_kernels_to_be_differentiated(input_buffer)

for output_index in range(output_dim):
    # 選擇要微分的輸出列
    select_index = np.zeros(output_dim)
    select_index[output_index] = 1.0
    e = wp.array(select_index, dtype=wp.float32)

    tape.backward(grads={output_buffer: e})
    jacobian[output_index, :] = tape.gradients[input_buffer].numpy()
    tape.zero()   # 下一列前歸零

多個獨立環境平行模擬時，整體 Jacobian 是塊對角矩陣——把選擇向量 np.tile(select_index, num_envs) 展開，就能一次算出所有環境的同一列（RL 常用技巧）。

kernel 內的 wp.grad()#

若只需要「單一 Warp 函式」的偏導數，wp.grad(func) 比 tape 高效——單次 launch 內同時算前向值與整列 Jacobian，不用每列跑一次反向：

@wp.func
def squared_norm(x: float, y: float):
    return x * x + y * y

@wp.kernel(enable_backward=False)
def compute_jacobian(x: wp.array[float], y: wp.array[float],
                     z: wp.array[float], J: wp.array2d[float]):
    i = wp.tid()
    z[i] = squared_norm(x[i], y[i])            # 前向
    grad_x, grad_y = wp.grad(squared_norm)(x[i], y[i])  # 偏導數
    J[i, 0] = grad_x
    J[i, 1] = grad_y

自訂梯度與 replay 函式#

鏈式法則 h′(x) = f′(g(x)) · g′(x) 中，函式需要提供前向重演版本（replay，預設同原定義）與導數（grad），兩者都可以自訂覆蓋自動生成的版本。典型應用——擋掉非有限梯度：

@wp.func
def safe_sqrt(x: float):
    return wp.sqrt(x)

# 自動微分在 x=0 會得到 inf 梯度;自訂 grad 函式擋掉它:
@wp.func_grad(safe_sqrt)
def adj_safe_sqrt(x: float, adj_ret: float):
    if x > 0.0:
        wp.adjoint[x] += 1.0 / (2.0 * wp.sqrt(x)) * adj_ret

# xs.grad: [0.5, 0.354, 0.]   ← x=0 處不再是 inf
  • 自訂 grad 的簽名＝前向函式的輸入＋各輸出的 adjoint 變數;用 wp.adjoint[參數] += ... 累加偏導數。
  • @wp.func_replay(myfunc) 自訂反向過程中重演前向的方式（例如避免重算昂貴的中間值）。
  • 限制：泛型參數（Any）的函式暫不支援自訂 replay／grad。
  • 更底層的 custom native functions 允許直接嵌入 C++/CUDA 程式碼並提供對應的 adjoint 實作。

梯度除錯#

  • warp.autograd.gradcheck()：以有限差分驗證 kernel 的自動微分梯度;gradcheck_tape() 驗證整卷 tape。
  • jacobian()／jacobian_fd()：算出 kernel 所有輸入／輸出組合的 Jacobian（autodiff 與有限差分版本），並可視覺化比較。
  • tape.visualize()：輸出 GraphViz dot 檔，檢視記錄下來的計算圖。
  • 覆寫追蹤：wp.config.verify_autograd_array_access = True 自動偵測會破壞梯度的陣列覆寫。

限制與對策#

Warp 用原始碼轉換做自動微分，反向需要前向的中間值，因此可微分 kernel 有幾條規則：

限制說明
就地乘除不可微+=／-= 可微;*=／/= 反向會得到錯誤結果（wp.config.log_level = wp.LOG_DEBUG 時編譯會警告）
向量／矩陣／四元數分量賦值區域變數的每個分量只能賦值一次;之後可用 +=／-= 更新，但重新 = 賦值會破壞梯度
動態迴圈反向不會重演／展開動態迴圈——adjoint 若依賴迴圈中間值（如連乘），梯度會錯;純累加（如連加）沒問題。對策：迴圈次數改用編譯期常數讓 Warp 展開，或把迴圈拆成多次 kernel launch

相關工具：warp.optim 提供 Adam、SGD 等最佳化器直接搭配 Warp 梯度使用;非平滑函式（如 wp.abs() 在 x=0）的次梯度處理見 FAQ;與 PyTorch autograd 的橋接（torch.autograd.Function）見互操作性。
