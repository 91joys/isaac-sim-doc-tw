泛型與程式碼生成#

整理自官方 Generics 與 Code Generation 章節。

泛型 kernel#

用 typing.Any 代替具體型別，同一份程式碼即可支援多種資料型別（模板）：

from typing import Any

@wp.kernel
def scale(x: wp.array[Any], s: Any):
    i = wp.tid()
    x[i] = s * x[i]

x16 = wp.array(data, dtype=wp.float16)
x32 = wp.array(data, dtype=wp.float32)

# Warp 依參數型別自動生成對應的 kernel 實例
wp.launch(scale, dim=n, inputs=[x16, wp.float16(3)])
wp.launch(scale, dim=n, inputs=[x32, wp.float32(3)])

型別推斷注意：Python 的 42 推斷為 wp.int32、0.5 為 wp.float32——其他型別要明確用 wp.int64(42)、wp.float16(0.5);向量／矩陣用 Warp 型別（wp.vec3f(...)）而非 tuple。

隱式 vs. 顯式實例化#

每次用新型別組合 launch 泛型 kernel 都會隱式生成新實例並重載模組（每次一百多毫秒），且舊 CUDA 驅動上 graph 擷取期間不允許模組重載。用 @wp.overload 預先宣告全部實例可讓模組只載入一次：

@wp.overload
def scale(x: wp.array[wp.float16], s: wp.float16):
    ...   # 本體用省略號,不重複程式碼

# 或函式形式,可用迴圈批次宣告:
for T in [wp.float16, wp.float32, wp.float64]:
    wp.overload(scale, [wp.array[T], T])

# 回傳值是具體 kernel 實例——launch 更快(免型別推斷),參數也更寬鬆
scale_f32 = wp.overload(scale, [wp.array[wp.float32], wp.float32])
wp.launch(scale_f32, dim=n, inputs=[x32, 3])

泛型函式（@wp.func＋Any）同理;kernel 內可用 type() 運算子取得變數型別做轉型（如 type(x)(0)）。限制：泛型 kernel 無法自訂梯度函式、無法在某些場景序列化——細節見官方 Limitations and Rough Edges。

程式碼生成模型#

kernel 以 Python 模組為單位編譯——同模組所有 kernel 一起編譯（比逐個快），launch 時檢查模組是否過期。執行期新增 kernel 會改變模組，導致下次 launch 重新編譯;編譯結果以內容雜湊為鍵快取，重跑程式時直接載入（(cached)）。

外部參照與常數：kernel 參照的外部 Python 變數在程式碼生成時固化為常數。變數之後改變不會自動反映——需呼叫 kernel.module.mark_modified() 或重新定義 kernel。

靜態運算式 wp.static()#

wp.static(expr) 在 kernel 定義時執行任意 Python 運算式，把結果直接寫死進生成的程式碼。結果可以是純量、向量、struct、字串或 Warp 函式（不可為陣列）：

import scipy.linalg

@wp.kernel
def my_kernel():
    static_var = wp.static(3 + 2)                                  # 常數折疊
    static_norm = wp.static(wp.float64(scipy.linalg.norm([3, 4]))) # 可呼叫任意 Python
    wp.printf("static_norm = %f\n", static_norm)

三個殺手級用途：

  • 靜態條件消除：if wp.static("red" in available_colors): ——常數分支在生成的程式碼中被完全消除，零分支開銷，適合生成特化 kernel。
  • 靜態迴圈展開：range 為編譯期常數的迴圈自動展開，配合 wp.static() 可在展開的每一步做不同的事（如依索引選不同函式）。
  • 函式指標：wp.static(func_table[i]) 在生成時選擇要呼叫的 Warp 函式——實現「編譯期分派」。

動態 kernel 建立（閉包）#

用 Python 閉包在執行期以不同的常數、型別或函式特化 kernel：

# 以常數特化
def create_kernel_with_constant(constant):
    @wp.kernel
    def k(a: wp.array[float]):
        tid = wp.tid()
        a[tid] += constant
    return k

k1 = create_kernel_with_constant(17.0)
k2 = create_kernel_with_constant(42.0)

# 以型別特化
def create_kernel_with_dtype(vec_type):
    @wp.kernel
    def k(a: wp.array[vec_type]):
        tid = wp.tid()
        a[tid] += float(tid) * vec_type(1.0)
    return k

k2d = create_kernel_with_dtype(wp.vec2)
k4d = create_kernel_with_dtype(wp.vec4)

函式（把 @wp.func 當參數傳入）與 struct 同樣可以閉包捕獲。每個閉包 kernel 都是獨立實例，有自己的模組雜湊——不同特化並存不衝突。

AOT 編譯工作流程#

除了 JIT，Warp 也支援 ahead-of-time 編譯，用於部署與 CI：

  • 手動編譯模組：wp.get_module(...).load(device) 不需 launch 任何 kernel 即可觸發編譯——CI 中預熱快取。
  • 散佈預編譯模組：把 kernel 快取目錄隨應用程式打包，目標機器免編譯直接載入（模組雜湊需一致）。
  • 無 CUDA 驅動編譯:Docker build 階段沒有 GPU 也能編譯 CUDA 模組（設 wp.config.cuda_output="ptx" 等），執行階段才需要驅動。

詳細範例（含 Dockerfile）見官方 Ahead-of-Time Compilation Workflows。

模組與 kernel 設定#

編譯行為可在三個層級調整（Configuration）：

層級方式常用選項
全域wp.config.*（import 後立即設定）mode="debug"、verify_fp、verify_cuda、enable_backward、kernel_cache_dir
模組wp.set_module_options({...})fast_math、enable_backward、max_unroll
kernel@wp.kernel(enable_backward=False, ...)不需微分的 kernel 關掉反向生成可加速編譯、減少記憶體
